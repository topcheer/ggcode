INVESTIGATION SUMMARY AT A GLANCE
─────────────────────────────────────────────────────────────────────────────

ROOT CAUSE:
  Authority epoch mismatch (token renewal) triggers aggressive history reset
  in relay, wiping ALL messages, causing mobile clients to see blank chat.

THE BUG:
  File: internal/tunnel/broker.go
  Lines: 869-873
  Function: relayRecoveryPlan()
  Decision: if auth_mismatch then reset=true (if relay has history)

THE CASCADE:
  1. Token renewal → authority epoch changes
  2. Broker detects mismatch → sets reset=true
  3. Broker sends replace_history mode to relay
  4. Relay calls clearHistoryLocked() → r.history = nil
  5. All messages deleted
  6. Mobile reconnects, gets replay_count=0
  7. User sees blank chat

THE FIX:
  Remove authority epoch mismatch ALONE as reset trigger.
  Keep other validation (hash, count) to catch real corruption.

EFFORT:
  1-2 lines of code change
  Low risk (isolated decision logic)
  High impact (prevents silent data loss)
