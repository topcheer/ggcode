# ggcode v1.1.87

## Highlights

- **6 new IM adapters** — Matrix, Signal, IRC, Nostr, Twitch, and WhatsApp, bringing the total to 13 supported platforms
- **WhatsApp companion device pairing** — full QR code scanning flow with whatsmeow SDK, including device identity (Chrome), presence management, app state sync, and channel-bound message filtering

## New IM Adapters

- **Matrix** — REST `/sync` long polling, any homeserver, DM/thread/mention support
- **Signal** — SSE long polling + JSON-RPC to signal-cli daemon
- **IRC** — pure TCP + TLS, NickServ auth, auto-join channels
- **Nostr** — WebSocket relay, NIP-04 encrypted DM, real secp256k1 crypto
- **Twitch** — IRC on tmi.twitch.tv with Twitch-specific tag parsing
- **WhatsApp** — whatsmeow SDK, QR pairing via TUI panel, companion device protocol

## Fixes and improvements

### WhatsApp adapter
- Set `PlatformType=CHROME` to avoid "unable to link device" rejection by WhatsApp servers
- Fix SQLite driver name: modernc registers as `"sqlite"` (not `"sqlite3"`) with `_pragma=foreign_keys(1)`
- Send `PresenceAvailable` after connect so WhatsApp server pushes messages
- Fetch `CriticalBlock` and `CriticalUnblockLow` app state for encryption key sync
- Remove `IsFromMe` early return — own messages are now processed (required for solo groups and self-testing)
- Filter inbound by bound channel ID after pairing — messages from unrelated groups are silently dropped
- Bridge whatsmeow internal logs to ggcode debug log system (replaces `waLog.Noop`)

### UI
- QR code overlay panel shared across WhatsApp/QQ/WeChat adapters
- WhatsApp panel auto-refresh via tick for live QR code display
- Matrix adapter: intercept paste events in create/edit mode
- `edit_file` auto-indent matching improvement

### Test coverage
- 57 new unit tests for IRC, Nostr, Twitch, and Signal adapters
- PTY integration tests for QR overlay rendering across IM panels

## Upgrade notes

- WhatsApp adapter now stores session data under `~/.ggcode/credentials/whatsapp/<name>/whatsmeow.db` (using `modernc.org/sqlite`, CGO-free)
- No breaking config changes; all new adapters use the existing `im.adapters` schema
