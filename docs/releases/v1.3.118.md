# v1.3.118

## Fixes

- **A2A continueTask data race**: Fixed a data race in `continueTask()` where `task.Snapshot()` was called without holding `h.mu`, racing with the execute goroutine's `updateStatus()` which writes to task fields under the same lock. The `Handle()` method already had this protection with an explanatory comment, but `continueTask()` was missed. The snapshot is now taken under the lock, matching the pattern in `Handle()`.
