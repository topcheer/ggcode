# ggcode v1.1.62

## Highlights

Final fix from Copilot code review: harness panel data loading is now fully asynchronous, eliminating the last UI main-thread blocking path.

## Fixes

### Harness panel async data loading
- **harness_panel.go**: `refreshHarnessPanel()` and `refreshHarnessPanelForced()` now return `tea.Cmd` that loads all 9 harness sections (Doctor, Monitor, Contexts, Tasks, Inbox, Review, Promote, Release, Rollouts) in a background goroutine. Results arrive via `harnessPanelRefreshResultMsg` without blocking the UI thread.
- All 17 call sites updated to handle the returned cmd (either return it, batch it, or ignore for non-cmd contexts).
- Auto-refresh handler (`harnessPanelAutoRefreshMsg`) now properly returns the async cmd via `tea.Batch`.
- Added `refreshHarnessPanelSync()` for test compatibility.

### Release pipeline fix (from v1.1.61)
- Split `knight/lock.go` into `lock_unix.go` + `lock_windows.go` with proper build tags, fixing Windows cross-compilation that broke Release pipeline since v1.1.57.
