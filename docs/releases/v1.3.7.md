# v1.3.7 — Mobile Tunnel Bridge, Multi-file Tools, Desktop Codesign

## Mobile Tunnel: Three-Endpoint Approval & Ask User Bridge

This release completes the mobile tunnel bridge between TUI, Desktop GUI, and the Flutter mobile app, enabling real-time remote interaction from your phone.

### Outbound (Agent → Mobile)
- **Stream events**: text chunks, tool calls, tool results, thinking, done, error — all pushed to mobile in real time
- **Sub-agent & swarm events**: spawn, text, tool call, tool result, completion — all relayed to mobile
- **Approval requests**: tool permission prompts pushed to mobile with Allow/Deny/Always Allow buttons
- **Ask User requests**: structured questionnaires (single/multi/text) pushed with full field support (prompt, choices, allowFreeform, placeholder)
- **Chat history**: last 100 messages replayed on mobile connect
- **Session info**: workspace, model, provider, mode, version shared on connect

### Inbound (Mobile → Agent)
- **User messages**: typed on mobile → routed through normal agent submit flow (idle → startAgent, busy → queuePending)
- **Approval responses**: Allow/Deny/Always from mobile → precise request ID matching, resolves pending approval
- **Ask User responses**: structured answers from mobile → resolves pending questionnaire
- **Mode changes**: switch permission mode from mobile (supervised/plan/auto/bypass/autopilot)
- **Interrupt**: cancel active agent run from mobile

### Bidirectional Sync
- Whether you approve/answer on desktop or mobile, the result syncs to the other endpoint — no stuck dialogs
- Request ID isolation prevents cross-request confusion
- Mutex-protected pending state prevents race conditions

### Mobile Chat Rendering
- Ask User requests now render in the mobile chat as `ask_user` tool call cards showing question summaries
- After answering, the card updates with the user's response (or "Cancelled")
- Approval prompts show as bottom sheets with full tool name + input preview

### Technical Details
- `tunnel.go`: complete rewrite (~570 lines), all `debug.Log()` instead of `log.Printf` to avoid TUI corruption
- `repl.go`: tunnel hooks in sub-agent OnUpdate/OnComplete and swarm OnUpdate/OnComplete callbacks
- `update_approval.go`: tunnel push for approval and ask_user outbound
- `model_update.go`: inbound handlers for tunnelApprovalResponseMsg and tunnelAskUserResponseMsg
- `agent_bridge.go`: desktop tunnel bridge with mutex-protected pending state and bidirectional sync
- `app.go`: desktop inbound command routing for approval_result and ask_user_response
- `ask_user_screen.dart`: answer updates written back to chat history
- 27+ tests for tunnel logic (nil-safety, pure functions, message history conversion)

## Tools

### Multi-file Read (`multi_file_read`)
- Read multiple files in a single tool call
- Returns numbered `cat -n` format, supports offset/limit per file
- Designed for coordinated multi-file edits

### Multi-file Edit (`multi_file_edit`)
- Batch edits across multiple existing files in one call
- Atomic mode (default): all-or-nothing — if any file fails, no writes happen
- Optional `partial_success` mode for best-effort writes
- Reuses `edit_file`/`multi_edit_file` matching semantics (numbered anchors, wrapper stripping, CRLF tolerance)
- Structured JSON result with per-file status

### Edit File Improvements
- Line-anchor aware matching — `read_file` numbered line prefixes (`   42\t`) work as anchors
- Better diagnostics on match failure (shows expected vs actual whitespace)
