# ggcode v1.1.34

## Highlights

- MCP OAuth 2.1 authentication for HTTP-type MCP servers with automatic metadata discovery, device flow, and token refresh.
- IM per-channel echo suppression: user mirror messages are no longer echoed back on the originating channel.
- Agent and TUI code split into focused files for maintainability.

## Fixes and improvements

### MCP OAuth 2.1

- Adds a full OAuth 2.1 client (`internal/mcp/oauth.go`) that handles protected resource metadata discovery (RFC 9728), authorization server metadata discovery (RFC 8414), dynamic client registration (RFC 7591), device flow authorization, and token refresh.
- HTTP-type MCP servers returning `401 Unauthorized` now trigger the OAuth flow automatically instead of failing silently.
- Built-in client ID updated to GitHub App (`Iv23liJhRuCX7QU1DfgT`) for the GitHub MCP server, enabling `ghu_` tokens with 8-hour lifetime and `ghr_` refresh tokens instead of 1-hour `gho_` tokens.
- Fixes premature token discard: `GetAccessToken` now returns the existing access token optimistically when no refresh token is available, avoiding unnecessary re-authentication (the 5-minute `IsExpired()` buffer previously caused tokens to be discarded too early).
- Adds `oauth_client_id` config field for MCP servers that do not support Dynamic Client Registration.
- Tokens are stored in `~/.ggcode/provider_auth.json` under provider ID `mcp:<server_name>`.

### IM per-channel echo suppression

- Adds `Manager.EmitExcept()` to send events to all bound IM channels except a specified adapter.
- User echo messages (prefixed with `【用户】`) are now suppressed on the originating channel when a remote message arrives from QQ, Telegram, Discord, Slack, DingTalk, or Feishu, preventing duplicate messages.
- Assistant responses still fan out to all bound channels.

### Code refactoring

- Splits `internal/agent/agent.go` into focused files: `agent_autopilot.go`, `agent_compact.go`, `agent_memory.go`, `agent_tool.go`.
- Splits `internal/tui/model.go` into focused files: `model_update.go`, `model_messages.go`, `model_approval.go`, `model_pending.go`, `model_clipboard.go`, `model_terminal.go`.
- Documentation updated: ARCHITECTURE.md, README.md, GGCODE.md now reflect the current code structure and new features.

## Upgrade notes

- If you previously used the built-in GitHub MCP OAuth App client ID, the token stored in `~/.ggcode/provider_auth.json` under `mcp:github` should be deleted to force re-authentication with the new GitHub App client ID:
  ```bash
  jq 'del(.["mcp:github"])' ~/.ggcode/provider_auth.json > /tmp/auth.json && mv /tmp/auth.json ~/.ggcode/provider_auth.json
  ```
- The re-authenticated token will have an 8-hour lifetime and automatic refresh, eliminating the need to re-authorize on every startup.
