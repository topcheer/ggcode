# v1.3.119

## Fixes

- **Cost Tracker data race**: Added `sync.Mutex` to `cost.Tracker` to prevent a data race between `Record()` (called from the agent's streaming goroutine) and `SessionCost()` (called from the TUI/manager goroutine). Both methods accessed `cost` fields without synchronization.

- **A2A continueTask data race** (from v1.3.118): Fixed a data race in `continueTask()` where `task.Snapshot()` was called without holding `h.mu`, racing with the execute goroutine's `updateStatus()`.
