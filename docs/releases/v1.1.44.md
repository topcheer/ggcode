# ggcode v1.1.44

## Highlights

- **Native text selection in the terminal** — the main chat panel no longer captures mouse events, so you can select and copy text naturally with your terminal's built-in selection. Clickable URLs and file paths work out of the box. File browser and preview panels retain mouse-wheel scrolling.
- **Relative paths everywhere** — tool outputs, command previews, and stream text now display paths relative to your working directory instead of verbose absolute paths.
- **Input drain fix** — the startup input drain window was increased from 50ms to 250ms to prevent leftover terminal sequences from being interpreted as real keypresses after a restart.

## Fixes and improvements

### TUI layout

- Main panel mouse mode changed from `CellMotion` to `None` to allow native terminal text selection and clickable URLs.
- Added a 1-column left gutter for visual symmetry with the right margin.
- Fixed `boxInnerWidth` calculation for lipgloss v2 — bordered panels now span the full assigned width instead of being 2 columns short.
- Lowered minimum available height from 8 to 3 rows so the composer is never pushed off-screen on very small terminals.
- Added a `MaxHeight` clamp on the left column to prevent overflow when header + conversation + composer exceed the terminal height.
- File browser and preview panels correctly retain `MouseModeCellMotion` for mouse-wheel scrolling.

### Path display

- Introduced `relativizeResult()` helper that converts absolute paths to relative paths using the current working directory.
- Applied consistently across: stream chunks, shell mode output, command previews, tool start/status args, session history rendering, and tool activity items.
- Removed redundant `truncateString` calls in command rendering — the viewport handles wrapping.

### Startup reliability

- Input drain timeout increased from 50ms to 250ms. Some terminals (especially when re-running the binary immediately after a build with leftover sequences in the input buffer) take longer to settle.
- Updated corresponding test harness wait time.
