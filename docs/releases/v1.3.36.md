# v1.3.36

## Bug Fixes

### Tunnel: UTF-8 corruption in tool args/results (root cause of mobile garbled text)

Byte-level string truncation (`s[:200]`, `s[:500]`) in the tunnel event pipeline could slice through multi-byte UTF-8 sequences (e.g. Chinese characters), producing invalid bytes that rendered as mojibake (U+FFFD replacement characters) on mobile clients. All five truncation sites now use `truncateRunes()` which splits on character boundaries. This is the root cause of garbled Chinese tool_call display names on iPad.

### Tunnel: reuse describeTool DisplayName for ToolDisplayName

`tunnel.go` previously used a separate `toolCallDisplayName()` function for the `ToolDisplayName` field instead of reusing `present.DisplayName` from the already-computed `describeTool()` call. The two functions had different formatting logic, causing inconsistency between TUI and mobile display. Now both paths use the same canonical display name.

### Tunnel: clear all broker state on SwitchSession

`SwitchSession` previously only reset text buffers and reasoning state. Several broker-internal maps and status fields from the previous session leaked into the new one:
- `toolArgs` / `subagentToolArgs` — stale tool arguments
- `currentStatus` / `hasCurrentStatus` — stale "thinking" status
- `currentActivity` / `hasCurrentActivity` — stale activity indicator
- `clientReplayInFlight` — stale replay-in-progress flag

Now all are cleared on session switch, ensuring workspace/session transitions start from a completely clean slate. Only the relay session (encryption token, WebSocket) is preserved.
