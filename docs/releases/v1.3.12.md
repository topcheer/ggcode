# v1.3.12

> **Historical note:** This release note captures the pre-event-log mobile tunnel behavior shipped in `v1.3.12`. Current `main` has since moved mobile reconnects to relay-owned incremental resume with stable `session_id` / `event_id` / `stream_id`, rather than reconnect-time `chat_clear` / `chat_history` replay.

## Mobile/Tunnel Session Lifecycle (historical: pre incremental resume)

- **TUI `/clear`**: Now creates a new session and notifies connected mobile clients with `chat_clear` + updated `session_info` (new session ID). Previously only cleared the local view.
- **GUI new session**: No longer closes the tunnel connection. Sends `chat_clear` + `session_info` to the mobile client so it stays connected and shows the new session.
- **GUI session switch** (`resumeSession`): Sends `chat_clear` → `session_info` → `chat_history` to the mobile client in the correct order.
- **TUI `/sessions` switch**: Sends `chat_clear` + `session_info` + `chat_history` to the mobile client when switching sessions.

## New: `/unshare` Command (TUI)

- Added `/unshare` slash command to stop an active tunnel/sharing session and disconnect the mobile client.
- Fixed ordering bug: `sharing_stopped` message is now sent **before** closing the tunnel connection, ensuring the mobile client receives the disconnect signal. This also fixes the same bug in `/share stop`.

## Bug Fixes

- **Mobile message double echo**: Fixed mobile and webchat messages appearing twice in the TUI when the agent was busy. The `chatWriteUser` call was made both before and inside `queuePendingSubmission`.
- **Missing English translations**: Added `tunnel.stopped`, `tunnel.not_active` to the English i18n catalog.
- **CI flaky test**: Added retry loop for `TestE2EChatWSMixedAttachments` race condition (WS ack read before `SendUserMessage` completes).

## CI/Release Workflow

- **Removed 4 smoke test jobs** (`release-smoke-linux`, `release-smoke-macos`, `release-smoke-windows`, `release-smoke-installer`) for faster release cycles.
- **Optimized dependency graph**: Desktop builds now depend only on `verify` (not CLI `release`), enabling parallel execution. `publish-site-release-branch` depends on all build jobs.
