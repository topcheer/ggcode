# ggcode v1.2.5

## Highlights

- **Ctrl+C deadlock fix**: Pressing ctrl+C with active swarm teammates caused the entire TUI to hang. The root cause was `CancelAll()` calling `program.Send()` through the `onUpdate` callback — Bubble Tea's message channel is unbuffered, so `Send()` blocks until the event loop reads from it, but the loop was stuck inside `Update()`. Cancel now cancels contexts and sets status only, no callbacks.
- **Sub-agent follow panel performance**: Opening the follow panel during sub-agent activity previously caused the spinner and entire UI to slow down. The root cause was per-token `Notify()` flooding the event loop with expensive `rebuildActiveView` calls. Now text is flushed and notified every ~200 tokens, and non-active agent updates skip the rebuild entirely.

## New features

- **Context window probing**: Background context window probing is now opt-in via `probe_context: true` in config. Uses smaller payloads and queries the models API for token limits.
- **Verbose debug level**: New verbose debug logging level to reduce noise in regular debug output.
- **Sidebar version display**: The current application version is now shown in the sidebar panel.
- **Plan mode tool labels**: `enter_plan_mode` and `exit_plan_mode` tools display the `description` parameter as the tool label.
- **Debug log categories restructured**: IM adapters now have per-platform categories (wechat, wecom, signal, mattermost, twitch, nostr, irc). LLM providers have pre-defined stream/chat/probe sub-tags. New categories: webui, harness. Category count: 29 → 38.

## Fixes

- **DeepSeek V4 compatibility**: Skip empty assistant messages in the conversation history to avoid API errors.
- **DeepSeek V4 reasoning_content**: Supply empty `reasoning_content` field for tool_calls requests.
- **Plugin process group cleanup**: Kill entire process group on context cancel (Unix) to prevent `CombinedOutput` from blocking forever when shell commands spawn child processes.
- **Remove max output token limit**: OpenAI and Gemini providers no longer cap output tokens.
- **OpenAI error logging**: Add baseURL to provider error logs for easier debugging.
- **Detailed retry logging**: Add retry/error logging in OpenAI ChatStream for troubleshooting transient failures.

## Upgrade notes

- No breaking changes. All changes are backward-compatible.
- `probe_context` defaults to `false` — existing behavior unchanged unless explicitly enabled.
