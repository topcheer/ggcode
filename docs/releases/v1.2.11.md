# v1.2.11

## Context Window Auto-Inference from Overflow Errors

When an LLM returns a context overflow error (e.g. "prompt too long: model context window exceeded"), ggcode now automatically infers the actual context window size and persists it. This eliminates the need for manual `context_window` configuration or the `ProbeContext` toggle for most models.

**How it works:**
1. Any LLM overflow error is detected reactively (zero extra API calls)
2. The context limit is extracted from the error message (exact value or tier-based estimate)
3. The inferred value is persisted to `~/.ggcode/state/context_windows.json`
4. All metrics (usage %, compact threshold, prompt budget) update immediately

**No configuration needed** — works automatically in TUI, daemon, and pipe modes.
