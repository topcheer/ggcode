# v1.3.8 — Incremental Rendering for TUI & Desktop

## TUI: Block-Level Markdown Render Cache

Streaming markdown rendering is now incremental at the AST block level. Instead of re-rendering the entire text on every chunk, the renderer:

1. Splits the markdown into independent blocks (headings, paragraphs, code blocks, lists) using the goldmark AST parser
2. Compares blocks against the previous render cache
3. Reuses cached output for unchanged prefix blocks
4. Only re-renders the tail block(s) that have changed

This reduces CPU usage from O(n²) to effectively O(1) per chunk during streaming, where n is the accumulated text length. The improvement is most noticeable during long assistant outputs.

**Technical details** (`internal/chat/stream_markdown.go`):
- `splitMarkdownBlocks()` — goldmark AST parser splits source into independent blocks
- `closeOpenFences()` — closes unclosed code fences during streaming (e.g., `` ```go `` without closing `` ``` ``)
- `renderStreamingMarkdown()` — prefix-matches against cache, reuses stable blocks
- Cache is cleared on `SetFinished()` / `SetReasoningFinished()` — final render always goes through full `markdown.Render`

Both text and reasoning streams use separate caches (`textCache`, `reasoningCache`) on `AssistantItem`.

Bug fix: `SetReasoning()` now resets `reasoningOk = false`, fixing a bug where reasoning updates after `SetReasoningFinished()` would not render.

## Desktop: Incremental Chunk Deltas

The desktop GUI previously sent the **full accumulated text** on every streaming update, causing Fyne's markdown widget to re-render the entire content. Now it sends **incremental chunk deltas**:

- `assistantInc` / `reasoningInc` — `strings.Builder` buffers that accumulate text since the last notification
- `takeAssistantIncrement()` / `takeReasoningIncrement()` — atomically extract and reset the delta under `ChatMu.Lock()`
- `chat_view.go` uses `AppendChunk(delta)` instead of `SetMarkdown(fullText)` on the markdown widget
- Throttle interval reduced from 300ms → 80ms (incremental chunks are much lighter)

**Agent panel state machine** (`appendAgentEvents`):
- `textMD` / `reasoningMD` tracked per agent panel — consecutive text/reasoning chunks merged into a single widget via `AppendChunk`
- Widget references cleared on event type transitions (text → tool_call → reasoning → error)
- Replaces the previous lookahead loop with a cleaner state machine pattern
- `reasoningMD` direct reference replaces fragile `cv.reasoningW.Items[0].Detail` type assertion chain

Status loop refresh rate improved from 500ms → 120ms to match the lighter rendering cost.
