# ggcode v1.1.52

## Highlights

- **WebUI markdown rendering fixed** тАФ code blocks now fill the display area; headers, lists, links, paragraphs, and horizontal rules all render correctly in chat messages, session history, and system prompt preview.
- **TUI-mode webui parity with daemon** тАФ MCP server status, IM adapter status, and A2A discovered instances now appear in the webui config page during TUI sessions, matching daemon-mode behavior.
- **LSP E2E install testing** тАФ bash-language-server and yaml-language-server auto-install end-to-end tests validate the full detectтЖТinstallтЖТuse lifecycle.
- **7,400+ lines of new tests** тАФ 39 test files added across 15 packages, with real LSP server integration tests for Go, TypeScript, Lua, Bash, and YAML.

## Bug Fixes

- **WebUI markdown not filling display area** тАФ removed `white-space: pre-wrap` from `.chat-text` that conflicted with `<p>/<br>` rendering; added `width:100%` to code blocks; deduplicated dead `renderMarkdown` function; applied `renderMarkdown()` to session detail and user messages.
- **WebUI session detail crash** тАФ `esc()` called `.replace()` on non-string JSON objects (e.g. `json.RawMessage` fields). Fixed with `String()` coercion and null-safe checks.
- **TUI-mode webui missing status data** тАФ `SetMCPStatusFn`, `SetIMStatusFn`, and `SetA2ADiscoverFn` were not wired in TUI mode, leaving the config page empty.
