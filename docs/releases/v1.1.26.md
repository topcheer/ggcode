# ggcode v1.1.26

## Highlights

- Promotes **LSP to a first-class product entry point** alongside MCP and skills instead of treating it
  like a narrow fallback feature.
- Ships the validated multi-language local-LSP flow across **Go, Python, Rust, and Lua**, including
  in-app installation from `/status`.

## Fixes and improvements

### First-class LSP workflow

- README and product positioning now present **LSP / MCP / skills** at the same level in the core
  workflow story.
- `/status` continues to be the control surface for local LSP readiness and now explicitly supports
  installing missing servers in place.
- Python workspaces with multiple recommended servers now open an install chooser before running the
  selected install flow.

### Installation and detection hardening

- Fixed Rust detection after `rustup component add rust-analyzer` by recognizing rustup-managed
  installs even when `~/.cargo/bin` is not on `PATH`.
- Added Go fallback detection for `gopls` installed into `GOBIN`, `GOPATH/bin`, or `~/go/bin`.
- Added Python workspace-venv detection so `.venv` / `venv` LSP binaries are recognized even without
  manual shell activation.
- Changed Python LSP installation to prefer an existing workspace venv, or create `.venv` on demand,
  before installing `pyright` or `python-lsp-server`.

### Reliability fixes

- Fixed the generated Python venv install command syntax used by `/status` so shell installation no
  longer fails with `syntax error near unexpected token 'then'`.
- Fixed `pyright-langserver` startup when discovered from a workspace venv so it still launches with
  the required `--stdio` argument.
- Added regression coverage for rustup-managed Rust installs, workspace-venv Python installs, and the
  updated discovery behavior.

### Validation fixtures

- Added and validated external fixture projects under `~/ggai` for:
  - Go
  - Python
  - Rust
  - Lua
- Verified semantic navigation flows across those projects before clearing the local servers for fresh
  install-from-scratch testing.

## Upgrade notes

- No migration is required.
- `/status` is now the expected first stop for local semantic tooling, not just a passive status view.
- For Python, ggcode now installs LSP servers into the workspace venv instead of assuming a global
  `pip install` path.
