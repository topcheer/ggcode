# ggcode v1.3.0

## Highlights

- **Desktop GUI application** — Native desktop application built with Fyne, providing a visual chat interface with Markdown rendering, IM platform integration, tool approval dialogs, permission mode switching, and session history sidebar.
  - macOS: Universal binary (amd64 + arm64) packaged as `.dmg`
  - Windows: Standalone `.exe`
  - Visual chat with streaming Markdown, image attachments, and code block rendering
  - IM Settings window: add, configure, enable/disable adapters (WeChat Work, Feishu, DingTalk, etc.) with pairing code display
  - Tool approval: popup dialog with Deny / Allow / Always Allow options and readable parameter display
  - Ask User: popup form supporting text, single-select, multi-select questions with freeform notes
  - Permission mode selector in toolbar (Supervised / Auto / Bypass / Autopilot)
  - Session history sidebar with resume support
  - Input history navigation with Up/Down arrow keys
  - Inbound/outbound IM message flow with round summaries and tool call events

- **WeChat Work adapter tool call push** — WeChat Work adapter now formats and sends `OutboundEventToolCall` and `OutboundEventToolResult` events, matching Feishu and other adapters.

- **MCP device code clipboard** — MCP server OAuth device flow automatically copies the user code to clipboard.

## Fixes and improvements

- Desktop: approval dialog uses `NewCustomWithoutButtons` to prevent crash on popup
- Desktop: `ask_user` returns proper choice IDs instead of labels for single/multi select
- Desktop: disabled adapters bound to current workspace shown in "This workspace" section
- Desktop: Enable button starts adapter immediately via `StartNamedAdapter`
- Config: tests updated for `ai-gateway` vendor restructure (gateway vendors consolidated under single vendor)
- Release workflow: `build-desktop-darwin` and `build-desktop-windows` jobs for automated desktop builds
- Release workflow: desktop artifacts (`.dmg`, `.exe`) uploaded to GitHub Release and site branch

## Upgrade notes

- This release introduces desktop application builds alongside the existing CLI. No migration needed for CLI users.
- The `ai-gateway` vendor consolidates several previously top-level vendors (aihubmix, getgoapi, novita, nvidia, openrouter, poe, requesty, together, perplexity, vercel) as endpoints. If you reference these directly in config, update to use `vendor: ai-gateway` with the appropriate endpoint name.
