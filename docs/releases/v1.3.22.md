# ggcode v1.3.22

## Highlights

- **Unified start_command display**: All surfaces (TUI, Desktop, IM, Mobile) now show a concise "Started" or "Failed" label for `start_command` results instead of raw job metadata or complete suppression. The new `tool.StartCommandResultText()` helper is shared across Go and Dart.
- **Tunnel event threading fix**: Sub-agent and swarm tunnel events are now dispatched through Bubble Tea messages instead of direct model mutation from background goroutines, eliminating data races in the TUI event loop.
- **Theme-aware mobile tab colors**: Chat screen tab bar uses `AppColors` (textPrimary/textMuted/accent) instead of hardcoded white/blue, fixing readability in light theme.
- **Tunnel tool call detail fallback**: When the manager callback omits `Detail`, the TUI now falls back to `describeTool()` to generate a useful display name for tunnel tool call events.

## Fixes and improvements

- **`internal/tool/labels.go`**: Added `StartCommandResultText()` — normalizes start_command output to "Started"/"Failed" based on Status line and error flag.
- **`internal/tui/chat_bridge.go`**: `suppressToolResult` now returns the formatted status instead of empty string for `start_command`.
- **`internal/chat/tools.go`**: `start_command` no longer has `suppressBody=true`; its body shows the concise status label.
- **`desktop/ggcode-desktop/chat_view.go`**: `addToolResult`/`formatToolResult` accept `isError`; header-only tools can use markdown renderer.
- **`internal/im/tool_format_helpers.go`**: Simplified `formatIMStartCommandResult` to delegate to `tool.StartCommandResultText`.
- **`mobile/flutter/lib/core/providers/session_provider.dart`**: Added `_formatStartCommandResult` with matching "Started"/"Failed" logic.
- **`internal/tui/repl.go`**: Sub-agent callbacks (`SetOnStreamText`, `SetOnToolCall`, `SetOnToolResult`) send typed Bubble Tea messages instead of directly calling model methods.
- **`internal/tui/update_subagent.go`**: New handlers `handleSubAgentTunnelStreamTextMsg`, `handleSubAgentTunnelToolCallMsg`, `handleSubAgentTunnelToolResultMsg`, `handleSwarmTunnelEventMsg` process tunnel events on the UI thread.
- **`internal/tui/model_messages.go`**: Added message types for tunnel event threading.
- **`mobile/flutter/lib/features/chat/chat_screen.dart`**: Tab bar colors use `AppColors` for light theme compatibility.
