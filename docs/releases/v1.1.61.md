# ggcode v1.1.61

## Highlights

Fixes the Release pipeline failure that affected v1.1.57–v1.1.60. The root cause was `syscall.Flock` in `internal/knight/lock.go`, which is Unix-only and caused Windows cross-compilation to fail.

## Fixes

### Release pipeline (Windows cross-compile)
- **knight/lock.go → lock_unix.go + lock_windows.go**: Split the lock implementation using Go build tags.
  - Unix (`lock_unix.go`): Uses `syscall.Flock` (LOCK_EX/LOCK_UN/LOCK_NB) — same as before.
  - Windows (`lock_windows.go`): Uses `golang.org/x/sys/windows.LockFileEx` with `LOCKFILE_EXCLUSIVE_LOCK|LOCKFILE_FAIL_IMMEDIATELY`.
- All 6 cross-compile targets (linux/darwin/windows × amd64/arm64) now build successfully.
