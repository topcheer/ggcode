# ggcode v1.1.70

## Highlights

Completes the strict write guard (all 8 mutating tools), updates design doc to reflect true state (~90%), and adds 9 integration tests covering suggest UX, pipe auto-run with stdin, strict guard scope, and dirty workspace policy.

## Changes

### Strict write guard: full tool coverage
- Added `git_add`, `git_commit`, `git_stash`, `notebook_edit` to the deny list.
- Strict mode now denies all 8 mutating tools: `write_file`, `edit_file`, `multi_edit_file`, `notebook_edit`, `run_command`, `git_add`, `git_commit`, `git_stash`.
- Guard applies globally (before routing), even when input is not routed to harness.
- Test verifies all 8 denied + read tools unaffected.

### Design doc updated
- "What is missing" section rewritten to reflect current gaps (5 items, down from 7).
- Readiness updated from 55% → 90%.
- Remaining gaps: daemon/IM integration, CTA UX, LLM classifier, worker guard exemption, integration tests.

### Integration tests (+9)
**TUI (auto_run_integration_test.go):**
- `TestSuggestMode_EnterConfirmsHarness` — Enter with pendingAutoRun triggers harness run.
- `TestSuggestMode_EscDismisses` — Esc with pendingAutoRun dismisses and submits normally.
- `TestSuggestMode_NewInputClearsStaleState` — Submitting new text clears stale pendingAutoRun.
- `TestStrictWriteGuard_GlobalOnMode` — Verifies all 8 tools denied, read tools unaffected.
- `TestStrictWriteGuard_GlobalEvenForQuestions` — Guard applies even for non-routed inputs.

**Harness (auto_run_integration_test.go):**
- `TestPipeAutoRun_UsesFullPromptForRouting` — fullPrompt with stdin code routes to harness.
- `TestPipeAutoRun_PureQuestionNotRouted` — Questions with stdin not routed.
- `TestDirtyWorkspace_FailFastWhenNoConfirmer` — No panic without confirmer.
- `TestStrictMode_AllToolsDeniedForAllRouteOutcomes` — StrictWriteGuard set even for questions.
