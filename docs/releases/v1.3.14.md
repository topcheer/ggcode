# v1.3.14

## Multi-Workspace Cache & Session History (Mobile)

- **Workspace cache system**: New `WorkspaceCacheNotifier` persists workspaces, sessions, and message snapshots to SharedPreferences. Up to 10 workspaces with 20 sessions each and 300 messages per session.
- **Workspace switcher**: Tap the workspace name in the AppBar to open a switcher. Lists all known workspaces with their sessions. Selecting a historical session shows cached messages with a banner indicating it's a cached view (input disabled).
- **In-chat QR scanner**: New QR scanner button in AppBar allows scanning a new workspace without leaving the chat screen.
- **Bootstrap reconnect**: On app launch, automatically reconnects to the last active workspace/session.
- **Live projection**: Chat messages, subagent state, session info, and agent status are continuously persisted. On disconnect/reconnect, the cached state is displayed while reconnecting.

## Cron Message Display

- Cron-triggered prompts now appear as system messages (`"Cron job triggered"`) in the mobile chat instead of user messages. The actual cron prompt is sent to the agent but displayed as a system indicator.
- New `MessageData` type with `text`, `display_text`, and `kind` fields. `kind: "cron"` triggers the system message display. `bindRemoteUserMessage` absorbs locally-typed messages that match the remote echo.

## Tunnel Status Persistence

- `Broker.PushStatus` now persists the current status in-memory. On relay reconnect, the status is re-seeded along with session info and history.
- `BrokerSnapshot` includes `Status` field. `SendSnapshot` is a convenience method that sends session_info + history + status in one call.
- New `currentTunnelStatus()` in TUI derives status from pending approvals, questionnaires, tool activity, and loading state.
- Desktop `AgentBridge.CurrentTunnelStatus()` mirrors the same logic.

## Input Bar Improvements (Mobile)

- Input bar is now always enabled — you can send a message while the agent is working (queued for next turn).
- Animated pulsing border glow when agent is active (thinking/running).
- Contextual hint text: "Viewing cached session...", "Disconnected", "Agent is working... you can still send a message", "Type a message..."

## Flutter 3.27+ Migration

- All `Color.withOpacity()` calls replaced with `Color.withValues(alpha:)` (recommended since Flutter 3.27, avoids `Color` round-trip overhead).
- `print()` replaced with `debugPrint()` in connection service.

## System Prompt

- Added 4 coding principle guidelines: reuse existing patterns, fix root causes, preserve existing behavior, validate incrementally.

## Deploy Automation (Fastfile)

- Complete Fastfile rewrite: `deploy_external` now waits for TestFlight processing (polls up to 20 min), promotes to External Testing, and auto-submits for App Store Review if no active submission exists.
- Shared `_submit_build_for_review` method used by both `deploy_external` and `submit` lanes.
- `_wait_for_valid_build` private method for reusable build processing wait.
- `_promote_to_external` private method for External Testing distribution.
- `remove_simulator_frameworks` helper strips IOSSIMULATOR-only frameworks (objective_c) with re-sign using Apple Distribution certificate.
- Fixed all `__FILE__` paths to `__dir__` for reliable path resolution.
- `store_deploy.sh`: `tail -10` → `tail -30` for better error visibility.
