# ggcode v1.3.23

## Highlights

- **Cancel safety overhaul**: Sub-agents and swarm teammates no longer start queued work after ctrl+c/interrupt. Pending agents are now properly cancelled, and terminal-state agents cannot be "resurrected" by late SetCancel calls.
- **Pending queue struct upgrade**: `pendingQueue` upgraded from `[]string` to `[]pendingSubmission` struct supporting hidden submissions (cron) with tunnel metadata overrides.
- **Shell mode tunnel support**: Shell commands and output stream to mobile via tunnel with kind metadata (shell_command/shell_output), rendered with terminal-style bubbles including ANSI color support.
- **Live tunnel status**: Mobile now receives real-time agent status (thinking/running/idle) instead of hardcoded states. Cancelled tools emit tunnel tool_result events.
- **CI token fix**: Release workflow now uses PAT for all write operations (GoReleaser, checkout with full history, release notes, asset uploads).

## Cancel safety fixes

- `subagent.Cancel()` now cancels pending agents (not just running)
- `subagent.CancelAll()` covers pending + running agents
- `subagent.SetCancel()` returns false for terminal agents, preventing resurrection of cancelled agents — also cancels the provided context immediately
- Runner checks SetCancel return and exits early if refused
- Swarm idle_runner checks `ctx.Err()` before every branch to avoid starting work after cancellation
- `tryClaimPendingTask` rolls back task to StatusPending on context cancellation
- `swarm.CancelAll()` cancels all live teammates including idle ones

## Tunnel and mobile improvements

- `pendingSubmission` struct with Text, Hidden, TunnelOverride fields
- `enqueueHidden()` for cron-triggered submissions that stay invisible
- `chatCancelAllRunningTools()` marks running tools as Canceled on interrupt
- Cancelled tools emit tunnel tool_result events for mobile
- `pushTunnelCurrentStatus()` reads live activity state for real-time mobile updates
- Shell commands stream via `PushTextData` with Kind=shell_output
- Mobile shell bubble: dark terminal style, `$` badge, ANSI 16/256-color rendering
- History replay preserves message kind through event reconstruction

## CI fixes

- Release workflow uses `TAP_GITHUB_TOKEN` (PAT) for GoReleaser, checkout, and asset uploads
- Checkout with `fetch-depth: 0` uses PAT to avoid "could not read Username" errors
- All write operations decoupled from default GITHUB_TOKEN (read-only in this repo)
