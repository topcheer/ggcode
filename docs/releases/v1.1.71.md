# ggcode v1.1.71

## Highlights

Completes the harness auto-run feature: daemon auto-run execution, guard exemption infrastructure, IM/WebUI event mirroring, and interactive one-key review CTA. The loop is now closed: prompt → auto-run → review (Enter) → promote.

## Changes

### Daemon auto-run via RunService
- `DaemonBridge.tryHarnessAutoRun()` checks routing decision before the agent loop.
- `RouteHarness` → runs via `RunService`, emits result to IM/WebUI subscribers.
- `RouteSuggest` → logs and falls through to agent (skill-guided).
- `RouteNone` → normal agent run, no intercept.

### Guard exemption infrastructure
- `ClearOverride(toolName)` added to `permission.ConfigPolicy` — removes a previously set override.
- Documented in `applyStrictWriteGuard()` comment: BinaryRunner subprocess not affected. Future subagent mode workers should call `ClearOverride()` or use a separate policy.
- Test: `TestConfigPolicy_ClearOverride` verifies deny → clear → no longer denied.

### IM/WebUI event mirroring
- TUI `harnessRunResultMsg` handler now broadcasts the result (including CTA) via `webuiBridge.BroadcastEvent()`.
- WebUI subscribers see the full run summary + next action in real time.
- Daemon path already emits via `b.emitter.EmitText()` → IM channels.

### Interactive review CTA (one-key approve/reject)
- After harness run completes with review-pending task, `pendingHarnessReview` is set on the model.
- User sees: "Press Enter to approve, Esc to skip."
- **Enter** → `handleHarnessReviewApprove()` calls `ApproveTaskReview()`, shows result.
- **Esc** → skips review, shows how to approve later with slash command.
- `harnessReviewResultMsg` handler renders approve result + promote CTA.
- Full loop: prompt → auto-run → Enter to review → Enter to promote.

### Strict guard documentation
- Comment updated on `applyStrictWriteGuard()` with exemption guidance for future subagent mode.
