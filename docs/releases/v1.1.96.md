# ggcode v1.1.96

## Highlights

- **9 reliability fixes** addressing atomicity, concurrency, and state consistency issues across Harness, IM, Knight, and TUI subsystems.

## Fixes

### High priority

1. **ApplyReleasePlan() ‚Äî snapshot-based rollback**. Previously, rollback was writing mutated values back (ineffective). Now captures pre-mutation snapshots via value copy (`snap := *loaded`), so rollback restores original state. Uses `atomicWriteFile` (write-to-temp + rename) to prevent partial JSON writes on crash.
2. **AdvanceReleaseWaveRollout() ‚Äî safe wave order + atomic writes**. Persists next wave (activate) BEFORE marking current wave (completed). Worst-case half-state now has a new active wave rather than no active wave. In-memory revert from snapshots on first-write failure.
3. **SaveTask() ‚Äî mutex protection**. Added global mutex to prevent concurrent read-modify-write from silently overwriting task state, reviews, or worker fields.
4. **IM binding ‚Äî persist before mutate**. Changed to probe-then-apply pattern: construct a copy with new values, persist the copy, then update the in-memory binding only on success. Prevents memory/disk state divergence on persist failure.
5. **Knight QueueTouchCount ‚Äî merge counting fix**. Fixed unconditional `++` that inflated queue touch count during merge. Now correctly computes `existing+1` or `max(existing, incoming)+1`.

### Medium priority

6. **Review reject ‚Äî new TaskRejected status**. "Execution succeeded but review rejected" is now `TaskRejected`, distinct from `TaskFailed` (execution error). Enables correct retry/stats/automation semantics.
7. **TUI sessionMutex ‚Äî eager initialization**. Initialized in `NewModel()` to prevent lazy-init race across Bubble Tea model copies.
8. **TUI approval channels ‚Äî non-blocking send**. All bare channel sends wrapped in `select { case ch <- v: default: }` to prevent goroutine leaks when receiver has already gone.
9. **IM dedup table ‚Äî amortized pruning**. Prune every 100 messages instead of every message, eliminating O(n) hot path in high-traffic group chats.
