# ggcode v1.1.59

## Highlights

Second round of fixes from Copilot code review, addressing a newly-introduced data race, completing partial fixes, and closing an entry-path inconsistency.

## Fixes

### Data race fix
- **submit.go**: Previous fix moved `AppendMessage()` outside `sessionMutex`, but `AppendMessage` mutates `ses.Messages`/`ses.UpdatedAt`, racing with the checkpoint handler. **Fixed**: Session object mutations stay inside `sessionMutex`; new `AppendMessageToDisk()` method (JSONL write + index update only) is called outside the lock.

### Completed partial fixes
- **view.go**: `sidebarGitBranch()` now fully caches the branch string, not just the `.git` directory. `View()` reads from a package-level cache; `gitBranchTickMsg` timer refreshes it every 2 seconds — zero disk I/O per frame.
- **file_browser.go**: Entry count limit is now a global counter passed through recursion (`*remain` pointer), not per-level. Deep directory trees are properly bounded to 2000 total entries.
- **harness_panel.go**: Removed unused async scaffolding. Synchronous `refreshHarnessPanel()` now loads all section data with error silencing (no early returns on individual section failures), and `refreshHarnessPanelForced()` skips debounce for auto-refresh checks.

### Entry path consistency
- **pipe.go**: `pipePermissionMode()` now reads `cfg.DefaultMode` instead of hardcoding `AutoMode`. Matches `root.go` behavior: if the config specifies `bypass` or `autopilot`, pipe mode respects it.
