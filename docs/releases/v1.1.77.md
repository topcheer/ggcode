# ggcode v1.1.77

## Highlights

LLM classifier now distinguishes task complexity: simple changes go to main agent, complex changes go to harness.

## Changes

### Complexity-aware LLM routing
- LLM classifier prompt now outputs `complexity`: "simple" or "complex".
- `simple` = typo fix, rename, single value update → main agent handles directly.
- `complex` = multi-file change, bug investigation, refactoring → harness isolation.
- Verified with glm-5.1: "把 README 版本号改成 2.0" → simple (1.00), "重构认证模块 JWT→session" → complex (0.98).

### Expanded LLM classifier coverage
- Previously: only score 1-2 went through LLM. Score 3+ routed directly to harness.
- Now: score 1-4 go through LLM. Score 5+ routes directly (extremely strong signal).
- This catches "fix the typo in README.md" (verb+path=score 4, but actually simple).
- Fallback: without LLM provider, score ≥ 3 still routes to harness (backward compatible).

### Timeout adjusted
- 8 seconds (CN proxy endpoints can be slow at 3-5s latency).

## Routing table

| Score | With LLM provider | Without LLM provider |
|-------|-------------------|---------------------|
| 0 | RouteNormal | RouteNormal |
| 1-4 | LLM classifies → complex=RouteHarness, simple=RouteNormal | score ≥ 3 → RouteHarness |
| 5+ | RouteHarness (skip LLM) | RouteHarness |
