# ggcode v1.1.77

## Highlights

LLM classifier now distinguishes task complexity: simple changes go to main agent, complex changes go to harness.

## Changes

### Complexity-aware LLM routing
- LLM classifier prompt now outputs `complexity`: "simple" or "complex".
- `simple` = typo fix, rename, single value update → main agent handles directly.
- `complex` = multi-file change, bug investigation, refactoring → harness isolation.
- Verified with glm-5.1: "把 README 版本号改成 2.0" → simple (1.00), "重构认证模块 JWT→session" → complex (0.98).

### Expanded LLM classifier coverage
- Previously: only score 1-2 went through LLM. Score 3+ routed directly to harness.
- Now: score 1+ goes through LLM when a classifier provider is available. High score means "likely code-related", not automatically "complex".
- This catches "fix the typo in README.md" (verb+path=score 4, but actually simple) and avoids sending simple changes to harness.
- Fallback: when the LLM provider is unavailable, errors, or times out, score ≥ 3 still routes deterministically (backward compatible).
- TUI classification runs off the Bubble Tea update path so a slow classifier cannot freeze the interactive UI.

### Timeout adjusted
- 8 seconds (CN proxy endpoints can be slow at 3-5s latency).

## Routing table

| Score | With LLM provider | Without LLM provider |
|-------|-------------------|---------------------|
| 0 | RouteNormal | RouteNormal |
| 1-2 | LLM classifies → complex/high-confidence may route or suggest, simple=RouteNormal | RouteNormal |
| 3+ | LLM classifies → complex=RouteHarness/RouteSuggest, simple=RouteNormal, unknown=RouteSuggest | score ≥ 3 → RouteHarness/RouteSuggest by mode |
