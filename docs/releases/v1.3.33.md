## v1.3.33

### Relay Server Offline Recovery

When the desktop (server) disconnects from the relay, connected mobile clients now receive a structured `server_offline` notice with a `retry_after_ms` hint instead of silently dropping. The relay waits 90s before expiring the room (up from 30s) so brief restarts recover transparently.

- **Relay**: broadcast `server_offline` to all room clients on server disconnect, include configurable retry hint
- **Client**: clients connecting to a room with no server receive `server_offline` immediately and close after 500ms
- **Relay client (Go)**: graduated reconnect delays (5s → 10s → 20s → 40s), transparent passthrough of `server_offline` / `sharing_stopped`
- **Flutter connection service**: permanent room failure detection (stale token, room not found, 410), retry_after_ms parsing, `_everConnected` state tracking

### Mobile Permanent Room Failure Handling

Mobile now detects permanent failures (stale/expired share tokens, room destroyed) and cleanly disconnects instead of infinitely retrying. The workspace selection and resume cursor are cleared so the user is returned to the scan screen.

- `isPermanentRoomFailureMessage()` detects stale/expired/410 patterns
- `_handlePermanentRoomFailure()` clears all state and disconnects
- `sharing_stopped` event treated as permanent failure
- `_clearPersistedResumeState()` helper deduplicates cleanup code

### Mobile Workspace Cache Simplification

Simplified `attachSessionToActiveWorkspace()` to no longer copy snapshots between rooms — new rooms start fresh. This prevents stale data from leaking across room switches.
