## v1.3.30

### Tunnel Generation-Based Lifecycle

Prevents stale async callbacks from a previous `/share` session from polluting a new one. Each `/share` increments a generation counter; all async events (inbound messages, mode changes, approvals) carry it and handlers discard stale ones.

- Double `/share` guard prevents duplicate tunnel starts.
- `/unshare` now works during the starting phase.
- Graceful close runs asynchronously to avoid blocking the TUI event loop.
- Cron prompts no longer pollute input history.

### Relay Cross-Room Session History

Relay now stores events in a global session table independent of room tokens. When a user switches workspaces (new token/room) but keeps the same session ID, history is automatically hydrated from the global store.

- New `relay_global_events` / `relay_global_sessions` SQLite tables.
- `persistEvent` dual-writes to both room-local and global tables.
- Room destroy preserves global session history for future cross-room reuse.

### Broker Event Cursor Advancement

On relay reconnect with retained history, the broker now advances its event ID cursor past the relay's last known event, preventing ID collisions for new live events.
