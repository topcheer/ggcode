# ggcode v1.1.24

## Highlights

- Publishes the complete provider-panel and vendor-catalog update with the corresponding README
  documentation included in the tagged release.
- Adds more built-in OpenAI-compatible vendors and keeps the `/provider` panel compact, predictable,
  and immediately usable after endpoint edits.

## Fixes and improvements

### Built-in vendor catalog

- Added built-in presets for:
  - `aihubmix` → `https://aihubmix.com/v1`
  - `getgoapi` → `https://api.getgoapi.com/v1`
  - `novita` → `https://api.novita.ai/openai/v1`
  - `poe` → `https://api.poe.com/v1`
  - `requesty` → `https://router.requesty.ai/v1`
  - `vercel` → `https://ai-gateway.vercel.sh/v1`
- Added default API key env-var mappings for those vendors and updated the bundled example config.
- Added the same provider/env-var information to the README so the public docs match the shipped
  presets.

### Provider panel UX

- Tightened the vendor/endpoint layout so the panel uses less vertical space.
- Added an env-var hint row showing which variable the currently selected vendor/endpoint expects.
- Increased the left vendor section height, moved the model section upward, and limited the model
  viewport to five visible entries so the panel no longer jumps around.

### Provider model refresh flow

- Trigger model discovery immediately after saving an endpoint API key or endpoint base URL.
- Newly configured endpoints now become selectable in-place without switching away from the vendor.

## Upgrade notes

- No migration is required.
- Existing configs continue to work.
- This release supersedes `v1.1.23` by including the missing README provider documentation in the
  tagged artifact set.
