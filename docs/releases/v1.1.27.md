# ggcode v1.1.27

## Highlights

- Fixes the **TUI composer height jitter** that made the input area grow after wrapping and then shrink
  again while you kept typing.

## Fixes and improvements

### Stable multi-line composer rendering

- Kept the existing `textinput` interaction model for short prompts so normal single-line editing behavior
  stays unchanged.
- Switched the composer to a stable wrapped renderer once input content exceeds a single line, so the
  visible input area now grows with the content instead of oscillating as the hidden horizontal scroll
  window moves.
- Preserved prompt-aligned continuation lines so wrapped input still reads like a single coherent draft
  instead of a clipped single-line field.

### Regression coverage

- Added a layout regression test that asserts composer height does not shrink as progressively longer
  input wraps across multiple lines.

## Upgrade notes

- No migration is required.
- This release is recommended if you use the interactive TUI heavily, especially for longer prompts.
