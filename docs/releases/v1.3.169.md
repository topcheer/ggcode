# ggcode v1.3.169

## Highlights

- **Session index debounce**: Eliminates per-message `fsync` by debouncing index updates to 5-second intervals — major I/O reduction for active sessions.
- **System prompt caching**: Skips redundant system prompt re-injection on every agent loop iteration when content hasn't changed.
- **WhatsApp i18n**: The last unlocalized IM adapter panel now has full Chinese/English translation support.

## New Features

### WhatsApp Panel Localization (`480d4475`)

Added `i18n_whatsapp.go` with complete en/zh-CN string catalog for the WhatsApp adapter panel. All 16 IM adapter panels are now fully localized.

### Mobile Reconnect Countdown (`a35631a6`)

The mobile connect screen now displays a reconnect countdown timer when the relay connection drops, giving users clear feedback during reconnection attempts.

### Background Compression Indicator (`ba9f06af`)

The TUI status bar now shows a live compression indicator (`🗜 ctx 45K→…`) when background context compression is running, so users know when the context window is being managed.

## Performance

### Session Index Update Debounce (`702fc478`)

`AppendMessageToDisk` previously triggered a full index rebuild on every message write, including `fsync`. This caused significant I/O on active sessions with many messages.

The index update is now debounced to 5-second intervals. The final update is flushed on session close to ensure no data loss.

### System Prompt Caching (`f6d446e3`)

The agent loop was re-injecting the full system prompt on every iteration even when the prompt content hadn't changed. A `lastInjectedSystemPrompt` cache now skips re-injection when content is identical, reducing token overhead.

## Fixes

### IM Tool Result Suppression (`5207ab6e`)

Tool results for `multi_file_edit`, `multi_file_write`, `notebook_edit`, and `git_show` were incorrectly suppressed in IM output. These now display their results properly.

### Streaming Code Block Fence Markers (`d4f1fdb6`)

Code block fence markers (` ``` `) were leaking into the streaming render output, causing visual glitches in the TUI.

### LSP JSON-RPC Write Race (`32cb3dd6`)

Fixed a data race in LSP JSON-RPC write path where `readLoop` death and `ApplyDisabledState` could race with concurrent writes.

### Daemon Detach Session Loss (`05f5a401`)

Daemon detach was losing the session reference, deleting the PID file prematurely, and tunnel toggle was not preserving `agentBusy` state.

### Swarm Teammate nil Agent Block (`10a4beb6`)

Nil agent in swarm teammate blocked `ReplyTo`, `tryClaimPendingTask` swallowed errors silently, and `EndedAt` was misused.

### Flaky Test TempDir Race (`91dc2144`)

Eliminated TempDir cleanup race in debounce tests by switching from `t.TempDir()` to `os.MkdirTemp` to avoid background goroutine access after test cleanup.

## Upgrade Notes

No breaking changes. All improvements are backward compatible.
