# ggcode v1.1.81

## Highlights

Unified panic recovery across all background goroutines. A panic in any subsystem (IM adapter, MCP client, A2A handler, WebSocket pump, etc.) now logs the full stack trace and recovers gracefully instead of crashing the entire process.

## Changes

### New package: `internal/safego`
- `safego.Go(name, fn)` — launches `fn` in a new goroutine with panic recovery.
- `safego.Run(name, fn)` — runs `fn` synchronously with recovery (for goroutine bodies you launch yourself).
- `safego.Recover(name)` — `defer`-friendly helper for manual goroutine bodies.
- `safego.PanicHook` — optional callback (name, recovered value, stack trace) for surfacing errors to TUI or shipping to monitoring.
- Recovered panics are logged via `internal/debug` with full stack traces.
- Circular import avoided: `safego` uses an injected `logFn` set by `debug.init()`.
- 6 unit tests covering recovery, hook invocation, and hook panic swallowing.

### Bug fix: circular import
- Copilot's original change introduced `debug → safego → debug` import cycle.
- Fixed by making `safego` use an injected `SetLogger()` function instead of directly importing `debug`.
