# ggcode v1.1.76

## Highlights

E2E auto-run tests (9) + real LLM classifier smoke tests (3).

## Changes

### E2E auto-run tests (+9)
All use real git repos, real harness init, real task lifecycle. Only `fakeRunner` replaces BinaryRunner subprocess (same pattern as existing e2e tests).

| Test | What it covers |
|------|----------------|
| `FullLifecycle` | Input → RouteHarness → RunService → task completed → review approved → promoted |
| `SuggestMode` | Returns RouteSuggest, does not auto-execute |
| `StrictMode_EnforcesWorktree` | StrictWriteGuard=true, worktree_mode=required |
| `QuestionNotRouted` | Questions stay RouteNormal |
| `DirtyWorkspace_RejectedByConfirmer` | Confirmer rejects dirty workspace → RunService fails |
| `OffModeSkipsEverything` | Returns RouteNone |
| `MultipleTasks_IndependentPromotion` | Promote task1 does not affect task2 |
| `LLMClassifierOverrides` | Low-score input → LLM classifies as code_change → RouteHarness |
| `AutoInit` | Auto-initializes harness project when auto_init=true |

### Real LLM classifier smoke tests (+3)
Skip unless `GGCODE_TEST_PROVIDER` is set. Keys read from standard env vars, never stored.

- `BugReport` — verifies LLM classifies bug as code_change
- `Question` — verifies LLM classifies question as conversation
- `AmbiguousInput` — verifies LLM resolves score-2 inputs

Soft assertions (log warnings, never fail CI).
