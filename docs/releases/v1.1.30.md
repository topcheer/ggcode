# ggcode v1.1.30

## Highlights

- Completes the TUI/browser stabilization release with the missing agent/runtime pieces included, and
  expands path-aware project memory plus first-class local LSP discovery across more languages and
  workspace-managed binaries.

## Fixes and improvements

### TUI, rendering, and file browsing

- Fixed the spinner / breathing animation fan-out bug that could leave stale tick chains running and
  degrade both UI responsiveness and streamed output throughput.
- Restored live Markdown rendering for conversation output with width-aware wrapping so streamed
  content, saved assistant history, and final flushes render consistently in the active pane.
- Fixed the partial-stream corruption case where Markdown could be prematurely finalized at tool or
  status boundaries.
- Added a dedicated fullscreen project file browser on `Ctrl-F` with:
  - tree indentation
  - distinct expanded / collapsed folder icons
  - `Enter` to expand or collapse directories
  - `/` filename filtering with `Esc` clearing the active filter before exiting
- Improved preview rendering so Markdown files render as Markdown again, source previews wrap to the
  pane width, binary files are blocked from preview, and UTF-8 content is no longer misdetected as
  binary.
- Skips common generated / intermediate directories such as `node_modules`, `dist`, `build`,
  `target`, `classes`, and similar cache or output folders when browsing large workspaces.
- Keeps the waiting indicator in the status bar instead of duplicating a second `Thinking...` line in
  the conversation pane before the first streamed output arrives.

### Project memory and agent/runtime alignment

- The agent now tracks already-loaded project memory files and can inject newly applicable nested
  `GGCODE.md` / `AGENTS.md` / `CLAUDE.md` / `COPILOT.md` guidance when a tool call targets a deeper
  path with additional local instructions.
- Added path-scoped project-memory resolution by walking the target file or directory ancestor chain
  instead of only relying on startup-time workspace loading.
- Aligned pipe mode with the interactive path by propagating loaded project memory files into the
  agent and setting the working directory before execution.
- Tightened the default system prompt so the agent batches obviously-related inspections, avoids
  progress-only filler replies, and treats `todo_write` as phase-level bookkeeping instead of
  micro-step churn.

### LSP discovery and compatibility

- Expanded built-in local LSP discovery to cover:
  - C / C++ / Objective-C via `clangd`
  - Swift via `sourcekit-lsp`
  - YAML via `yaml-language-server`
  - JSON / JSONC via `vscode-json-language-server`
  - Dockerfile / Containerfile via `docker-langserver`
  - Shell / Bash via `bash-language-server`
- Added filename-based matching for Dockerfile-style workspaces and broader workspace evidence
  detection for status reporting.
- Added workspace-managed binary fallback lookup from `.ggcode/tools/` before giving up on local tool
  discovery.
- Kept the previously added TypeScript/JavaScript, Java, and C# compatibility improvements wired into
  the same release line instead of splitting the runtime and TUI changes across mismatched versions.

## Upgrade notes

- No migration is required.
- If you relied on the old clickable file-path decorations inside conversation output, use `Ctrl-F`
  for file browsing and preview instead.
