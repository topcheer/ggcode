# ggcode v1.1.50

## Highlights

- **File edit undo now works** — `/undo` and the inspector checkpoint panel were fully implemented in the UI but the backend never saved checkpoints. This release wires the checkpoint system so every file edit (write_file, edit_file, multi_edit_file, LSP code actions) automatically saves old content for undo/revert.
- **Empty sessions no longer pollute disk** — sessions with no user interaction are no longer persisted, preventing accumulation of empty session files from "open and close" scenarios across all modes (TUI, daemon, ACP, A2A).
- **Three-tier integration tests** — integration tests are now classified into CI-safe (Tier 1), local-only (Tier 2), and service-dependent (Tier 3) tiers, fixing a long-standing CI flakiness issue.

## New Features

### File Edit Checkpoint (Undo/Revert)

The checkpoint system was created and mounted (`checkpoint.NewManager(50)`) and the TUI had full UI for `/undo` and the inspector checkpoint panel — but the write side was never connected. No file editing tool ever called `checkpoint.Save()`.

This release adds a pre-write hook in `atomicWriteFile` that automatically saves old content before overwriting:

```
atomicWriteFile(path, newData)
  → if file exists: read old content → checkpoint.Save(old, new)
  → write new content atomically
```

Works in all modes:
- **TUI**: `/undo` reverts last edit, inspector panel shows full history
- **Daemon**: checkpoint manager now created + hooked (was missing entirely)
- **ACP**: checkpoint manager hooked (was created but never called)

### Empty Session Cleanup

When a session has no user interaction (no user message with actual text content), it is not persisted to disk. This prevents accumulation of empty session files from scenarios like:

- User opens TUI, doesn't type anything, exits
- IDE opens ACP connection, creates session, closes without prompting
- Daemon starts but never receives a message

Applied globally in `internal/session/store.go` (TUI, daemon, A2A) and `internal/acp/session.go` (ACP):
- `Save()`: skips write, deletes existing file if no user interaction
- `EnsureMeta()`: does not create meta file for empty sessions
- `List()`: automatically cleans up orphaned empty session files
- Handler cleanup on disconnect: deletes empty session files on EOF/cancel

### ACP Checkpoint Persistence

ACP now persists compacted context after auto-summarize. Previously, context compaction ran in-memory but the compressed messages were never saved — on session resume the full history would reload and trigger another expensive compact.

### Three-Tier Integration Test Build Tags

Integration tests are now classified into three tiers with build tags:

| Tag | CI-safe? | Content |
|-----|----------|---------|
| `integration` | ✅ CI runs these | Pure mock/local tests, no external dependencies |
| `integration_local` | ❌ Local only | Needs API key for LLM provider |
| `integration_service` | ❌ Local only | Needs external IM services (QQ, Telegram) |

CI now runs `go test -tags=integration` (Tier 1 only) instead of `go test -tags=!integration`.

## Fixes

- **CI harness test failure**: `harness_test.go` git helper set `GIT_CONFIG_GLOBAL=/dev/null` to skip global hooks, but this also masked CI's global git config. Tests that call `commitHarnessScaffold` failed with "empty ident name". Fixed with `initTestRepo()` helper that sets local config.
- **Pre-push hook**: Added `.githooks/pre-push` that runs `verify-ci.sh` before every push, catching CI failures locally.
- **ACP empty session cleanup**: ACP sessions with no conversation are deleted on disconnect (EOF/cancel).
