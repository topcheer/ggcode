# v1.2.1

## Tool Description Labels — TUI 显示优化

所有内置工具现在都会要求 LLM 在调用时传入 `description` 参数，用于在 TUI 聊天区域显示人话描述，替代之前的硬编码标签（如 "Read"、"Edit"、"Run"）。

### 变更详情

**Tool Schema 修复与增强（48 个工具）**
- 修复了 8 个工具的 JSON Schema 完全损坏（`description` 字段在 `properties` 外，缺少闭合大括号）
- 修复了 12 个工具的 `description` 字段在根级而非 `properties` 内
- 所有工具的 `description` 参数标记为 `required`，描述文案强调 `REQUIRED.` + `You MUST always provide this field.`
- 新增 27 个工具的 `description` 参数（config、sleep、save_memory、cron、worktree、MCP、A2A、swarm、team、skill 等）

**TUI 渲染修复**
- `tool_labels.go`：所有 48 个工具的 `describeTool()` 现在优先检查 LLM 传入的 `description`，有值用人话描述，无值 fallback 到本地化标签
- `NewFileToolItem`：改用 `toolName` 而非 `displayName` 决定 body 渲染模式（editdiff/linecount），修复了 description 替换 displayName 后 edit_file 显示 "0 lines" 的问题
- `renderFileLineCount`：错误结果（文件不存在）不再显示误导性的 "0 lines"
- `renderEditDiff`：空 old_text/new_text 不再被 `strings.Split("")` 错误计为 1 行

**todo_write 工具描述增强**
- 工具描述现在明确要求创建 todo 后必须持续更新状态

## Bug 修复

- **自更新后无法重启**：`/update` 命令的 helper 进程原来 `Stdin = nil` 导致断开终端，改为 `Stdin = os.Stdin` 保持终端连接，更新后重启可正常恢复（by GitHub Copilot）
- `command_job_tools.go`：`start_command` 修复了重复的 `description` key 和 `timeout` 缺少 `"type": "integer"`
