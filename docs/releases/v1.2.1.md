# v1.2.1

## Tool Description Labels - TUI Display Optimization

All built-in tools now require the LLM to pass a `description` parameter when invoking, used to display human-readable descriptions in the TUI chat area, replacing the previous hardcoded labels (e.g., "Read", "Edit", "Run"...)

### Change Details

**Tool Schema Fixes and Enhancements (48 tools)**
- Fixed 8 tools with completely broken JSON Schema (`description` field outside `properties`, missing closing braces)
- Fixed `run_command` missing `"additionalProperties": false` causing invalid tool call parameters
- Fixed `stop_command`/`read_command_output` referencing non-existent `list_commands` response format
- Fixed `git_show`/`git_diff` duplicate `description` key in schema

**TUI Label System Refactoring**
- New `labels.go` unified label system, replacing previous per-tool hardcoded labels
- Tool calls show LLM-provided description in TUI, significantly improving readability
- Sub-agent/swarm tool calls also show descriptions

**Label Display Rules**
- File operations: show file path
- Search operations: show search pattern
- Command execution: show command summary
- Git operations: show operation type
- Other tools: show LLM-provided description

## Bug Fixes

- **Unable to restart after self-update**: `/update` command's helper process had `Stdin = nil` causing terminal disconnect, changed to `Stdin = os.Stdin` to maintain terminal connection, restart after update works correctly now
- `command_job_tools.go`: `start_command` fixed duplicate `description` key and `timeout` missing `"type": "integer"`
