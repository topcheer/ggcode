# ggcode v1.3.21

## Highlights

- **Mobile build number moved to pre-push hook**: Build numbers are now bumped locally during `git push` (when the top commit is a release), eliminating CI race conditions where Android and iOS jobs both tried to commit back to main. CI no longer bumps or commits version numbers.
- **Git tool calls visible in TUI**: `git_show`, `git_blame`, `git_branch_list`, `git_remote`, `git_stash_list`, `git_add`, `git_commit`, `git_stash` now render tool headers in the TUI chat area, consistent with mobile.
- **Tunnel reconnect no longer force-resets existing clients**: When a mobile client joins a relay room that already has retained history for the active session, the broker skips the authoritative snapshot broadcast, preventing disruption to other connected clients.
- **Mobile version synced**: Flutter version bumped to `1.3.20+2026052302`.

## Fixes and improvements

- **CI mobile-release.yml**: Removed `version_sync.sh` calls and both "Commit build number" steps from Android and iOS jobs. Version numbers are now set by the pre-push hook before the tag is created.
- **pre-push hook**: Detects `release: vX.Y.Z` commits, runs `version_sync.sh` locally, and commits the bumped build number before the push completes.
- **chat_bridge.go**: Removed "Secondary git tools" skip blocks that prevented git tool headers from rendering in the TUI while being visible on mobile via the unfiltered tunnel path.
