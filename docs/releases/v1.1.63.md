# ggcode v1.1.63

## Highlights

Fixes remaining issues from latest Copilot review: daemon approval semantics, MCP startup timeout, and pipe plugin error handling.

## Fixes

### Daemon approval handler — design documentation
- **daemon.go**: The approval handler always returns `Allow` in daemon mode — this is **by design**, not a bug. Daemon has no TUI; BypassMode and AutoMode naturally auto-approve. For SupervisedMode, the higher-level `ask_user` tool (wired to IM via `DaemonBridge.HandleAskUser`) handles user confirmation when the agent decides to ask. The permission system's `Ask` callback cannot do async IM interaction, so daemon unconditionally allows tool calls and relies on the agent to use `ask_user` when user input is needed. Added detailed comment documenting this design decision.

### MCP ConnectAll startup timeout
- **pipe.go, daemon.go**: `ConnectAll(context.Background())` had no timeout — a broken MCP server could hang startup indefinitely. Added 30-second timeout via `context.WithTimeout`.

### Pipe plugin registration error
- **pipe.go**: `pluginMgr.RegisterTools(registry)` return value was discarded, silently skipping plugin registration failures. Now logs a warning on stderr.
