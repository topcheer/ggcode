# ggcode v1.1.73

## Highlights

Critical fix: promote CTA now targets only the specific task, not all approved tasks. Regression test added.

## Changes

### Fix: single-task promote (P1)
- `handleHarnessPromoteApply()` now calls `PromoteTask(ctx, project, taskID, note)` instead of `PromoteApprovedTasks()`.
- Previous behavior: pressing Enter could batch-promote all approved tasks if multiple existed.
- New behavior: only the task shown in the CTA is promoted.
- Regression test: `TestPromoteTask_OnlyTargetsSpecificTask` — 3 approved tasks, promote task-2, verify task-1 and task-3 untouched.

### Design doc accuracy
- Status updated to v1.1.72 with single-task promote note.
