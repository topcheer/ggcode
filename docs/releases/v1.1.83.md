# ggcode v1.1.83

## Highlights

**WeChat iLink: context_token persistence and reliable outbound delivery** â€” Previously, WeChat outbound messages stopped after ~2 messages because the iLink `context_token` was only held in memory and lost on restart. Now, the token is persisted to the binding store file (`~/.ggcode/im-bindings.json`), updated on every inbound message, and survives process restarts. The `channel_version` was also upgraded from `1.0.0` to `2.0.0` per iLink protocol spec.

**WeChat & DingTalk: tool call/result push support** â€” Both adapters now use the shared `outboundText()` pipeline (same as QQ, Telegram, Discord, Slack, Feishu), so tool calls and tool results are formatted and pushed to WeChat and DingTalk channels in real-time. Previously, only plain text messages were delivered.

**WeChat adapter: Send uses ChannelID consistently** â€” Fixed `Send` to use `binding.ChannelID` as the destination (same as all other adapters), instead of the ambiguous `TargetID` fallback. WeChat panel `Bind` now also sets `TargetID` to the workspace directory name, consistent with other adapter panels.

## Bug Fixes

### WeChat iLink adapter
- **context_token persistence**: Token is now stored in `ChannelBinding.ContextToken` and persisted to disk via `UpdateBindingContextToken()`. Survives restarts. Updated on every inbound message.
- **channel_version 2.0.0**: All API calls now use `channel_version: "2.0.0"` per iLink protocol spec (was `1.0.0`).
- **outboundText unified**: `Send()` now uses `a.outboundText(event)` instead of `event.Text`, enabling tool call/result/status push to WeChat (same formatting as other adapters).
- **ChannelID for Send**: Uses `binding.ChannelID` as `toUserID` (all other adapters use `ChannelID`; WeChat was incorrectly using `TargetID` with fallback).
- **EOF recovery**: Poll errors from EOF/timeout are now classified as recoverable and trigger silent retry instead of publishing an error state.
- **Panel bound count fix**: Bound count now correctly includes adapters bound to the current workspace (not just "occupied by other workspace").
- **Panel Bind sets TargetID**: Consistent with other adapter panels â€” sets `TargetID` to `filepath.Base(workspace)`.

### DingTalk adapter
- **outboundText unified**: `Send()` now uses `a.outboundText(event)` instead of filtering `event.Kind == OutboundEventText`. All event types (tool call, tool result, status, approval) are now pushed to DingTalk.

### IM runtime
- **Dynamic platform name in pairing messages**: Hardcoded "QQ" in blacklist/rebind/active-pairing messages replaced with dynamic `msg.Envelope.Platform` so WeChat and other adapters see correct platform names.

### Feishu adapter
- **Last error cleared on reconnect**: `publishState(true, "online", "")` now clears the error message on successful WebSocket reconnect (was showing stale `"websocket connected (SDK)"`).
