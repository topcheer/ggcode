# ggcode v1.1.91

## Highlights

- **Fixed tool header truncation** — Tool headers in the TUI were silently truncated to ~60 characters due to a Go struct embedding bug. `BaseToolItem.RenderParams()` now uses a `params` field set at construction time, eliminating the need for subclass overrides that were never called.

- **Precise display width control** — TUI tool headers truncate at 120 rendered characters (with word-wrap), status bar activity truncates at 30 rendered characters, and IM command display is no longer truncated.

- **Eliminated fake inheritance** — Removed all Go struct embedding with value-copy method overrides across `BashToolItem`, `FileToolItem`, `SearchToolItem`, `ListToolItem`, `WebToolItem`, `GitToolItem`, `CmdToolItem`, and `LspToolItem`. All rendering logic now lives in `BaseToolItem` directly.

## Fixes and improvements

- **TUI**: Tool header text no longer truncated at 60 characters for `run_command` and other tools.
- **TUI**: Tool headers truncate at 120 rendered visual width characters, then word-wrap to terminal width.
- **TUI**: Status bar tool name+arg truncated to 30 rendered characters with ellipsis.
- **TUI**: `suppressHeader` handling moved into `BaseToolItem.Render`.
- **IM**: Command lines in IM notifications (`⚡`) and results (`✅`/`❌`) no longer truncated.
- **IM**: Successful command results show only `✅ \`cmd\`` (no output); failed commands show `❌ \`cmd\`` with stderr.
- **Internal**: `BaseToolItem.params` field set at construction, `RenderParams()` returns it directly.
- **Internal**: Deleted 7 subclass struct fields (`command`, `filePath`, `pattern`, `path`, `url`, `subCmd`, `detail`, `location`) and all corresponding `RenderParams`/`Render`/`RenderBody` overrides (-177 lines, +24 lines net).

## Upgrade notes

- No config changes required.
- No breaking API changes.
