# ggcode v1.3.171

## Highlights

- **Windows PowerShell-first**: PowerShell is now the primary shell on Windows (was Git Bash), with cached detection for faster startup.
- **PowerShell dangerous command detection**: 16 dangerous PowerShell command patterns added to the safety system.
- **Shell type in system prompt**: The agent now knows which shell it's running in, enabling OS-aware command suggestions.

## New Features

### PowerShell as Primary Windows Shell (`d0596abb`)

Windows shell detection priority is now:

1. **PowerShell** (`pwsh` or `powershell.exe`) — primary
2. **Git Bash** (`bash`) — fallback

This replaces the previous Git Bash-first approach. PowerShell provides better Windows-native integration and is pre-installed on all modern Windows systems.

Standard flags: `-NoProfile -ExecutionPolicy Bypass` (accepted trade-off: profile scripts like nvm/conda are not loaded, but system environment variables are still inherited).

### Shell Type in System Prompt (`bffe5794`)

The agent's environment info now includes the detected shell type (e.g., `Shell: zsh`, `Shell: powershell`). This enables the agent to generate shell-appropriate commands without trial and error.

### PowerShell Dangerous Command Detection (`d1992243`)

Added 16 PowerShell-specific dangerous command patterns to `internal/permission/dangerous.go`, including:

- `Remove-Item -Recurse -Force`
- `Format-Volume`
- `Stop-Computer` / `Restart-Computer`
- `Set-ExecutionPolicy Unrestricted`
- Registry deletion (`Remove-Item HKLM:\...`)
- Process kill patterns (`Stop-Process -Force`)
- PowerShell history deletion
- Disk wipe commands

The `verify.go` command execution path now uses `util.NewShellCommandContext` instead of hardcoded `sh -c`, enabling Windows shell-aware verification.

## Performance

### Cached Shell Detection (`4f0134f5`)

`DetectShell()` is now cached with `sync.Once`, eliminating repeated shell probing on every command execution. Additionally, detection now uses `-c` instead of `-lc` (login shell), which is faster and avoids loading profile scripts.

This is especially impactful on Windows where PowerShell detection was being re-run on every `run_command` call.

## Diagnostics

### IM Runtime Silent Error Paths (`872b2129`)

Added `debug.Log` to two silent error paths in `internal/im/runtime.go`:

- `AllPersistedBindings` — error loading binding file was silently swallowed
- `ClearSessionBindingsGlobal` — error clearing session bindings was silently swallowed

### Shell Detection Logging (`33681687`, `de483eac`)

Shell type is now logged on first detection via `debug.Log` (category: `util`), replacing an unconditional `log.Printf`. This helps diagnose shell-related issues on Windows without adding noise on subsequent detections.

## Upgrade Notes

**Windows users**: If you relied on Git Bash-specific features or environment variables set in `.bashrc`/`.bash_profile`, these will no longer be available by default. System-level environment variables are still inherited. To restore Git Bash as the shell, set `SHELL=bash` in your environment.

No breaking changes for macOS/Linux users.
