# v1.3.6 — Desktop UX Improvements & Mermaid Fix

## Desktop: UX Improvements

### File Browser
- **Refresh button** next to search box — reloads file tree from disk without searching
- Better icon for "New Session" button (`DocumentCreateIcon`)
- Better icon for "Add Image" button (`FileImageIcon`)

### Mermaid Diagram Rendering
- **Fixed crash** when rendering Mermaid diagrams — Fyne canvas.Image does not support SVG; switched to PNG format
- **Dual backend fallback**: kroki.io (primary) → mermaid.ink (fallback)
- **Local file cache** under `.ggcode/mermaid-cache/` — SHA-256 hash-keyed PNG cache avoids redundant API calls
- Rendering failure shows raw code + "Open in Mermaid Live Editor" button
- PNG header validation prevents crashes on invalid response data

### Preview Panel
- Preserved 75:25 sidebar ratio when showing file preview (was incorrectly 50:50)
- Text files (.txt, .log, .csv, .ini, .env) use word wrap instead of horizontal scroll
- Code files keep horizontal scroll with line numbers
- Markdown uses markdownx with GFM table support instead of Fyne's built-in RichText

### About & Updates
- Help → About dialog with app icon, version, description, GitHub links
- Help → Check for Updates queries GitHub API for latest release

## Documentation
- Added `goolm` build tag requirement to GGCODE.md
- Updated README with desktop file browser features, Linux binary, and build instructions
- Release scripts (darwin/linux/windows) now inject `main.Version` for About dialog

## Testing
