# ggcode v1.3.1

## Highlights

- **CI reliability fixes** — Desktop build jobs now wait for the GoReleaser release job to complete before uploading, preventing "release not found" failures. Windows desktop artifact path fixed for site deployment.
- **Build cache enabled** — Go module cache (`cache: true`) enabled across all CI/release workflows, significantly reducing build times for subsequent runs.

## Fixes and improvements

- Desktop macOS build job waits for `release` job to create GitHub Release before uploading
- Desktop Windows build job waits for `release` job similarly
- Windows desktop artifact upload path fixed (`dist\` instead of `dist/*.exe` glob)
- Go module caching enabled in `release.yml` and `ci.yml` (`cache: true`)
- Site deployment branch includes desktop artifacts in asset directory list
