# ggcode v1.1.21

## Highlights

- Added a fullscreen file preview flow in the TUI with markdown rendering, syntax highlighting, and scrollable full-file viewing.
- Fixed several TUI interaction regressions around clickable preview links, preview dismissal, tool status updates, and stream/runtime responsiveness.
- Moved active task tracking into the right sidebar and expanded runtime i18n coverage for harness, MCP, and provider surfaces.

## Fixes and improvements

### Fullscreen preview and file navigation

- Added clickable file preview targets in conversation output with corrected hit testing against the final rendered frame.
- Replaced the side/bottom snippet preview with a fullscreen modal viewer that renders the full file, wraps naturally, and supports mouse, arrow key, and page navigation.
- Markdown files now render as markdown in preview, and common source/config files render with terminal syntax highlighting.
- Escape now closes preview before canceling an active run.

### Runtime and streaming polish

- Fixed stale tool-count reporting in the status bar so live tool usage reflects the event loop state.
- Reduced redundant per-chunk writing status updates during streaming to cut TUI message churn.
- Avoided expensive full-history preview-link scans during view rendering, which previously could stall the UI and interfere with cancellation.

### Task tracking and localization

- Removed the old in-stream “advancing tasks” blocks from the main pane.
- Added a sidebar task tracker that takes over the right panel while tasks are active, orders tasks by newest start time, and color-codes status markers.
- Localized additional user-visible runtime copy across harness flows, harness progress formatting, MCP panel actions, and provider runtime activation warnings.

## Upgrade notes

- No migration is required.
- Existing sessions and configs continue to work as before.
