# ggcode v1.1.58

## Highlights

Addresses 10 issues identified by Copilot code review, including one **deadlock**, four **lifecycle/resource leaks**, two **TUI performance bottlenecks**, and three **entry path inconsistencies**.

## Fixes

### P0: Self-deadlock
- **session/store.go**: `List()` held `s.mu` then called `repairIndex()` → `s.Load()` → `s.mu.Lock()` again. Fixed by calling internal `loadSession()` (no lock) instead of public `Load()`.

### P1: Resource leaks
- **pipe.go**: `mcpMgr.ConnectAll()` had no corresponding `Close()`. Added `defer mcpMgr.Close()`.
- **plugin/mcp_loader.go**: OAuth-required early return left MCP client process running. Added `client.Close()` before returning `MCPOAuthRequiredError`.

### P1: Subagent manager concurrency
- **subagent/manager.go**: `Complete()` could overwrite a cancelled agent's terminal state. Added terminal state check (`Cancelled/Completed/Failed` → skip). `onComplete`/`onUpdate` callback reads/writes now protected by `m.mu`.

### P1: TUI performance
- **view.go**: `sidebarGitBranch()` resolved `.git` directory via stat-walking on every `View()` render. Added `resolveGitDirCached()` with package-level cache, invalidated every 2s via `gitBranchTickMsg` timer.
- **harness_panel.go**: `refreshHarnessPanel()` did 9+ serial harness reads on main thread. Added 500ms debounce (skipped if data already loaded within window) and `refreshHarnessPanelForced()` for auto-refresh checks. Laid groundwork for future full async `tea.Cmd` conversion.
- **file_browser.go**: `buildFileBrowserEntries()` could scan unbounded directory trees. Added 2000-entry limit to prevent UI freeze on large repos.
- **preview_panel.go**: `buildPreviewPanelStateForPath()` synchronously read arbitrarily large files. Added 1MB size limit — larger files show "too large" error instead of blocking.
- **submit.go**: `appendUserMessage()` held `sessionMutex` during disk I/O. Restructured to release lock before calling `AppendMessage()`.

### P1: Entry path consistency
- **pipe.go**: Missing `SetHookConfig`, `SetCheckpointManager`, `SetPreWriteHook`, `SetSupportsVision` — now aligned with root.go and daemon.go.
- **daemon.go**: Missing `SetProjectMemoryFiles()` call — now passes memory file list to agent for live tracking.
