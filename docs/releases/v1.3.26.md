## v1.3.26

### Desktop Tunnel Fix — Critical

Desktop's tunnel event pushing was completely misaligned with the TUI reference implementation, causing mobile clients to receive no agent streaming data (text, tool calls, tool results) when connected to a desktop instance.

#### Root Cause

Desktop `onEvent` callback diverged from TUI `pushTunnelEvent` in three critical ways:

1. **`StreamEventError` completely missing** — if the agent errored mid-stream, the text stream was never closed. Mobile stayed in streaming state forever with no recovery path.

2. **`StreamEventDone` pushed idle on every LLM turn** — agent loops have multiple turns (text → tool → text), and each `Done` event set status to idle. This caused mobile status bar to flicker idle between turns, and more importantly could interfere with event ordinal tracking.

3. **Agent run success path did not push idle** — only the error path pushed idle + cleared activity. A successful agent run left mobile stuck in busy state indefinitely.

#### Fix

Aligned desktop `onEvent` with TUI `pushTunnelEvent`:
- Added `StreamEventError` handler: `flushText` + `PushError`
- `StreamEventDone`: only `flushText` (removed premature idle/activity clear)
- `StreamEventSystem`: removed spurious `PushSystemMessage`
- Agent run completion: always push idle + clear activity, regardless of success/error

#### Mobile Stop Button — Desktop

Desktop `OnCommand` was missing the `interrupt` command handler entirely. Tapping stop on mobile sent `interrupt` but desktop silently ignored it. Now calls `agentBridge.Cancel()` to cancel the running agent context.

#### Test Fix

`TestStartAgentWithExpandPushesInitialTunnelBusyAndActivity` leaked a goroutine that continued writing to TempDir after test cleanup, causing flaky CI failures. Added proper context cancellation with cleanup delay.
