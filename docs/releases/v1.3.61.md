# v1.3.61

Wails 桌面端全面重构：事件驱动渲染、IM 管理 API 修复、Debug Console、TUI 工具结果显示修复。

## Highlights

### Wails 桌面端事件驱动渲染
- 从 250ms 全量轮询 `GetSessionHistory` 改为事件驱动增量渲染
- ChatView 切换页面时保持消息状态（display:none 而非 unmount）
- IM/移动端用户消息推送到前端显示（`user_message` 事件 + 来源标识）
- 非手动发送的用户消息显示 "via IM" / "via Mobile" 标识

### IM 管理操作修复
- 所有 IM 管理操作（enable/disable/mute/unmute/save/delete/bind/unbind）现在调用 runtime 启停
- Mute/Unmute 是运行时状态，从 `Manager.IsMuted()` 读取而非持久化 store
- 启动 adapter 前绑定 session，修复 `HandlePairingInbound` 返回 `ErrNoSessionBound`
- 桌面端 Discord/QQ/Telegram 等适配器配对和消息收发完整可用

### Debug Console
- 新增运行时日志查看器，无需 `GGCODE_DEBUG` 环境变量
- LogStream 独立于文件日志，支持按 category 筛选和搜索
- IM/Relay/Tunnel/Broker 等全部 category 有独立颜色标识

## Fixes and improvements

### Desktop (Wails)
- 修复 ChatView 切换页面后消息丢失
- 修复 ChatView 无法滚动（flex 容器高度约束）
- 修复 streaming tool call 显示静态点而非 spinner
- 修复 `run_done` 不关闭 tool call streaming
- 修复 reasoning 消息渲染
- 修复 sub-agent/swarm 队列面板渲染
- 新增模型选择器下拉菜单
- 新增文件浏览器（语法高亮、Markdown、图片预览）
- 新增 Share 对话框（QR 码）
- 新增权限模式选择
- 新增 MCP/IM 实时状态面板

### TUI
- 修复 tool result error message 行数计算（缺少 word-wrap 导致内容溢出面板）

### Config
- 移除 config 加载/保存时的刷屏日志（只保留错误级别）

### Tunnel
- 统一 TunnelHost 管理三端（TUI/Daemon/Wails）tunnel 事件推送
- 移除 legacy tunnel 死代码

### Config Tool
- 新增 provider hot-reload（vendor/endpoint/model/api_key 变更后自动刷新）
- 新增 model 列表和发现功能
- 新增 UI 刷新回调
