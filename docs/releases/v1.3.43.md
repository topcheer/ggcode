# v1.3.43

## Features

### Per-turn Usage History

- **Individual LLM call records**: Each LLM API call now persists a `UsageEntry` to the session JSONL with turn index, model, vendor, endpoint, and full token usage (input, output, cache read, cache write, prompt tokens total).
- **Turn tracking**: `usageTurnIndex` increments once per user turn (including cron, tunnel, and all agent run entry points). Restored from history max on session resume.
- **Sub-agent/teammate coverage**: Usage from sub-agents and swarm teammates flows back to the parent session via `onUsage` callback.

### Async Performance Metrics

- **LLM metrics**: Time to first token (TTFT), thinking/reasoning duration, total LLM call wall time.
- **Tool metrics**: Tool name, success/failure status, error message, execution duration.
- **Non-blocking**: `metrics.Collector` uses a buffered channel (256) with background goroutine persistence. Events are dropped (never blocked) if the channel is full, so the agent is never slowed down.
- **Lifecycle**: Collector is stopped on REPL exit, Desktop close, and Daemon close to drain remaining metrics before session save.

### Coverage

- **TUI**: Full metrics via `SetMetricHandler` on REPL agent.
- **Desktop**: Full metrics via `SetMetricHandler` on AgentBridge agent.
- **Daemon (IM)**: Full metrics via `SetMetricHandler` on DaemonBridge agent.
- **pipe mode**: Not yet wired (no session persistence in pipe mode).

## Bug Fixes

- **TUI turn index**: `startAgentWithExpand` (the main text submission path) was missing `usageTurnIndex++`, causing metrics from normal chat submissions to be attributed to the wrong turn.
