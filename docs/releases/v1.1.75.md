# ggcode v1.1.75

## Highlights

Completes the harness auto-run feature. All 7 phases implemented including LLM classifier. Review/promote CTA integration tests added. Design doc marks 100% ready.

## Changes

### LLM classifier (4th routing layer)
- `ClassifyWithLLM()` uses LLM to classify ambiguous prompts (structural score 1-2) as code_change or conversation.
- Only called in "on"/"strict" modes when provider is available and input ≥ 20 chars.
- 3-second timeout. On error/timeout: falls back to deterministic RouteNormal (safe default).
- High confidence (≥0.8) → RouteHarness. Medium (≥0.5) → RouteSuggest. Low → RouteNormal.
- Strict mode: ≥0.5 → RouteHarness.
- Provider passed from all 3 entry points: TUI (`m.agent.Provider()`), pipe, daemon.
- 15 unit tests covering: parse, nil provider, timeout, provider error, truncation, route decisions.
- 4 router integration tests: LLM override, score-too-high skip, no-provider skip, suggest-mode skip.

### Review/promote CTA integration tests (+9)
- `TestReviewCTA_EnterApproves` / `_EscSkips` — Enter triggers approve cmd, Esc dismisses.
- `TestPromoteCTA_EnterPromotes` / `_EscSkips` — Enter triggers promote cmd, Esc dismisses.
- `TestCTA_PriorityOrder_ReviewBeforePromote` — review handler fires before promote.
- `TestCTA_StaleCleanupOnNewInput` — pendingAutoRun cleared on new text submit.
- `TestHarnessRunResult_SetsPendingReview` — CTAReview sets pendingHarnessReview.
- `TestReviewResult_SetsPendingPromote` — ReviewApproved sets pendingHarnessPromote.
- `TestPromoteResult_ClearsPendingState` — promoted result clears all pending state.

### Design doc: Completed
- Status changed from "In Progress" to "Completed".
- Readiness: 100%.
- "What is missing" section: all gaps resolved. Only optional future enhancements remain.
- All 9 success criteria verified and met.
