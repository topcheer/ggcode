# v1.3.96

## Fixes

- **Session persistence for PermissionMode and SidebarVisible**: Fixed a bug where `PermissionMode` and `SidebarVisible` were stored on the `Session` struct but never serialized to the JSONL meta record. Now both values are correctly written to and read from the session file, so resuming a session restores your permission mode and sidebar visibility. (`SidebarVisible` uses `*bool` to distinguish "never set" from "explicitly hidden".)
- **Ctrl+R sidebar toggle now persists to session**: Toggling the sidebar with Ctrl+R previously wrote to global config (`SaveSidebarPreference`), affecting all sessions. Now it saves to `session.SidebarVisible` via `persistSidebarPreference()`, leaving global config unchanged.
- **lanchat tool result no longer truncated**: The `lanchat` tool output in the TUI was truncated to the default max lines. Now it renders in full with markdown formatting (`noTruncate` + `markdownBody`).

## Features

- **IM tool for LLM agents**: The `im` tool is now available to the LLM in all modes (TUI, daemon, desktop). It allows the agent to manage IM adapters (status, mute/unmute, disable/enable) and send messages. The tool is always allowed in every permission mode (including plan mode). Manager injection happens at runtime in all three host modes.
- **IM tool always allowed**: `im` and `lanchat` are both always allowed without approval in all permission modes via `IsAlwaysAllowedTool()`.
- **Daemon mode IM tool wiring**: Daemon mode now correctly injects the IM runtime manager into the `im` tool at startup.

## Documentation

- Updated `GGCODE.md` with IM tool details, mute vs disable semantics, workspace-scoped bindings, session persistence fields, and `im` always-allowed policy.
- Updated `docs/guide/modes.md` to document session-scoped persistence for both permission mode and sidebar visibility.
- Updated `docs/guide/im-integration.md` with full `im` tool reference (actions, mute vs disable, workspace scope).
- Updated `docs/ARCHITECTURE.md` session store description with persisted meta fields.
- Consolidated 54 auto-memory files into 21 focused files.
