# ggcode v1.3.20

## Highlights

- **Tunnel event persistence and replay**: Tunnel events are now persisted to session JSONL files. When mobile clients reconnect, the broker replays canonical events for seamless session recovery instead of relying solely on snapshots.
- **Relay event deduplication**: The relay server deduplicates events by eventID, preventing history bloat on repeated reconnects.
- **Documentation audit**: Full review and update of GGCODE.md and ARCHITECTURE.md — fixed stale file counts, added 12 missing packages, corrected tool counts, added tunnel/relay/swarm architecture docs.

## Fixes and improvements

- **Mobile connection feedback**: QR code overlay auto-closes when mobile connects; system message confirms connection.
- **Manual workspace URL entry**: Mobile scanner supports pasting a workspace URL directly.
- **DingTalk Markdown**: DingTalk adapter sends replies in Markdown format.
- **Discord community links**: Added across README, npm/Python packages, Desktop app, TUI QR overlay.
- **A2A Token validator**: `NewTokenValidator` requires non-empty issuer URL; new `WithValidIssuers` and `WithHMACSecret` options.
- **A2A local auth**: `isLocalRequestHost()` matches all local interface IPs, not just 127.0.0.1/::1.
- **A2A client error handling**: HTTP error body extracted for non-200 responses instead of blind JSON decoding.
- **Relay backpressure**: Peer writes use blocking sends with 30s deadline instead of channel drops.
- **snapshot_reset eventID fix**: Control events no longer consume event ordinals, keeping IDs contiguous after replay.
- **Tunnel events completeness**: `TunnelEventsComplete` flag ensures only fully-recorded event sets are used for replay.
