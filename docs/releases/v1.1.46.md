# ggcode v1.1.46

## Highlights

- **A2A multi-scheme authentication** — the A2A (Agent-to-Agent) server now supports five authentication schemes, individually configurable or combined: API Key, OAuth2 + PKCE, OAuth2 Device Flow, OpenID Connect (with JWKS key rotation), and mutual TLS. Built-in provider presets for GitHub, Google, Auth0, and Azure enable zero-config setups. The client auto-negotiates the strongest available scheme from the server's agent card.
- **OAuth2 token persistence** — OAuth2/OIDC tokens are cached per-client at `~/.ggcode/oauth-tokens/{provider}-{clientID}.json` with 0600 permissions. The agent reuses cached tokens on restart without re-authentication. Same `client_id` shares tokens; different `client_id` = isolated cache, preventing overwrites between instances.
- **IM adapter management slash commands** — daemon mode now supports `/listim` (list adapters with status), `/muteim <name>` (mute specific adapter), `/muteall` (mute all except sender), `/muteself` (mute current adapter), `/restart` (restart daemon, unmutes all). Mute is in-memory only — not persisted to binding store.
- **Built-in WebUI** — new embedded SPA with REST API for config management. Supports vendor/endpoint CRUD, API key management, MCP server status and tools view, IM adapter management grouped by platform, system prompt editing, and impersonate settings.
- **A2A spec v1.0 alignment** — full A2A protocol implementation: push notification callbacks, extended agent card, SSE streaming with final field, task observability in TUI sidebar, daemon follow display, and IM notifications. 5-instance mesh E2E test verifies all auth methods interop in <1s.
- **Instance-level A2A override** — `.ggcode/a2a.yaml` in workspace root merges into global A2A config, allowing per-project auth settings.

## Fixes and improvements

### A2A authentication

- OAuth2 + PKCE flow with dynamic port (`localhost:0`) for callback URL.
- OAuth2 Device Flow with auto clipboard copy of device code and user-friendly URL display.
- OIDC Discovery with JWKS key rotation, supporting HS256/RS256/ECDSA JWT validation.
- mTLS with per-instance certificates, verified against shared CA.
- Client auto-negotiation: Discover → NegotiateAuth → setAuth. Picks the strongest scheme both sides support.
- GitHub zero-config: built-in public client_id in preset, just set `provider: "github"`.
- Server auth is config-driven (no persistence needed); client tokens survive restart via cache.
- Instance-level config override via `.ggcode/a2a.yaml` with field-level merge.

### IM integration

- `/listim` shows adapter name, platform, health status, and mute state.
- `/muteim <name>` mutes a specific adapter; refuses to mute self (suggests `/muteself`).
- `/muteall` uses `MuteAllExcept(selfAdapter)` — sender's adapter is never muted, preventing the mute-then-unmute race.
- `/muteself` emits warning before muting (500ms delay ensures message delivery), suggests `/restart` for recovery.
- `/restart` hook triggers daemon self-restart, recovers all muted adapters (mute is in-memory, not persisted).

### WebUI

- Embedded SPA served from `/` with REST API at `/api/*`.
- Vendor & endpoint management: full CRUD with API key input (masked).
- MCP server page: shows command args, env vars, runtime status, and available tools.
- IM adapters page: grouped by platform with runtime status and persisted bindings merged.
- System prompt page: markdown preview and plain text editing.
- Impersonate page: preset switching, version display, custom headers.
- Restart button triggers daemon self-restart and closes the window.
- Config structs: added missing JSON tags for clean API serialization.

### MCP

- OAuth device code auto-copied to clipboard during MCP OAuth 2.1 flow.
- Daemon follow mode handles MCP OAuth device code flow prompts.
- MCP page loads from `/api/mcp` (with runtime status) instead of `/api/config`.

### Daemon mode

- `/restart` and `/help` IM slash commands trigger daemon self-restart.
- WebUI restart button triggers daemon self-restart.
- IM adapter config properly separated from ghost persisted bindings.

### Configuration

## Upgrade notes

- **A2A is now enabled by default** — set `a2a.disabled: true` to opt out.
- **`a2a.auth.api_key` replaces `a2a.api_key`** — the legacy field still works but the new nested form is preferred.
- **Token cache location** — OAuth2 tokens are cached at `~/.ggcode/oauth-tokens/`. If you have multiple ggcode instances with different OAuth configs, they now use isolated cache files.
- **IM mute is in-memory** — restarting the daemon recovers all muted adapters. No persistent mute state.
