# v1.3.90

## LAN Chat: Smarter Messaging, Less Noise

### Nick Resolution for Send Action

The `send` action now accepts **nicks** (e.g. `dd_dev_agent`) and
**partial nicks** (e.g. `dd`) as the `to` parameter, not just raw
node IDs. Previously, LLMs naturally used nicks as identifiers but the
Hub only accepted node IDs, causing "unknown peer" errors on every send
attempt.

Resolution order:
1. Exact `node_id` match
2. Exact `human_nick` or `agent_nick` match (case-insensitive)
3. Unique prefix match (e.g. `dd` matches `dd_dev_agent`)

When no recipient is found, the error includes the full participant list
to help the LLM self-correct.

### Team Filter for List Action

The `list` action now accepts an optional `team` parameter:
```
lanchat(action='list', team='platform')
```
Returns only members of the specified team. Useful for the LLM to quickly
find who belongs to a specific team without parsing the full list.

### Anti-Noise Rules

Comprehensive anti-noise guidance added across three surfaces:

**lanchat tool description** — new ANTI-NOISE RULES section with 7 rules:
- Do NOT broadcast unless user explicitly asks
- Do NOT send confirmation/acknowledgment messages
- Do NOT reply to broadcasts unless you have actionable information
- Check `agent_busy` before messaging — don't ping busy agents
- One message per task — no follow-up pings
- Prefer targeted DMs over broadcasts

**System prompt** (Collaboration routing):
- Removed "use lanchat FIRST" and "ALWAYS try lanchat FIRST"
- Added "ALWAYS prefer targeted DMs over broadcasts"
- Added "Do NOT send acknowledgment messages"
- Added "Do NOT broadcast status updates or routine coordination"

**a2a_remote tool description**:
- Removed broadcast encouragement
- Synchronized with the same anti-noise guidance

### Default `as_agent=true`

The lanchat tool is almost always called by the agent itself.
`as_agent` now defaults to `true`. Explicit `as_agent=false` still works
for forwarding as the human user. `to_role` default changed to `agent`
to match.
