# ggcode v1.1.90

## Highlights

- **Plan mode display overhaul** — `enter_plan_mode` shows a clean "✓ Planning" indicator; `exit_plan_mode` renders the plan as a markdown assistant message (● prefix) instead of a tool card, consistent across TUI, daemon follow, and IM
- **Task tools hidden across all surfaces** — internal LLM task tracking tools (`task_create`, `task_get`, `task_update`, `task_list`, `task_stop`) no longer produce visual noise in TUI, daemon follow, or IM

## Tool display changes

### Plan mode (TUI / daemon follow / IM)
- `enter_plan_mode` — TUI: `✓ Planning` header only; daemon follow: header; IM: `📝 Planning...`
- `exit_plan_mode` — TUI: plan content rendered as ● assistant message with markdown; daemon follow: plan as markdown, no header; IM: plan content as text

### Hidden tools (TUI / daemon follow / IM)
- `task_create`, `task_get`, `task_update`, `task_list`, `task_stop` — completely hidden; these are internal LLM self-constraint tools, not user-facing

## Fixes and improvements

- Fix `suppressHeader` to render body-only instead of hiding the entire tool item
- Fix IM `enter_plan_mode` notification not being sent (TUI only emitted `OutboundEventToolResult`, not `OutboundEventToolCall`)
- Fix `exit_plan_mode` plan extraction — use `rawArgs` JSON field instead of parsing result text
- Add `List.RemoveByID` helper to chat list
- Align cron tool display across TUI/daemon/IM

## Upgrade notes

- No breaking changes. Plan mode and task tools simply appear cleaner.
