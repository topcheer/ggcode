# ggcode v1.3.2

## Highlights

- **Sub-agent/teammate tab recycling** — Follow strip tabs for completed sub-agents and idle teammates are now properly cleaned up after a grace period. Previously, tabs could remain visible indefinitely because the TUI was not notified when agents completed.
- **CI build reliability** — Windows desktop build now uses absolute output paths to prevent file-not-found errors. Artifact upload runs before release upload to ensure files are available for site deployment.
- **Go module cache** — Enabled across all CI workflows, significantly reducing build times for subsequent runs.

## Fixes and improvements

- `subagent/manager.go`: call `notifyUpdate()` in `Complete()` so the TUI follow strip starts the grace-period timer when a sub-agent finishes
- `swarm/idle_runner.go`: set `EndedAt` when a teammate finishes a task, enabling proper grace-period based cleanup of teammate slots
- Windows desktop build: PowerShell script resolves output directory to absolute path before `Push-Location`
- Desktop artifact upload order: artifact upload runs first, release upload second (prevents empty artifact on Windows)
- All CI workflows: `cache: true` for Go module caching
