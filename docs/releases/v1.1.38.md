# ggcode v1.1.38

## Highlights

- **Homebrew tap distribution** — install and update ggcode on macOS and Linux with `brew install topcheer/ggcode/ggcode`.
- **Chinese README** — full Chinese documentation at [`README_zh-CN.md`](../README_zh-CN.md) with language switch links.
- **IM tool result formatting overhaul** — command output now uses code blocks in IM channels for better readability.
- **Website updated** — Homebrew install card, IM feature highlights, and version badge refreshed.

## Fixes and improvements

### Homebrew

- Adds GoReleaser `brews` section for automatic Homebrew tap publishing to `topcheer/homebrew-ggcode`.
- Website and README now show `brew install topcheer/ggcode/ggcode` as a first-class install option.

### IM integration

- Command tool results (bash, PowerShell) now render with `\`\`\`bash` code blocks in IM channels instead of truncated inline previews.
- `ask_user` IM emission logic refactored: questions are emitted when the `AskUserMsg` creates the questionnaire state, not from the agent round message.
- Fixed `TestAskUserRoundEmitsExplicitQuestionMessage` to match the new emission flow.

### CI / Release pipeline

- Windows smoke test set to `continue-on-error: true` with Defender exclusion hints.
- Removed `release-smoke-windows` from downstream job `needs` to prevent Defender timeouts from blocking macOS/Windows package builds.

### Documentation

- Added `README_zh-CN.md` with complete Chinese documentation.
- Added logo banner to both English and Chinese READMEs.
- Updated `docs/site/index.html` with IM-native remote coding feature card, Homebrew install card, and version badge.
- Updated `ggcode.example.yaml` with full IM adapter configuration for all 6 platforms (QQ, Telegram, Discord, Feishu, DingTalk, Slack).
- Updated `README.md` with IM integration section, Homebrew install, and updated slash commands.
