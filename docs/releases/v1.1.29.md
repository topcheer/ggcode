# ggcode v1.1.29

## Highlights

- Stabilizes the TUI after the recent rendering regressions by fixing runaway spinner ticks, restoring
  responsive live Markdown rendering, and shipping a dedicated fullscreen file browser with richer
  preview behavior.

## Fixes and improvements

### TUI responsiveness and rendering

- Fixed the spinner / breathing animation fan-out bug that could leave stale tick chains running and
  degrade both UI responsiveness and streamed output throughput.
- Restored live Markdown rendering for conversation output with width-aware wrapping so streamed
  content, saved assistant history, and final flushes render consistently in the active pane.
- Fixed the partial-stream corruption case where Markdown could be prematurely finalized at tool or
  status boundaries.

### Fullscreen file browser

- Added a dedicated fullscreen project file browser on `Ctrl-F` with:
  - tree indentation
  - distinct expanded / collapsed folder icons
  - `Enter` to expand or collapse directories
  - `/` filename filtering with `Esc` clearing the active filter before exiting
- Replaced the old in-conversation clickable path preview flow with the browser-driven preview
  experience.
- Improved preview rendering so Markdown files render as Markdown again, source previews wrap to the
  pane width, binary files are blocked from preview, and UTF-8 content is no longer misdetected as
  binary.
- Skips common generated / intermediate directories such as `node_modules`, `dist`, `build`,
  `target`, `classes`, and similar cache or output folders when browsing large workspaces.

## Upgrade notes

- No migration is required.
- If you relied on the old clickable file-path decorations inside conversation output, use `Ctrl-F`
  for file browsing and preview instead.
