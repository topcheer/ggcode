# v1.3.97

## Fixes

- **Autopilot goal check leaking into non-autopilot modes**: The `hasAutopilotGoal()` check in the agent loop (line 642) was missing a mode guard, unlike the other three autopilot heuristics (`shouldAutopilotContinue`, `shouldAutopilotAskUser`, `isAutopilotGoalComplete`). When a user switched from autopilot to another mode mid-session, the residual goal could trigger a goal-check follow-up prompt after compaction summaries, preventing the agent loop from stopping naturally. Fixed by adding `a.currentMode() == permission.AutopilotMode` to the condition.

## Features

- **`lanchat set_identity` action**: The `lanchat` tool now supports `action=set_identity` to change your own nick, role, and/or team from the LLM. Previously only available via the TUI `/nick` slash command. Unprovided fields are preserved.
- **System prompt overhaul**: Reworked `DefaultSystemPrompt` `## Collaboration routing` into a structured decision tree covering all 5 collaboration mechanisms (`spawn_agent`, `teammate_spawn`/`swarm_task_create`, `lanchat`, `a2a_remote`, `delegate`). Added `## Permission modes` descriptions (each mode now has a one-line explanation). Added proactive parallelism guidance. Eliminated ~200 tokens of duplicated antinoise rules between `FormatRemoteAgents` and `DefaultSystemPrompt`.
- **Simplified path relativization**: `util.RelativizePaths` and `tool.RelativizePath` are now pass-through functions. Absolute paths are shown as-is in all surfaces (TUI, IM, mobile). This removes fragile string replacement that could produce misleading relative paths.

## Documentation

- Updated `docs/guide/modes.md` with session-scoped persistence for both permission mode and sidebar visibility.
- Updated `docs/ARCHITECTURE.md` session store description with persisted meta fields.
- Consolidated 54 auto-memory files into 21 focused files.
