# ggcode v1.1.20

## Highlights

- Fixed the TUI activity list so parallel tool calls now show completion and output on the correct tool item.
- Added a program-level Bubble Tea harness so TUI tests can drive a live event loop instead of only calling `Model.Update(...)`.

## Fixes and improvements

### TUI tool activity rendering

- Tool activity completion now matches the correct started item by stable tool call ID.
- Parallel or batched tool calls no longer leave earlier items stuck in a “running/no output” state when results arrive out of order.
- Added regression coverage for out-of-order completion of concurrent tool activity items.

### TUI test infrastructure

- Added a live Bubble Tea program harness for tests.
- Tests can now start a real `tea.Program`, send interaction messages into the running loop, wait for barrier/snapshot points, and assert asynchronous UI state.
- Added coverage for live key-driven mode switching and async clipboard image attachment flows.

## Upgrade notes

- No migration is required.
- This release improves TUI correctness and test coverage only; no config changes are needed.
