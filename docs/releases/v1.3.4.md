# ggcode v1.3.4

## Highlights

- **IM adapter QR code support** — Desktop app now shows QR code buttons on adapter cards and supports scan-to-onboard for WeChat and other platforms.

## Desktop IM improvements

- WeChat adapter: "Scan QR Code" button on adapter card triggers iLink API QR auth flow (request → display → poll → save bot_token → start adapter)
- WeChat adapter: Add Adapter flow skips form fields, opens QR scan window directly
- WeChat adapter: re-scanning updates bot_token on existing adapter in place
- QQ, Telegram, Discord, Slack, Signal, WeCom adapters: "Show QR Code" button displays contact URI as scannable QR image with copy-link support
- WhatsApp adapter: marked as QR-auth platform
- `platformMeta` struct extended with `QRAuth` and `ShowContactQR` fields
- Generic `showContactQRWindow()` renders QR from `AdapterState.ContactURI` using go-qrcode

## Other fixes

- Accidental 93MB desktop binary removed from git, added to `.gitignore`
- Previous python-based code insertions silently failed due to tab indentation mismatch — all QR logic now verified present
