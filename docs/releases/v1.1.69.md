# ggcode v1.1.69

## Highlights

Fixes the critical gaps identified in v1.1.68: suggest mode Enter/Esc now works independently of autocomplete, pipe auto-run uses fullPrompt with stdin, strict write guard is global, RunService covers manual runs via CTA generation, and dirty workspace checkpoint is fail-fast by default.

## Changes

### Suggest mode: independent modal state
- `pendingAutoRun` Enter handler moved BEFORE autocomplete check in `model_update.go`.
- `pendingAutoRun` Esc handler moved BEFORE autocomplete check.
- Stale state cleanup: any new text submission clears `pendingAutoRun`.
- No longer requires autocomplete to be active for Enter/Esc to trigger.

### Pipe auto-run: uses fullPrompt with stdin
- `checkPipeAutoRun()` and `runPipeHarness()` now receive `fullPrompt` (which includes stdin data) instead of raw `prompt`.
- Image inputs skip auto-run entirely (harness uses text-only goals).

### Strict write guard: global enforcement
- `applyStrictWriteGuard()` called in `checkAutoRun()` when mode is "strict", BEFORE routing decision.
- Applies regardless of whether the input is routed to harness or goes to the normal agent.
- `run_command` added to the deny list (can be used to bypass file write restrictions).

### RunService: unified CTA for manual runs
- `GenerateCTA()` exported for use by TUI handler.
- `harnessRunResultMsg` handler in `model_update.go` generates CTA from summary when `CTAMessage` is not provided.
- Manual `/harness run` and `/harness rerun` now show post-run CTA (review, promote, rerun).

### Dirty workspace: fail-fast policy
- `RunService.Run()` injects a fail-fast `ConfirmDirtyWorkspace` when none is provided.
- Auto-run and pipe mode will NOT silently auto-commit dirty workspace.
- Error message instructs user to commit/stash changes first or use interactive `/harness run`.

### Documentation accuracy
- Design doc status changed to "In Progress" (not "Implemented").
- v1.1.68 release note overclaim corrected.
