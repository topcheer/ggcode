# ggcode v1.1.74

## Highlights

WebUI broadcast fix for review/promote results + design doc version sync.

## Changes

### WebUI event mirroring for review/promote
- `harnessReviewResultMsg` handler now broadcasts to WebUI subscribers via `webuiBridge.BroadcastEvent()`.
- `harnessPromoteResultMsg` handler now broadcasts to WebUI subscribers.
- Previously only `harnessRunResultMsg` (initial run result) was broadcast. Now all three stages of the lifecycle reach WebUI: run → review → promote.

### Design doc accuracy
- Status line updated from v1.1.72 → v1.1.73 to match actual code.
