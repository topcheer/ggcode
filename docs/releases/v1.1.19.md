# ggcode v1.1.19

## Highlights

- Added repo-managed GitHub Release notes so each release can ship with curated notes from the repository.
- Made debug logging safer for long sessions with asynchronous writes, 50MB rotation, and automatic cleanup on exit.
- Persisted the selected permission mode so ggcode reopens in the same mode you last used.

## Fixes and improvements

### Release workflow

- Added `docs/releases/<tag>.md` support to the release workflow.
- Added a fallback renderer so releases still get structured notes when a tag-specific file is missing.
- Added release-notes documentation and a reusable template for future versions.
- Fixed the debug logger tests so the release validation path passes cleanly in CI.

### Runtime and TUI

- Debug logs now write asynchronously instead of blocking request and UI paths on file I/O.
- Debug logs rotate at 50MB and keep at most three files during a running session.
- ggcode now removes its debug log files on normal exit so transient debug traces do not accumulate.
- Mode changes from both `Shift+Tab` and `/mode ...` now persist to `default_mode`, so the next launch restores the previous mode.

## Upgrade notes

- No manual migration is required.
- If you relied on `/tmp/ggcode-debug.log` after the app exits, note that normal shutdown now deletes debug logs automatically.
- The last selected mode is now written back to `default_mode` in your config file when you switch modes in the TUI.
