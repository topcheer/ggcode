# ggcode v1.3.24

## Highlights

- **Tunnel replay gap recovery with retry**: Mobile WebSocket now detects event ordinal gaps and automatically retries replay requests (3 attempts with backoff), then falls back to full history reconnect if all retries fail. Prevents the "stuck forever" issue when events are lost in transit.
- **RelayClient write-failure message preservation**: Failed WebSocket writes are queued in a pendingFront buffer and retried on reconnect, ensuring no messages are silently dropped.
- **Knight disabled by default**: Knight is now opt-in. Startup shows current status and how to toggle.
- **/knight on|off runtime toggle**: Slash commands that both persist config AND immediately start/stop Knight in daemon mode.
- **Structured task tool results**: Task tools (task_create/get/update/list/stop/output) now display compact summaries with expandable details across TUI, Desktop, and Mobile.

## Tunnel message reliability

- `ConnectionNotifier`: gap detection triggers watchdog timer with configurable retry (default: 3 retries at 1s/2s/4s backoff)
- After retry exhaustion, sends `resume_hello` requesting full history
- If full history also times out, triggers clean reconnect
- All reset paths (disconnect, resume_miss, snapshot_reset) properly clean replay recovery state
- `RelayClient.pendingFront`: write failures push current message to front of retry queue
- `writePump` drains pendingFront with priority over new sendCh messages on reconnect
- New test: `TestRelayClientPendingMessagesTakePriorityOverSendChannel`

## Knight default-off and toggle

- `DefaultKnightConfig().Enabled = false` (was `true`)
- `SetDefaults()` no longer forces `Enabled=true` when unset
- `ggcode.example.yaml`: `enabled: false` with hint comment
- `/knight on`: persists config + calls `knight.Enable()` in daemon mode (immediate start)
- `/knight off`: persists config + calls `knight.Disable()` in daemon mode (immediate stop)
- TUI mode: persists only, shows "restart to apply"
- Startup hint: system message showing knight status + toggle instructions
- Daemon mode: stderr message showing status
- `SaveKnightEnabled()` in config_save.go for atomic YAML persistence
- `Knight.Enable()`: sets `cfg.Enabled=true` then calls `Start()`
- `Knight.Disable()`: sets `cfg.Enabled=false` then calls `Stop()`

## Structured task tool results

- `ToolResultPresentation{Summary, Payload, PayloadMode}` shared semantic type
- `DescribeTaskToolResult()` generates human-readable summaries for all task_* tools
- TUI: `suppressToolResult` shows summary instead of raw JSON
- Desktop: collapsible summary + expanded payload sections via `taskToolResultSections()`
- Mobile: `_ToolResultCard` collapsed summary, tap to expand structured payload
- `ToolResultData`/`SubagentToolResultData`: +summary, +payload, +payload_mode protocol fields
- Broker caches toolArgs at PushToolCall, uses them at PushToolResult for structured output

## Cancel safety (subagent/swarm)

- `subagent.Cancel()` now cancels pending agents (not just running)
- `subagent.CancelAll()` covers pending + running agents
- `subagent.SetCancel()` returns false for terminal agents, preventing resurrection
- Swarm idle_runner checks `ctx.Err()` before every branch to avoid post-cancel work
- `swarm.CancelAll()` cancels all live teammates including idle ones
