# ggcode v1.1.84

## Highlights

**YouTube Live Stream** — Built-in live streaming to YouTube (and other RTMP targets) directly from the TUI. Renders terminal output as a real-time image stream with embedded DejaVu Mono font, CJK wide-character support, and FFmpeg H.264 encoding. New `/stream` slash command and configuration panel for managing stream targets, start/stop, and resolution.

- `internal/stream/` — New package (~3500 LOC): Manager, DirectRenderer, Encoder (software + hardware), Target, presets, font handling
- `internal/tui/stream_commands.go` — `/stream start|stop|status|config` commands
- `internal/tui/stream_panel.go` — Stream configuration panel with key/URL editing
- Orientation lock on first frame: landscape → 1920×1080, portrait → 1080×1920
- Auto-calculated fontSize based on terminal width, clamped 10–28pt
- Embedded DejaVu Sans Mono Bold + Regular fonts for consistent rendering

**Knight Governance & Self-Reflection** — Major Knight expansion with governance policies, semantic memory, skill promotion pipeline, A/B replay, and self-reflection capabilities.

- `internal/knight/governance.go` — Policy governance with scope arbiter
- `internal/knight/semantic_memory.go` — Persistent semantic memory for cross-session learning
- `internal/knight/self_reflection.go` — Periodic self-reflection on session outcomes
- `internal/knight/skill_promoter.go` — Automatic skill promotion from successful patterns
- `internal/knight/ab_replay.go` — A/B testing replay for policy evaluation
- `internal/knight/budget_buckets.go` — Token budget bucketing by scope

## Features & Improvements

### TUI & Slash Commands
- **`/restart [debug]`** — Restart now accepts an optional `debug` argument to restart with `GGCODE_DEBUG=1`. Tab-completion shows `[debug]` placeholder instead of immediately executing.
- **`/stream` and `/restart` bypass agent busy state** — These commands now execute immediately even when the agent is running, instead of being queued.
- **Ctrl+V silent on non-image clipboard** — Pasting when the clipboard contains text (not an image) is now a silent no-op instead of showing "请求失败" and interrupting the agent loop.

### Plan Mode
- **UI correctly reflects plan mode** — Entering plan mode now switches the input border to blue and shows "plan" badge. Root cause: `replModeSwitcher` was reading `model.program` from the REPL's model copy (always nil after `NewProgram` copies the model). Fixed by storing `*tea.Program` directly on the switcher.
- **Read-only tools bypass sandbox** — In plan mode, all read-only tools (read_file, grep, glob, etc.) can access files outside the sandbox. Plan mode is strictly read-only, so there is no security risk.
- **MCP tools allowed in plan mode** — All `mcp__*` tools are now permitted in plan mode (they have their own server-side access control).
- **Expanded read-only tool list** — `IsReadOnlyTool` now includes grep, LSP tools, git read tools, web_fetch, web_search, sleep, task_list, cron_list, and all MCP tools.

### read_file & edit_file Tools
- **Indent and encoding detection** — `read_file` now prepends a metadata hint line (e.g., `[indent: tab]` or `[indent: 2 spaces]`) so the LLM knows the exact whitespace characters. Indent unit is calculated via GCD of all observed widths.
- **edit_file match failure diagnostics** — When `old_text` is not found, the error message now includes actionable hints: tab vs space mismatch, CRLF vs LF, or partial match with whitespace differences.

## Bug Fixes

### Stream
- **Content not filling frame** — DirectRenderer now receives `termCols, termRows` directly instead of computing them from pixel dimensions, ensuring terminal content fills the encoder resolution.
- **fontSize auto-calc skipped** — Manager now always auto-calculates fontSize (ignoring config default 14).
- **fontSize crushed by row count** — Calculation uses width only (`encW / cols * 1.6`), letting excess rows clip off like a real terminal.

### Plan Mode
- **modeChangeMsg never sent** — `replModeSwitcher.SetMode()` used `s.model.program` which was always nil because Bubble Tea's `NewProgram` copies the model. Fixed by storing `*tea.Program` on the switcher directly.
- **MCP tools blocked** — `isReadOnlyFileTool` only listed 4 tools; now delegates to `IsReadOnlyTool` which includes all read-only tools + MCP tools.

### Clipboard
- **Non-image paste crashes agent loop** — `handleClipboardPaste` returned `errMsg` for `ErrClipboardImageUnavailable`, which set `m.loading = false` and `m.runFailed = true`. Now silently returns `nil`.
