## Fixes and improvements

### Chat rendering engine

- Replaced the legacy `viewport`-based conversation rendering with a `chatList` virtual-scrolling list backed by the new `internal/chat/` package.
- `AssistantItem` caches rendered markdown output during streaming; only re-renders on text change.
- `ToolBodyBehavior` provides a single source of truth for how each tool's body is rendered: `BodySuppress` (header only), `BodyFormatJSON` (structured KV), `BodyMarkdown` (markdown rendering), `BodyDefault` (truncated raw text).
- Smart truncation: body content uses tail-truncation, path-heavy content uses head-truncation.
- Consecutive system messages are merged to reduce visual noise.
- `AgentToolItem` renders nested tool results as a lipgloss tree with indentation.

### Tool display polish

- All built-in tools have i18n display labels registered in `internal/im/tool_labels.go`.
- Tool parameter extraction handles `map[string]string` â†’ `map[string]any` conversion.
- `task_create` and `task_get` bodies render as formatted JSON; `task_update` and `task_stop` bodies are suppressed.
- `save_memory` body suppressed (no useful info); `spawn_agent` shows task description in header.
- `todo_write` and `list_commands` have i18n labels.

### Plan mode

- `exit_plan_mode` now guides the LLM to use `task_create` for breaking the plan into structured tasks.
- `task_create` gained `addBlocks` / `addBlockedBy` fields for setting dependencies at creation time.
- `enter_plan_mode` description simplified â€” tasks are created after exit, not during.

### IM integration

- `im.output_mode` config: `verbose` (default), `quiet`, `summary`.
- IM panel shows current output mode with description; switch with `v`/`q`/`s` keys.
- Daemon follow mode hotkeys: `v`/`q`/`s` for output mode, `M`/`U` for mute/unmute all.
- `formatToolSummary()` renders aggregated tool counts with status icons for quiet/summary modes.
- `MuteAll()` / `UnmuteAll()` available from both TUI IM panel (`M`/`U` keys) and daemon follow.
- IM panel hints updated to include output mode and mute/unmute all keys.

### Daemon mode

- `--resume-picker` flag added (matching root command behavior).
- Follow display prefix `â—Ź ` and text now on the same line (was split across two lines).
- Glamour leading/trailing whitespace trimmed to eliminate blank lines.
- Startup keys hint updated to list all available shortcuts including `r` restart.

### Configuration

- `/config set apikey` allows setting API keys from the TUI.
- `/config add-endpoint` and `remove-endpoint` for endpoint management.
- Plaintext secret auto-migration extended to IM adapter configs and MCP server env vars.
- `ggcode.example.yaml` updated with `im.output_mode` documentation.

### Worktree support

- Git tools sync `WorkingDir` via reflection before execution, fixing path issues in worktrees.
- Sub-agents, swarm teammates, and Knight inherit the parent's `workingDir`.
- Worktree tool improvements: tests, IM formatting, TUI labels.

### Self-restart

- TUI restart uses `syscall.Exec` instead of background script for cleaner process replacement.
- `onRestart` callback properly re-registered after config changes.
- Input history restored from session on resume.

### Reliability

- Cron trigger sends system messages instead of user prompts; sleep limited to 30 min.
- Stalled commands auto-backgrounded after 120s.
- Sandbox defers to permission layer in non-plan modes (Ask-level commands auto-allowed in bypass/autopilot).
- Agent loop executes all tool calls before injecting user interruptions.
- Composer auto-expands height on word wrap; paste allowed while agent is running.

### Architecture

- New `internal/chat/` package: virtual-scrolling list with item interface, caching, and styled rendering.
- `internal/daemon/` package: follow display, background forking, session picker, i18n labels.
- `internal/knight/` package: autonomous background agent.
- `internal/a2a/` package: Agent-to-Agent protocol server, client, registry.
- Harness migrated from SQLite to JSON snapshots for event storage.
- Categorized debug logging with `GGCODE_DEBUG` env var and per-category control.
