# ggcode v1.1.55

## Highlights

- **Knight management UI in WebUI** — full skill lifecycle management directly from the browser configuration panel.

## Knight WebUI Page

New **Knight** page in the WebUI sidebar with:

- **Status dashboard** — running state, budget usage bar (with K/M formatting), active/staging/candidate counts
- **Active skills** — list with scope badge, usage count, freeze/unfreeze toggle, rollback button, content viewer
- **Staging skills** — yellow-bordered cards awaiting approval with Approve (✓) and Reject (✗) buttons, content viewer
- **Candidate queue** — sortable table showing name, category, score, evidence count, description, with delete action
- **Quick actions** — "Analyze Now" and "Validate Now" buttons to trigger Knight tasks on demand
- **Skill content modal** — view raw markdown of any skill file

## API Endpoints (backend, from v1.1.54)

- `GET /api/knight` — full status with budget, skills, and queue
- `GET /api/knight/skills` — active + staging only
- `POST /api/knight/action` — 9 lifecycle actions (promote, reject, freeze, unfreeze, rollback, record_effectiveness, analyze, validate, delete_queue)
- `GET /api/knight/skill-content?name=X&staging=Y` — raw skill file content
