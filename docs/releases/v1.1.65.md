# ggcode v1.1.65

## Highlights

Implements the Harness Auto-Run foundation: deterministic router, auto-init, configuration, and bundled skills. This is the first phase of the harness-auto-run-design.md roadmap.

## New Features

### Harness Auto-Run Configuration
- **config.HarnessConfig**: New configuration section `harness:` with:
  - `auto_run`: Controls automatic routing mode (`"off"`, `"suggest"`, `"on"`, `"strict"`)
  - `auto_init`: When true, automatically creates minimal harness.yaml on first detected code-change task
- Default: `auto_run: "off"`, `auto_init: false` (zero-impact upgrade)

### Deterministic Router (`internal/harness/router.go`)
- Three-layer classification pipeline:
  1. **Exclusion**: Slash commands, shell directives, explicit "don't change", too-short inputs
  2. **Structural features**: File paths, action verbs, code blocks, task goals — scored to determine confidence
  3. **Conservative default**: Ambiguous inputs go to normal agent
- `DecideRoute()`: Simple API for routing decisions
- `DecideRouteWithFeatures()`: Rich API with selection context and recent files
- Full Chinese language support for action verbs and exclusion patterns
- Zero LLM dependency — all decisions are deterministic

### Auto-Init (`internal/harness/auto_init.go`)
- `AutoInit(dir)`: Creates minimal harness.yaml + state directories without AGENTS.md, runbooks, or scaffold commit
- Idempotent: refuses if harness.yaml already exists
- Designed for automatic project onboarding when `auto_init: true`

### Auto-Run Integration Hook (`internal/harness/auto_run.go`)
- `ShouldAutoRun(cfg, input, ctx)`: Single entry point that:
  1. Checks config mode
  2. Discovers or auto-initializes harness project
  3. Runs deterministic router
  4. Returns `RouteDecision` + resolved `Project` + human-readable message
- Downgrades `RouteHarness` to `RouteSuggest` when no project is available (graceful degradation)

### Harness Bundled Skills
- `harness-run`: Execute a coding task in isolated harness mode
- `harness-review`: Review and approve/reject harness task results
- `harness-promote`: Promote approved changes to main branch
- `harness-diagnose`: Diagnose harness task failures and configuration issues
- All four are user-invocable via slash commands
