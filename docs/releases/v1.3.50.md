# v1.3.50

## Mobile

### Relay Sync Progress Banner

A new banner appears in the chat screen during relay connection synchronization, showing real-time progress through three phases:

- **Waiting**: Restoring local session, waiting for authoritative relay updates
- **Replaying**: Catching up with remaining replayed updates (shows count)
- **Snapshot**: Refreshing session state after a relay-initiated rebuild

If sync takes longer than 30 seconds, the banner switches to a stalled warning state with an amber border.

### Connection Generation Guard

All async callbacks now use a connection generation counter to prevent stale service callbacks from corrupting the current connection state. This eliminates race conditions when:
- Rapidly switching between different rooms/URLs
- Old connection's error propagates after reconnecting to a new room
- Provider is disposed while async operations are in-flight

### Sub-agent Cleanup

Sub-agent tab cleanup now uses generation-aware timers instead of `Future.delayed`, ensuring timers are properly cancelled on reconnect or dispose.
