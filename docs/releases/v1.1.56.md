# ggcode v1.1.56

## Highlights

- **Knight production hardening** — fixes from 12-hour overnight analysis revealing critical issues in skill generation, budget management, and data quality.

## Fixes

- **Max iterations 15 → 50**: LLM needs more turns to read session files and generate quality skills
- **Daily token budget 5M → 50M**: the previous 5M budget was exhausted in ~12 hours
- **Candidate retry backoff**: candidates that fail LLM generation 3 consecutive times are now abandoned instead of retrying every hour forever (new `GenFailCount` field)
- **Skill output extraction**: LLM reasoning/thinking preamble is now stripped before the YAML `---` frontmatter, so staging files no longer contain "Now let me look at..." text
- **Candidate name validation**: garbage names like `--`, `call`, `git`, `correction-123`, `llm--200` are now rejected at detection time via `isValidCandidateName()` (requires ≥8 chars, ≥50% alphanumeric, no pure-numeric suffix)
