# ggcode v1.3.10

## TUI Tool Call Display Improvements

- **Description-first labels**: Tool calls now show the LLM-provided description as the primary label (e.g. "Build project (Bash)") instead of generic tool names. Falls back to prettified tool name when no description is available
- **Friendly tool names**: `run_command` displays as "Bash", `start_command` as "Background Bash" in parentheses

## Mobile App Integration

- **Download links in UI**: TUI QR overlay and GUI Share dialog now show iOS TestFlight and Android testing links below the QR code
- **Mobile beta links in docs**: All READMEs (en, zh-CN, npm, PyPI) now include mobile app download section with clear instructions (Android requires joining Google Group first)

## iOS Deployment

- **Smart submit logic**: Fastfile `submit` lane detects existing READY_FOR_REVIEW submissions and only promotes to External Testing instead of failing
- **External Testing via API**: `promote_external` lane uses Spaceship API to add builds directly to the "ext" beta group
- **CI fix**: `go vet` / `go test` in release workflow excludes desktop packages that require CGO
