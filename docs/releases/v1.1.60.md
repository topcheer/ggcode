# ggcode v1.1.60

## Highlights

Addresses all remaining issues identified in the Copilot code review: checkpoint handler lock contention, session index inconsistency, precompact goroutine lifecycle, and OpenAI-compatible tool-call flush edge cases.

## Fixes

### Checkpoint handler lock-held I/O
- **repl.go**: Checkpoint handler now mutates Session object under `sessionMutex`, then unlocks before persisting to disk. New `AppendCheckpointToDisk()` method writes JSONL + updates index without modifying the Session object, eliminating TUI thread blocking during checkpoint saves.

### Session index consistency
- **store.go**: Added `indexDirty` flag on `JSONLStore`. When `updateIndex` fails (e.g., disk full), the flag is set. On next `loadIndex`, if dirty, a repair scan reconciles JSONL files with the index. Also handles corrupt index JSON by triggering repair.

### Precompact goroutine lifecycle
- **agent.go**: Agent now has a `shutdownCtx` cancelled by `Close()`. `StartPreCompact` derives its background context from `shutdownCtx` instead of `context.Background()`, so precompact goroutines are cancelled on agent shutdown, not just on timeout.
- Added `Agent.Close()` method that cancels shutdown context + precompact.

### OpenAI-compatible tool-call flush
- **openai.go**: Flush now triggers on both clean EOF and `context.Canceled` (previously only EOF). Added validation: tool calls must have non-empty `Name` + `ID`, and `Arguments` must be valid JSON. Incomplete tool calls are skipped instead of emitted with broken data.
