## v1.3.27

### Cron Scheduler Integration for Desktop

Desktop now supports the full cron scheduling system (previously TUI/daemon only).

- **Cron tools registered**: `cron_create`, `cron_delete`, `cron_list` are available in desktop agent sessions.
- **Hidden prompt delivery**: Cron-triggered prompts are injected as hidden text — they drive the agent but don't appear in the chat history or session JSONL, matching TUI behavior.
- **Busy-queue support**: If agent is working when a cron fires, the prompt is queued and delivered when the current run finishes, without creating a visible user message.
- **Interruption handler**: Agent loop checks for queued messages (visible or hidden) between turns, enabling cron prompts to interleave with user work.

### Pending Message Queue Overhaul (Desktop + TUI)

- **Desktop**: `pendingMsg string` → `pendingMsgs []pendingMessage{Text, Hidden}` — FIFO queue supporting multiple queued messages with hidden/visible distinction.
- **TUI**: `consumeDetailed()` changed from "drain all and merge" to "consume one hidden or batch adjacent visible" — hidden entries (cron triggers) are consumed individually, visible entries are merged as before.
- **`drainPendingInterrupt()`**: Hidden entries skip `appendUserMessage` in both TUI and desktop, preventing cron prompts from polluting chat history.

### Structured Tool Result Display (Cron Tools)

- **`DescribeToolResult`**: Generalized from `DescribeTaskToolResult` — now covers task_* tools AND cron_create/cron_delete/cron_list.
- **Cron result views**: `cron_create` shows schedule details (ID, cron expression, mode, next fire, prompt). `cron_delete` shows deleted job ID. `cron_list` shows job count + full list.
- **Desktop UI**: Cron tools removed from `tcSuppress` list — they now render with structured collapsible sections instead of being hidden entirely.
- **All platforms**: Broker, TUI snapshot, desktop snapshot, and WebUI all use the unified `DescribeToolResult`.

### Test Coverage

New tests for cron result parsing, broker structured cron events, hidden pending queue behavior, and desktop pending message drain.
