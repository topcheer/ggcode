# ggcode v1.1.43

## Highlights

- **Composer input space fix** — trailing and middle spaces in the input field are now rendered correctly. Previously, typing a space after existing content appeared invisible until the next non-space character was typed.
- **Narrow layout redesign** — the two header boxes (logo card + meta card) are removed in narrow windows. The logo is now rendered borderless inside the conversation area and scrolls with the content, giving more vertical space for messages.
- **Parallel IM sending** — outbound IM messages are now sent concurrently with a 10s timeout per channel, instead of sequentially.
- **A2A protocol** — multi-instance ggcode agents can discover and collaborate with each other via the Agent-to-Agent protocol (MCP bridge, auto-discovery, multi-turn support).

## Fixes and improvements

### Composer input rendering

- Fixed trailing/middle spaces not rendering in the composer input field. Root cause: `wordwrap.String()` trims trailing spaces, breaking cursor positioning. Fix: only apply wordwrap when a line actually overflows the available width.
- Fixed cursor appearing stuck on the previous character when typing a space at end of line. Root cause: wrapping `textinput.View()` output in `lipgloss.Bold(true).Render()` mangled the virtual cursor's ANSI sequences. Bold is now applied directly to the textinput's own styles.
- Fixed cursor blink messages not reaching the textinput in the catchall handler, causing the cursor to be invisible on some platforms.

### TUI layout

- Narrow window mode: removed the two header boxes (logo card and meta card). The logo now appears borderless at the top of the conversation viewport and scrolls with content.
- Sidebar uses content-based height calculation instead of fixed height, improving vertical space usage.

### IM integration

- Parallel IM sending with 10s timeout, automatic retry, and non-blocking buffer for all adapters.
- Dummy IM adapter for automated testing and evaluation.
- Comprehensive i18n tests for outbound text formatting.

### Agent-to-Agent (A2A) protocol

- A2A protocol server, client, and registry for multi-instance ggcode collaboration.
- Runtime auto-discovery via `a2a_remote` tool with periodic cache refresh.
- Multi-turn support with `input-required`, `tasks/resubscribe`, and full spec coverage.
- MCP bridge for exposing A2A capabilities to LLM tools.

### Reliability

- Fixed `pendingQueue` pointer sharing across Model value copies causing empty interrupts.
- Fixed streaming LLM responses being interrupted by queued user messages — now injected between iterations.
- Fixed lazy-init race on `pendingMu`/`sessionMu` in `NewModel`.
- Paste routing fixed for all IM panel input fields (QQ/TG/Discord/Slack/Feishu/DingTalk).
