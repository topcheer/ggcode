# v1.3.41

## Features

### Token Usage Propagation

- **Sub-agent usage tracking**: Token usage from sub-agents (spawn_agent) is now propagated back to the parent session. Previously, sub-agent LLM calls were invisible in the session usage display.
- **Swarm teammate usage**: Swarm teammate token consumption is also tracked and accumulated into the session total.
- **Skill tool usage**: Skill invocations that use LLM calls now report usage back to the session.
- **Summarize usage**: Context compaction (summarize) token costs are now included in session usage via the usage handler propagated to ContextManager.

### Cache Hit Percentage

- **New field**: `TokenUsage.PromptTokensTotal` captures the total prompt tokens from the API response.
- **Cache hit display**: Both TUI and Desktop sidebar now show a `cache hit` percentage in the session usage section. Calculated as `CacheRead / PromptTokensTotal * 100%`.
- **Fallback logic**: For providers that don't report `PromptTokensTotal`, uses `InputTokens + CacheRead + CacheWrite` as an estimate.
- **i18n**: `cache hit` / `缓存命中`.

## Fixes

### Desktop

- **Onboard endpoint selector**: Setup wizard now has a 4-field flow: Vendor → Endpoint → API Key → Model.
