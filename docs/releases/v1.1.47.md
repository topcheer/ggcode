# ggcode v1.1.47

## Highlights

- **A2A LAN discovery via mDNS** — instances on the same network automatically discover each other using mDNS (multicast DNS). Enable with `a2a.lan_discovery: true` and configure auth (`a2a.auth.api_key` or stronger). No manual endpoint configuration needed.
- **WebUI real-time A2A panel** — the A2A configuration page now auto-refreshes discovered instances every 10 seconds. New or disappeared instances update without manual page reload.
- **Windows support for A2A** — cross-compiled Windows ARM64 binary tested. Daemon restart (`r` key) works on Windows via `exec.Command` + `os.Exit` instead of `syscall.Exec`.
- **Terminal display integrity** — all A2A/mDNS diagnostics and third-party library output (hashicorp/mdns) no longer write to stderr. Prevents TUI corruption and daemon follow display interference.

## Fixes and improvements

### A2A / mDNS

- **mDNS registration** uses system `avahi-publish` on Linux for reliable service broadcasting, with pure-Go `hashicorp/mdns` fallback for macOS/Windows.
- **mDNS lookup** filters dead PIDs — instances discovered via mDNS are checked for process liveness before being merged into the local registry.
- **Stale registry file cleanup** — `Register()` now deletes old files from the same PID+workspace before writing a new one. `discoverLocal()` deduplicates by PID+workspace, keeping only the latest.
- **Process group kill for avahi-publish** — subprocess is placed in its own process group (`Setpgid: true`) so `kill(-pgid)` terminates it reliably when the parent dies.
- **Silenced hashicorp/mdns library logs** — `[INFO] mdns: Closing client {...}` messages no longer appear on terminal. Library logger set to `io.Discard`.
- **All A2A/mDNS diagnostics use `debug.Log()`** — no `fmt.Fprintf(os.Stderr, ...)` calls remain in `internal/a2a/`.

### WebUI

- **A2A auto-refresh** — discovered instances poll every 10 seconds when the A2A tab is visible.
- **A2A status JS syntax error fixed** — nested ternary in A2A status template was missing `: ''` else branch, causing `Uncaught SyntaxError`.

### Platform support

- **Windows cross-compile** — platform-specific code (`Setpgid`, `syscall.Kill`) extracted to `mdns_proc_unix.go` / `mdns_proc_windows.go` with build tags.
- **Windows daemon restart** — uses `exec.Command` to spawn new process + `os.Exit(0)` instead of `syscall.Exec` (Unix-only).

### Testing

- A2A cluster tests pass with same-PID instances (different workspaces) — dedup key is `PID+workspace`, not just PID.
- All existing tests pass on macOS, Linux, and Windows ARM64.

## Upgrade notes

- **Enable LAN discovery** — set `a2a.lan_discovery: true` and `a2a.auth.api_key` in your config. The bind host auto-switches to `0.0.0.0` when auth is configured.
- **No breaking changes** — all existing A2A configs continue to work.
