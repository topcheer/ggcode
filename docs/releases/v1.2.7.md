# ggcode v1.2.7

## Highlights

- **Shell mode output fix**: Fixed critical off-by-one bug in command job polling that caused every poll cycle to return already-seen lines, producing duplicate output. Shell command output now renders incrementally in a single message bubble instead of one bubble per poll.
- **`/compact` loading feedback**: The `/compact` slash command now shows a spinner with "Compressing context..." status during the potentially long LLM summarization call, and reports token reduction stats on completion.
- **Context overflow detection improved**: Consolidated 25+ vendor-specific "prompt too long" error patterns into a single `IsContextOverflowError()` function. Context overflow errors are now classified as non-retryable, preventing wasted retries on oversized prompts. Reactive compaction sends progress notifications.
- **Config file compaction**: Config files are now automatically stripped of default values on save, keeping `ggcode.yaml` minimal. Instance-level configs store only the diff from global defaults.
- **Test isolation**: All config tests now use `withTestHome(t)` to redirect `~/.ggcode/` writes to temp directories, preventing test pollution of real user config.

## Bug fixes

- **Critical**: `selectCommandLines` off-by-one caused shell mode to duplicate output on every poll cycle — `sinceLine` was inclusive instead of exclusive
- **Fix**: Shell command prefix changed from `❯ $ cmd` to `$ cmd` — the double prefix was visually confusing
- **Fix**: `/compact` returned `streamMsg` which didn't reset loading state; now uses dedicated `compactResultMsg` with proper cleanup
- **Fix**: `status.compacting` text shortened from 57 chars to fit status bar without truncation
- **Fix**: Provider context overflow errors (OpenAI, Anthropic, Gemini, Groq, Mistral, etc.) no longer trigger wasteful retries
- **Fix**: Reactive compaction and auto-compaction now emit `StreamEventSystem` progress notifications

## Internal

- `SystemItem.SetText()` / `AppendText()` — support in-place update and append for system messages
- `UserItem.SetPrefix()` — allow custom prefix for user messages (used by shell mode)
- `chatUpdateSystemText()` / `chatAppendSystemText()` — TUI helpers for modifying existing system messages
- `IsContextOverflowError()` in `internal/provider/retry.go` — 25+ error patterns from 12+ vendors
- `stripDefaultsFromYAML()` / `compactArraysInYAML()` — config file minification on save
- `marshalInstanceDelta()` — instance configs now store only diff fields (403 → minimal)
- `recompactConfigFile()` — re-strip defaults after secret migration
- 6 new tests for config `withTestHome` isolation across `api_keys_test.go`, `config_test.go`, `coverage_test.go`, `im_config_test.go`, `instance_test.go`, `onboard_test.go`
- 3 new tests for shell mode output deduplication (`shell_dup_test.go`)
- 1 new test for command job polling correctness (`command_jobs_poll_test.go`)
- 18 new vendor error patterns in `TestIsPromptTooLongError`
