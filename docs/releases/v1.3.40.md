# v1.3.40

## Features

### Token Usage Tracking

- **Session usage sidebar**: Both TUI and Desktop now display a session token usage breakdown in the sidebar: total, input, output, cache read, and cache write tokens.
- **OpenAI cache tokens**: Parse `PromptTokensDetails.CachedTokens` from OpenAI responses. Streaming requests now include `StreamOptions.IncludeUsage=true` to get accurate final usage.
- **Anthropic cache tokens**: Parse `CacheReadInputTokens` and `CacheCreationInputTokens` from Anthropic responses.
- **Persistent usage**: Token usage is persisted per-session via incremental `AppendMetaToDisk` (no full rewrite). Usage survives session restore and compaction.

### Desktop Onboard Improvements

- **Endpoint selector**: Setup wizard now has a 4-field flow: Vendor → Endpoint → API Key → Model. Users with multi-endpoint vendors can pick the right one before entering credentials.
- **Localized error messages**: Onboard validation errors now use i18n translations instead of hardcoded English strings.
- **Larger panel**: Minimum onboard panel size increased to 720×420 for better form readability.
- **Language switch**: Changing language in settings now re-renders the onboard/welcome screen immediately.

## Fixes

- **Cron tunnel events**: Cron prompts no longer emit `system_message` to mobile — only `user_message` (kind=cron).
- **Tool label metadata**: `BaseToolItem` preserves original tool name and raw JSON args for tunnel history, fixing garbled localized tool descriptions on mobile.
- **Ask User delivery**: Mobile responses to `ask_user` are now delivered synchronously, fixing silently dropped answers.
- **Broker enqueue**: Unified through `enqueueWithBytes` under single lock, eliminating event ID ordering race.
