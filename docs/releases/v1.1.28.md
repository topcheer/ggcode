# ggcode v1.1.28

## Highlights

- Ships the validated **mainstream-language LSP hardening batch** across Java, C#, Python, and
  TypeScript/JavaScript, and promotes the website-backed **latest downloads** path into the updater as
  the final fallback after GitHub and the built-in proxy.

## Fixes and improvements

### LSP reliability and compatibility

- Added stronger multi-language local-LSP support for:
  - Java / jdtls
  - C# / csharp-ls
  - Python / pyright
  - TypeScript / JavaScript
- Fixed the C# `.slnx` compatibility gap by generating a compatible `.sln` for `csharp-ls` when a
  workspace only ships `.slnx`.
- Fixed Pyright workspace initialization so semantic requests no longer stall when the server expects
  post-`initialized` workspace notifications.
- Hardened `lsp_hover`, `lsp_definition`, and `lsp_references` so they retry nearby identifier starts
  when the initial cursor position lands near, but not exactly on, the target symbol.
- Added jdtls compatibility for:
  - pull-diagnostics fallback
  - string JSON-RPC request ids
  - `codeAction` requests that require empty diagnostics to be sent as `[]`

### Update and release distribution

- `/update` now prefers:
  1. GitHub Releases
  2. the built-in proxy fallback
  3. `https://ggcode.dev/downloads/latest` as the final fallback
- Added deployment-branch publishing so validated release artifacts are mirrored into
  `copilot/site-release-update/docs/site/downloads/latest/`.
- The website now exposes a latest-downloads page from the deployment branch, so the official site can
  serve the newest native packages directly.

## Upgrade notes

- No migration is required.
- If direct GitHub release downloads are unreliable in your environment, `/update` now has more
  fallbacks before it gives up.
