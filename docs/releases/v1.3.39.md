# v1.3.39

## Fixes

### Tunnel / Cron

- **Cron tunnel events fixed**: Cron-triggered prompts now emit `user_message` (kind=cron) instead of `system_message` to mobile. Previously, cron events sent both a system message and a user message through the tunnel, causing duplicate or incorrect display on mobile.

### Desktop

- **Onboarding endpoint selector**: The setup wizard now includes an Endpoint dropdown between Vendor and API Key. Users with multi-endpoint vendors (e.g. regional endpoints) can pick the right one before entering credentials. The model list loads only after both vendor and endpoint are selected.

### Commits (since v1.3.38)

```
fix(tunnel,cron): cron events use user_message not system_message, add endpoint selector to desktop onboard
```
