# ggcode v1.1.53

## Highlights

- **Knight skill detection completely redesigned** — replaced useless tool-name-sequence matching with semantic-level detection: user corrections and failure→fix chains. Skills now teach real behavioral rules (e.g. "always read code before concluding") instead of trivial patterns like "run-read".

## Fixes and improvements

### Knight background agent

- Replaced tool-call-counting heuristics (`detectToolPatterns`, `detectRepeatedPatterns`, `toolPattern`) with two semantic detectors:
  - **User corrections** — detects when the user says "不要/错了/wrong/instead" and captures what the AI did wrong + what the user said instead
  - **Failure→fix chains** — detects tool error followed by successful correction, captures the actual error and fix method
- Session analysis now **filters by workspace** — only analyzes sessions from the current project
- Session analysis now has **priority sorting** — sessions with corrections (+5) or errors (+3) are analyzed first
- Session **dedup with re-analysis** — `analyzedSessions map[string]time.Time` skips unchanged sessions but re-analyzes active sessions that received new messages
- **Scope decided by LLM** — removed keyword-based `inferCorrectionScope`; the LLM now decides `project` vs `global` based on evidence content
- **Skill semantic validation** — rejects trivial skills ("run commands sequentially") and warns about skills lacking concrete details
- **Staging auto-rejection** — skills that fail validation 3 consecutive times are auto-rejected instead of blocking forever
- **Correction candidates stage immediately** — a single user correction is worth staging; failure-fix patterns still require 2+ occurrences

### Daemon mode

- Fixed terminal corruption in follow display — Knight task output and shutdown message now use `\r\n` instead of `\n` for raw terminal mode compatibility

### Configuration

- Fixed `knight.enabled` default — omitting the `knight:` section no longer disables Knight; `enabled` defaults to `true` when not explicitly set

## Upgrade notes

- Existing Knight skill data (candidate queue, staging, snapshots) from the old detection system should be cleared on first run. Old candidates like `run-read`, `edit-grep` will not be generated by the new system.
- No config changes required. The `knight:` section works as before, with `enabled` now defaulting to `true`.
