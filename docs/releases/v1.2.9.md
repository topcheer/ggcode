# ggcode v1.2.9

## Highlights

- **Token estimation fix for Anthropic/Gemini**: Both providers severely undercounted tokens by only counting `Text` fields and dividing by 4. Now they use the same `estimateTokensForMessages()` as OpenAI, counting Text + tool_result Output + tool_use Input with a /3 divisor. Typical agent sessions see 2-3x higher (more accurate) token counts.
- **Context manager version counter**: Replaced 3 `reflect.DeepEqual` calls (O(n) deep comparison on potentially MB-scale message arrays) with an O(1) integer version counter. Also fixed a pre-existing bug where `CompactSnapshot` used shallow copy for Content blocks, meaning snapshot and live data shared memory.
- **IO size limits**: 67 unprotected `io.ReadAll` calls across 26 files now use `util.ReadAll(r, limit)` with 5 scenario-specific limits (1MB for auth, 50MB for API, 100MB for MCP, 10MB for WebSocket/general). Prevents OOM from malicious or broken HTTP responses.
- **Stream retry exhaustion fix**: All three streaming providers (OpenAI, Anthropic, Gemini) silently closed the stream when all retry attempts were exhausted. Now sends a `StreamEventError` so users see a clear error message.

## Bug fixes

- **Fix**: IM adapter binding dead loop — disabled adapters with no channel bound could not be enabled ("no channel bound") and could not be bound ("is disabled"). All 13 adapter panels now auto-enable when user explicitly binds from panel.
- **Fix**: `isRetryableForContext()` in `provider/retry.go` treated all `ctx.Err() != nil` as non-retryable, preventing HTTP client timeouts from being retried. Now only `context.Canceled` and caller-context `context.DeadlineExceeded` are non-retryable.
- **Fix**: Anthropic/Gemini `CountTokens` only counted `Text` chars / 4, ignoring `Output` (tool_result) and `Input` (tool_use) — the largest token consumers in agent sessions. Now uses same estimation as OpenAI (Text + Output + Input) / 3.
- **Fix**: Stream retry exhaustion (OpenAI, Anthropic, Gemini) — all retry attempts exhausted without sending an error event, causing silent failure. Now reports `StreamEventError`.
- **Fix**: `CompactSnapshot` shallow-copied `Content` slice headers, meaning snapshot and live data shared the same underlying array. Microcompact modifying `Output` after snapshot would silently corrupt the snapshot. Now deep-copies Content blocks.

## Performance

- **Context change detection**: `reflect.DeepEqual` on full message arrays replaced with O(1) version counter. Three hot paths affected: `Compact()`, `ApplyCompactResult()`, `CheckAndSummarize()`.
- **IO protection**: `util.ReadAll(r, limit)` with scenario-specific constants prevents unbounded memory allocation from 67 HTTP response reads.

## Structural refactoring (from v1.2.8)

12 large files (1500+ lines each) split into 39 focused sub-files. Zero logic changes. See [v1.2.8 release notes](v1.2.8.md) for details.

## Internal

- `internal/util/readlimit.go`: New `ReadAll(r, limit)` with 5 named constants
- `internal/context/manager.go`: `version int64` counter + `contentFingerprint()` helper
- `internal/provider/anthropic.go`, `gemini.go`: `CountTokens` delegates to `estimateTokensForMessages()`
- `internal/provider/openai.go`, `anthropic.go`, `gemini.go`: stream retry exhaustion sends `StreamEventError`
- `internal/tui/*_panel.go`: 14 adapter panels auto-enable on bind attempt
