# ggcode v1.1.57

## Highlights

- **Knight instance locking** — only one ggcode process per workspace can run Knight, preventing data corruption from concurrent skill/budget access.
- **WebUI overview data loading fix** — A2A and Impersonate sections no longer show "N/A" on page load.
- **TUI mode WebUI Knight panel** — Knight status, skills, and actions are now fully wired in TUI mode (previously only available in daemon mode).
- **mDNS noise suppression** — spurious `[ERR] mdns: Failed to unpack packet` messages no longer pollute the terminal.

## Fixes

- **Knight cross-process lock** (`internal/knight/lock.go`): uses `flock` on `<proj>/.ggcode/knight.lock` to ensure exclusive Knight access. Second instance sees a startup hint in both TUI chat area and daemon stderr: `🌙 knight: skipped — another instance (PID xxx) already running in this workspace`. Lock is crash-safe (kernel releases on process exit).
- **Knight startup hint in TUI** (`knightStartupHintMsg`): lock conflict is shown as a system message in the chat area at startup, not buried in debug logs.
- **Knight startup hint in daemon**: lock conflict prints to stderr alongside the follow display.
- **Knight `Status()` shows deferred state**: `/knight status` now reports `deferred — instance PID xxx holds lock` when another process has the lock.
- **WebUI overview race condition** (`internal/webui/dist/index.html`): `a2aData` and `impersonateData` API calls now complete before `renderOverview()`, fixing the "N/A" display.
- **WebUI Knight callbacks in TUI mode** (`cmd/ggcode/root.go`): `SetKnightStatusFn`, `SetKnightActionFn`, and `SetKnightSkillContentFn` are now wired for TUI mode, matching daemon mode's behavior. Knight page in WebUI now shows live data in both modes.
- **mDNS server log suppression** (`internal/a2a/mdns.go`): `mdns.Config.Logger` set to `silentLogger` for the broadcast server (was already silenced for queries), stopping `[ERR] mdns` noise from other devices' mDNS packets on port 5353.

## New files

- `internal/knight/lock.go` — `flock`-based cross-process mutual exclusion for Knight
- `internal/knight/lock_test.go` — tests for lock acquire/release/reacquire/PID reporting
