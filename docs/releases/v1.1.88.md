# ggcode v1.1.88

## Highlights

- **TUI/daemon tool output body styling overhaul** — every built-in tool now shows a concise, color-coded body line in both TUI chat view and daemon follow mode instead of raw text dumps

## Tool rendering changes

### File tools
- **read_file / write_file** — body shows green line count (e.g. `42 lines`)
- **edit_file** — body shows green `+N` and red `-N` diff summary

### Search tools
- **search_files / glob** — body shows green match count (e.g. `5 matches`)

### Git tools
- **git_status** — body shows colored summary of modified/added/deleted/untracked files
- **git_diff** — body shows green `+N` and red `-N` line summary
- **git_log** — body shows green 3-line commit display

### Command & agent tools
- **start_command** — header shows "Run in background" label
- **spawn_agent** — header shows "Starting subagent" label

### Hidden tools (no header/body shown)
- Background command lifecycle: `read_command_output`, `wait_command`, `stop_command`, `write_command_input`, `list_commands`
- Secondary git: `git_show`, `git_blame`, `git_branch_list`, `git_remote`, `git_stash_list`, `git_add`, `git_commit`, `git_stash`
- Agent management: `wait_agent`, `list_agents`
- All LSP tools: `lsp_*`
- Skill, save_memory, web_fetch, web_search — body suppressed

## Fixes and improvements

- Fix package build scripts for release pipeline
- File tool body rendering uses line count only for cleaner output

## Upgrade notes

- No breaking config changes
- Tool output styling is automatic in both TUI and daemon `--follow` mode
