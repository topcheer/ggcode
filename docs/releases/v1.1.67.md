# ggcode v1.1.67

## Highlights

Closes the harness auto-run loop: all entry points now actually execute harness tasks, strict mode enforces real write protection, suggest mode has Enter/Esc interaction, and the context prompt is skipped for auto-routed tasks.

## Changes

### Pipe auto-run: real execution
- `runPipeHarness()` now calls `harness.RunTaskWithOptions()` inline, waits for completion, and streams output to stdout.
- Returns proper exit codes (0 success, 1 failure).
- No longer just prints "auto-routing" and exits.

### Strict mode: real enforcement
- **Write guard**: `applyStrictWriteGuard()` now calls `policy.SetOverride("write_file", Deny)`, `SetOverride("edit_file", Deny)`, `SetOverride("multi_edit_file", Deny)` on the `ConfigPolicy`. The main agent is physically blocked from writing files in strict mode.
- **Worktree propagation**: `ShouldAutoRun()` now returns the overridden config in `AutoRunResult.Config` with `Run.WorktreeMode = "required"`. The caller passes this config to the actual run instead of re-reading from disk (which would lose the override).
- New test: `TestStrictWriteGuard_DeniesWriteTools` verifies actual Deny decisions.

### TUI auto-run: skip context prompt, direct execution
- `handleAutoRun()` no longer delegates to `/harness run` (which enters context prompt). Instead, `executeAutoHarnessRun()` directly calls `executeHarnessRun()` with empty contexts, matching the existing harness run flow.
- Skips all context selection UI for auto-routed tasks.

### Suggest mode: Enter/Esc interaction
- RouteSuggest now saves `pendingAutoRun` + `pendingAutoRunText` on the model.
- **Enter** (empty input): confirms → routes to harness via `handleAutoRun()`.
- **Esc**: dismisses → falls back to normal agent via `submitText()`.
- Messages: "→ Running in harness..." / "→ Running normally (harness skipped)."

### Daemon: informational logging
- `checkAutoRunSuggestion()` logs harness routing suggestions for daemon observability.
- Daemon mode does not auto-execute harness (by design — non-interactive agent decides via skill instructions).

## Test Coverage
- `strict_guard_test.go`: 2 tests verifying real Deny enforcement
- `auto_run_test.go`: +1 test for strict config override propagation
- All existing tests pass, `make verify-ci` green.
