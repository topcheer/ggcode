# ggcode v1.1.31

## Highlights

- Polishes the new Markdown preview path in the fullscreen file browser by toning down aggressive code
  coloring and prewarming the Markdown renderer so the first `.md` preview opens faster.

## Fixes and improvements

### Markdown preview rendering

- Treats fenced code blocks without an explicit language as plain `text` during terminal Markdown
  rendering, which keeps directory trees, dependency chains, and other ASCII / box-drawing blocks from
  being colorized like source code.
- Rebalanced the terminal Markdown palette so inline code and code-block tokens use calmer blue / gray
  accents instead of the previous heavy red emphasis.
- Preserves the non-TTY Markdown fallback path so tests, exports, and other non-interactive rendering
  flows keep their previous plain-output behavior.

### Markdown preview responsiveness

- Prewarms the width-aware Markdown renderer cache after the TUI receives its first terminal size,
  reducing the cold-start cost the first time a Markdown file is opened in the file browser preview.
- Keeps the existing per-preview render cache intact, so repeated scrolling / resizing of the same
  preview still avoids unnecessary rerender work.

## Upgrade notes

- No migration is required.
