# v1.3.5 — File Browser & Preview

## Desktop: Workspace File Browser & Preview

New **Files** tab in the sidebar with a full-featured file browsing and preview system.

### File Tree
- Lazy-loading file tree with directory/file icons
- Directories sorted before files, alphabetically within each group
- Skips `.git`, `node_modules`, `vendor`, `build`, etc.
- Search/filter input with recursive matching
- Root directory name displayed at top

### File Preview (VS Code style — covers chat area)
- **Code files** (.go, .py, .js, .ts, .rs, .java, etc.): Read-only monospace Entry with line numbers, horizontal scroll
- **Text files** (.txt, .log, .csv, .ini, .env, etc.): Word-wrapped read-only view
- **Markdown** (.md): Rich text rendering with GFM table support
- **Mermaid diagrams**: Rendered via mermaid.ink API
- **Images** (.png, .jpg, .gif, .svg, .webp): Scaled to fit
- **HTML** (.html, .htm): Local static server + browser preview with view-source fallback
- **Binary files**: File info (name, size, type, modified date)
- **Large files** (>1MB): "Too large to preview" message
- Close via X button or Escape key
- Preserves 75:25 sidebar ratio

### Chat Integration
- Bare file paths in agent messages auto-linkified (workspace-relative)
- Click to open file preview, target line scroll-to support

### About Dialog
- Help → About shows app icon, version, description, links
- Help → Check for Updates queries GitHub API for latest release

## Bug Fixes
- **DingTalk WS concurrent write**: Added `writeMu` mutex for serialized ping + message writes
- **DingTalk/WhatsApp goroutine leaks**: Replaced bare goroutines with `safego.Go`
- **Desktop QR buttons**: Fixed missing QR code buttons in IM Settings (WeChat/WhatsApp/QQ/Telegram/Discord/Slack/Signal/WeCom)
- **Flaky test**: `TestCommandJobPollingNoDuplicateLines` timing fix

## Testing
- 75 UI automation tests (Fyne test package)
- Tests now verify rendering output (not just "no panic")
- Code preview tests check Entry widget type, line numbers, content integrity
- HTML preview tests verify HTTP server lifecycle
