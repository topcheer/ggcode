# ggcode v1.1.23

## Highlights

- Added more built-in OpenAI-compatible vendors so common router-style endpoints are available out
  of the box.
- Reworked the `/provider` panel layout to stay compact and predictable, including visible API key
  env-var hints and a stable five-row model window.
- Saving endpoint credentials or base URLs in the `/provider` panel now refreshes models
  immediately, so newly configured endpoints are usable without leaving the panel.

## Fixes and improvements

### Built-in vendor catalog

- Added built-in presets for:
  - `aihubmix` → `https://aihubmix.com/v1`
  - `getgoapi` → `https://api.getgoapi.com/v1`
  - `novita` → `https://api.novita.ai/openai/v1`
  - `poe` → `https://api.poe.com/v1`
  - `requesty` → `https://router.requesty.ai/v1`
  - `vercel` → `https://ai-gateway.vercel.sh/v1`
- Added default API key env-var mappings for those vendors and updated the bundled example config.

### Provider panel UX

- Tightened the vendor/endpoint layout so the panel uses less vertical space.
- Added an env-var hint row showing which variable the currently selected vendor/endpoint expects.
- Fixed the left vendor column and right model column sizing so the panel does not jump around as
  model lists change.
- Limited the provider model viewport to five visible entries for a more stable layout.

### Provider model refresh flow

- Trigger model discovery immediately after saving an endpoint API key or endpoint base URL.
- Kept the existing asynchronous refresh flow and success/error messaging, but moved it closer to
  the save action so newly configured providers are ready right away.

## Upgrade notes

- No migration is required.
- Existing configs continue to work.
- To use the new router presets, set the corresponding API key env var or override the endpoint key
  directly in `/provider`.
