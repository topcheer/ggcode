# v1.3.44

## Features

### Per-Endpoint Usage & Metrics Aggregation

- **Endpoint-scoped stats**: Usage and metrics are now aggregated per `vendor/endpoint` pair. When using multiple endpoints in a session, each endpoint's data is tracked independently.
- **`Session.EndpointUsage` / `Session.EndpointMetrics`**: New maps for per-endpoint aggregated data, rebuilt from JSONL history on load.
- **`UsageForEndpoint()` / `MetricsForEndpoint()`**: Return stats for the current endpoint only. TUI sidebar, Desktop, and Daemon bridge now display current-endpoint data instead of total session data.
- **Metric events enriched**: Each `MetricEvent` now includes `Model`, `Vendor`, and `Endpoint` fields.

### Model Discovery Cache

- **Disk cache with 6h TTL**: Model list API responses are cached to avoid repeated network calls on startup and model picker refresh.
- **Mutex-protected**: Thread-safe cache with transparent fallback to network fetch on cache miss.

### Vendor-Specific Auth Headers

- **Xiaomi MiMo support**: Auto-detects MiMo base URL and uses `api-key` header instead of `Authorization: Bearer`.
- **Extensible**: `vendorSpecificAuthHeaders()` function can be extended for other vendors.

## Bug Fixes

- **Copilot provider**: `NewCopilotProvider` was not passing `apiKey` to `NewOpenAIProviderWithConfig`, causing authentication failures for Copilot provider.
- **Flaky test**: `TestPanicHookFires` in `internal/safego` could fail under heavy CI scheduler pressure. Replaced atomic counter with channel-based synchronization.
