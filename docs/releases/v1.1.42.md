# ggcode v1.1.42

## Features

### Session Checkpoints

Summarize compaction now persists the compacted message state as a **checkpoint** in the session JSONL file. When you `--resume` a session, ggcode loads from the latest checkpoint instead of re-compacting the entire history. This makes session recovery near-instant even for long conversations.

### Improved Auto-Compact Strategy

- **Microcompact** no longer replaces tool results with opaque placeholders. Instead, it preserves the first 10 and last 5 lines so the LLM retains enough context to avoid redundant operations.
- **Summarize prompt** is now optimized for coding contexts: preserves key decisions, file paths, code structure, error resolutions, and pending work.
- **Lower thresholds** (75%/65%) trigger compaction earlier, leaving more buffer before hitting the context window limit.
- **Summary payload** pre-truncates large tool results to prevent the summary request itself from triggering overflow.

### IM Tool Call Display

All IM adapters (Telegram, QQ, Discord, Slack, DingTalk, Feishu) now use a unified format with emoji icons, ✓/✗ status, and code blocks with no truncation:

```
✓
```bash
ls -la
```
```
total 32
drwxr-xr-x  5 user staff 160 Jul 10 12:00 .
```
```

Path relativization now works correctly in both TUI and daemon modes.

### ask_user in IM

IM mode now sends questions one at a time and collects answers interactively, matching IM's conversational UX. TUI mode still shows all questions in tabs for batch answering.

### Provider Error Detection

- **Anthropic**: detect `stop_reason=max_tokens` and `stop_reason=refusal`
- **Gemini**: detect `FinishReason=MAX_TOKENS`, `SAFETY`, `RECITATION`, etc.
- Default `max_output_tokens` raised from 8192 to 16384

## Fixes

- Fix IM emitter `workDir` being empty in TUI mode (used `autoCompleteWorkDir` which was not set at init time)
- Fix ask_user text-type answers lost in TUI mode when answered remotely via IM (`saveActiveQuestionInput` overwrote remote answers)
- Fix duplicate ask_user messages in daemon mode (`StreamEventDone` and `HandleAskUser` both emitted)
- Fix sub-agent panic causing `Wait()` to hang forever (added panic recovery)
- Fix skill tool fork execution missing timeout (now 5 minutes)
