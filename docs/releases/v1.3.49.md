# v1.3.49

## Features

### Share Protocol v2 (End-to-End Encrypted Session Sharing)

New end-to-end encrypted share protocol for securely sharing sessions between devices via the relay server.

- **`internal/tunnel/share_protocol.go`** — Full share v2 protocol implementation: ticket request, relay-issued share tickets, peer discovery, encrypted handshake
- **`ggcode-relay/share_auth.go`** — Relay-side share token signing and verification with Ed25519 signatures, replay protection, and expiry enforcement
- **`internal/tunnel/session.go`** — Extended session management supporting multi-session switching and share ticket lifecycle
- **`internal/tunnel/relay_client.go`** — Relay client v2 handshake with share ticket authentication

### Relay Share Ticket Issuance

The relay server now issues cryptographically signed share tickets that clients use to authenticate share requests, preventing unauthorized session access.

- **`ggcode-relay/relay.go`** — New share ticket issuance endpoint
- **`ggcode-relay/share_auth_test.go`** — Full test coverage for token signing, verification, and expiry
- **`internal/tunnel/share_protocol_test.go`** — Protocol-level tests for ticket request/issue flow

### Flutter Connection v2

Flutter mobile client rewritten to support the new share v2 protocol:

- **`connection_service.dart`** — New connection service with v2 handshake support
- **`connection_provider.dart`** — Updated provider with share-aware session management

### Desktop & TUI Adaptation

- Desktop app and TUI tunnel module adapted for the new connection flow
