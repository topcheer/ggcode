# v1.3.49

## Security Fixes

- **Config file permissions**: All config files (ggcode.yaml, instance configs, A2A migration) are now written with 0700 directory and 0600 file permissions via atomic write.
- **WebUI auth timing attack**: Token comparison uses SHA256 + constant-time compare (`subtle.ConstantTimeCompare`) instead of plain string equality.
- **Relay /nuke auth**: The `/nuke` endpoint now requires `GGCODE_RELAY_ADMIN_TOKEN` env var. Supports `Authorization: Bearer` and `X-GGCode-Admin-Token` headers. Returns 503 when token is not configured.
- **DingTalk log redaction**: All accessToken, ticket, webhook, and clientSecret values are redacted from debug logs and error messages (recursive JSON + URL query param redaction).
- **Desktop temp files**: Clipboard paste images and dock icon now use `os.CreateTemp` with 0600 permissions instead of predictable fixed paths.

## Race Condition Fixes

- **Session endpoint stats**: `EndpointUsage`/`EndpointMetrics` maps now protected by `sync.RWMutex`. `RebuildEndpointStats` atomically swaps maps under write lock. Read paths use `RLock`.
- **Relay room lifecycle**: Fixed lock ordering (`hub.mu` -> `room.mu`) in `removeRoomIfEmpty`, `scheduleRoomExpiry`, `expireRoom`. `stopOfflineTimer` always called under `room.mu`. `notifyRoomRecovering` no longer resurrects destroyed rooms.
- **TokenUsage value semantics**: `TokenUsage.Add` changed from pointer mutation to value-receiver returning new value. All call sites updated.
- **Metrics Collector lifecycle**: `NewCollector` accepts `context.Context`. `Stop()` now blocks until drain completes. All three entrypoints cancel context before Stop.

## Other Improvements

- Relay `destroyRoom` now also cleans `relay_client_cursors` SQLite table.
- TUI metrics digest uses tunneled system message channel.
- Flutter `workspace_cache` errors reported instead of silently swallowed.
- Deleted stale Android `MainActivity.kt` from old package path.
- New Flutter sanity test validates SDK floor and file cleanup.
