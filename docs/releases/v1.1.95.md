# ggcode v1.1.95

## Highlights

- **Onboard wizard — complete redesign** — First-time users get a polished 6-step setup wizard: language → vendor → endpoint + API key → model (with online discovery) → optional settings → IM channels. Supports English and Chinese UI. Auto-saves and restarts on completion.

## Fixes and improvements

- **Onboard**: 6 steps with clean left-aligned TUI, i18n support (EN/中文), color-coded permission modes with descriptions.
- **Onboard**: Model step tries `DiscoverModels` API for live model list, falls back to static presets, max 20 shown with filter.
- **Onboard**: IM step supports Telegram (bot token), Discord (bot token), QQ (App ID + App Secret), WeChat (prompts to use TUI for QR scan).
- **Onboard**: Knight/A2A only enabled when user explicitly opts in (no accidental `Disabled: true`).
- **Onboard**: File split into 4 focused files: `onboard.go`, `onboard_i18n.go`, `onboard_update.go`, `onboard_view.go`.
- **Config**: `NeedsOnboard()` correctly detects unresolved `${VAR}` references via `os.LookupEnv`.
- **Config**: `VendorPresets()` returns sorted list with `BaseURL` for model discovery.
- **WebUI**: IM adapter disable/enable persists to config (was in-memory only).
- **WebUI**: TUI-mode WebUI now has IM action handler (was missing entirely).
- **WebUI**: IM status reads config as source of truth for disabled state.

## Tests

- 11 onboard config tests covering: nil config, empty vendor, missing endpoint, unset env vars, resolved `${VAR}` refs, DefaultConfig with cleared env, partial config round-trip, all vendor preset NeedsAPIKey validation.

## Upgrade notes

- No breaking changes.
- Existing configs unaffected — onboard only triggers when no working LLM provider is configured.
