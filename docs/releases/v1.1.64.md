# ggcode v1.1.64

## Highlights

Fixes the last consistency bug found in Copilot review: session index repair results being silently overwritten.

## Fixes

### Session index repair result preserved
- **store.go**: `loadIndex()` called `repairIndex()` when `indexDirty` but discarded the result, returning the stale pre-repair `idx`. Subsequent `updateIndex()`/`removeFromIndex()` calls would then `saveIndex()` the stale data, wiping out the repair.
- Fix: After `repairIndex()` writes the merged index to disk, `loadIndex()` re-reads the index file to return the repaired data. Callers now always get the correct, up-to-date index.

## Review closure

All issues identified across 3 rounds of Copilot code review are now resolved. Remaining items are documented design decisions and architectural debt (entry-path consistency across root/pipe/daemon), not bugs.
