# ggcode v1.1.68

## Highlights

Advances the harness auto-run feature through Phase 4–6: shared RunService, strict isolation enforcement, review/promote CTA, bypass flag, and `/harness auto` command. Remaining gaps: suggest mode UX, RunService coverage for manual runs, full review/promote workflow.

## Changes

### Phase 4: Shared RunService
- New `RunService` in `internal/harness/run_service.go` wraps `RunTaskWithOptions` with timeout management and post-run CTA generation.
- `RunServiceInput` carries project, config, goal, runner, and options.
- `RunServiceResult` includes `RunSummary`, error, `CTAAction`, and `CTAMessage`.
- TUI `executeAutoHarnessRun` and pipe `runPipeHarness` both use RunService.
- CTA actions: `CTAReview`, `CTAPromote`, `CTARerun`, `CTAInspectLog`, `CTAInspectReport`, `CTANone`.
- `FormatRunServiceResult` renders summary + CTA in one string for pipe output.

### Phase 5: Strict isolation enforcement
- **Worktree propagation verified**: `ShouldAutoRun` returns `AutoRunResult.Config` with `WorktreeMode: "required"` in strict mode. This config is passed to the actual run (not re-read from disk).
- **Write guard**: `applyStrictWriteGuard()` calls `SetOverride("write_file", Deny)`, `SetOverride("edit_file", Deny)`, `SetOverride("multi_edit_file", Deny)` on the real `ConfigPolicy`.
- **Bypass flag**: `--no-harness` flag on pipe mode skips auto-run routing entirely.
- **`/harness auto` command**: `/harness auto off|suggest|on|strict` sets auto-run mode at runtime.
- **Never auto-promote**: `PromoteTask` requires `ReviewStatus == ReviewApproved` — enforced by `taskPromotionReady()`.

### Phase 6: Review/promote UX with CTA
- **Post-run CTA in TUI**: After `harnessRunResultMsg`, the handler appends the CTA message to the rendered summary. Shows exact slash commands: `/harness review approve t-123 --note "lgtm"`, `/harness promote apply t-123`, `/harness rerun t-123`.
- **CTA generation logic**: Completed+verification passed → review. Approved → promote. Failed → rerun with log path.
- **Daemon informational logging**: `checkAutoRunSuggestion()` truncates long inputs for clean log lines.

### Design doc updated
- `docs/design/harness-auto-run-design.md` status changed from "Proposed" to "Implemented (v1.1.68)".
