# ggcode v1.1.54

## Highlights

- **Knight status API for WebUI** — two new REST endpoints expose Knight agent status, skill data, and candidate queue to the WebUI frontend.

## New APIs

- `GET /api/knight` — full Knight status: enabled/running state, budget usage, active skills, staging skills, and candidate queue
- `GET /api/knight/skills` — lightweight endpoint returning only active and staging skills
