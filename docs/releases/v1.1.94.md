# ggcode v1.1.94

## Highlights

- **Onboard wizard** — First-time users are now greeted with a step-by-step setup wizard instead of a cryptic error. The wizard guides through: language selection → AI provider → API key → model → optional settings (permission mode, Knight, A2A). On completion the config is saved and ggcode restarts automatically.

## Fixes and improvements

- **Onboard**: Detects when no working LLM config exists (missing vendor, missing API key, unresolved endpoint) and launches the wizard automatically.
- **Onboard**: Skipped in non-interactive modes (pipe, daemon, resume, bypass flag).
- **Onboard**: Supports all 23 built-in vendor presets with real-time model discovery via API where available.
- **Onboard**: Optional configuration step for permission mode, Knight agent, and A2A server.
- **Onboard**: Uses existing `applyFirstLaunchAnthropicBootstrap` as fallback — if bootstrap succeeds, wizard is skipped.
- **WebUI**: IM adapter disable/enable now persists to config (was only in-memory).
- **WebUI**: TUI mode WebUI now has IM action handler (was missing entirely).
- **WebUI**: IM status correctly reads config as source of truth for disabled state.

## Upgrade notes

- No breaking changes.
- Existing configs are unaffected — onboard only triggers when no working LLM provider is configured.
