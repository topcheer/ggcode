# v1.3.37

## Relay + Mobile Reconnect Overhaul

This release fixes a critical chain of issues where mobile clients could not recover conversation history after relay restarts or app reconnections.

### Relay

- **ACK-based cursor protocol**: Relay and mobile now use an explicit ACK protocol for replay event deduplication. Events already cached on the device are skipped and ACKed, preventing duplicate rendering.
- **`active_session` on reconnect**: Relay now sends an `active_session` message (containing only `session_id`) before replaying events. Mobile uses this to restore its local SQLite snapshot before processing replay.
- **Server reconnect sends `active_session`**: When desktop reconnects to relay after a restart, the broker now re-announces `active_session` to trigger relay hydration from SQLite. Previously, relay never learned the sessionID and never hydrated history.
- **Session change hydrate**: When `handleServerBroadcast` detects a sessionID change, it clears old history and loads the new session's history from DB.
- **12-hour expiry**: Both DB cleanup and in-memory room eviction now use 12-hour retention (was 72h). Rooms with no events for 12 hours are cleaned up. DB and memory cleanup run as independent hourly goroutines.
- **Dead code removal**: Removed `session_id` query param parsing from `handleWS` (URL never carried it), removed unreachable hydrate code in server connect path.

### Broker (Tunnel)

- **Event ID ordering race fix**: Eliminated a race condition where concurrent outbound events could get out-of-order event IDs.
- **Reconnect reseeds `active_session`**: Server reconnect goroutine now sends `active_session` before snapshot events, ensuring relay hydrates history from DB before processing new events.

### Mobile (Flutter)

- **Split `session_provider.dart`**: The 3500-line god file has been split into 4 focused files:
  - `connection_provider.dart` (1110 lines) — ConnectionNotifier + tunnel event handling + ACK protocol
  - `chat_provider.dart` (764 lines) — ChatMessage + ChatNotifier
  - `workspace_cache.dart` (1369 lines) — SQLite cache layer + WorkspaceCacheNotifier
  - `ui_providers.dart` (234 lines) — agentStatus/subagent/approval/ask/sessionInfo providers
  - `session_provider.dart` (5 lines) — barrel export only
- **Smart session restore on connect**: When connecting via share URL, the app now checks if the current UI already matches the persisted session. If it matches, UI is preserved and ordinal dedup skips overlapping replay events. If not, UI is cleared and restored from SQLite snapshot.
- **Ordinal dedup**: Events with ordinal <= lastAppliedEventId are skipped and ACKed instead of re-rendered.
