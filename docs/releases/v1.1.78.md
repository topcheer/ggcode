# ggcode v1.1.78

## Highlights

Copilot-optimized harness routing with async TUI, unified LLM coverage, and strict-mode correctness. Reviewed with one UX fix.

## Changes

### Copilot improvements (reviewed & approved)
- **Async TUI routing**: auto-run check now runs as `tea.Cmd` off the Bubble Tea update path. LLM classifier no longer freezes the TUI.
- **Unified LLM coverage**: all score ≥ 1 inputs go through LLM when provider available. High score ≠ complex task.
- **LLM failure fallback**: error/timeout falls back to deterministic score ≥ 3 routing (not just when provider is nil).
- **suggest mode with LLM**: suggest mode now uses LLM for complexity judgment instead of skipping it.
- **strict mode correctness**: all confident code changes route to harness (write guard already blocks main agent).
- **Complexity normalization**: only "simple"/"complex" accepted. Unknown → conservative suggest.
- **Short input threshold**: 10 runes (aligned with router), supports CJK characters.
- **Case-insensitive classification**: `strings.EqualFold` handles LLM output variance.

### Review fix
- **User input shown immediately**: before async check, user message is written to chat. No blank spinner delay.
