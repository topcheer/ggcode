# ggcode v1.2.8

## Highlights

- **Major codebase structural refactoring**: 11 files over 1500 lines each were split into focused sub-files. The largest file (`model_update.go`) went from 2441 → 587 lines (76% reduction). Total of 39 new well-scoped files created across 6 packages. Zero logic changes — pure code-move refactoring.
- **IM binding dead loop fix**: Disabled IM adapters that never had a channel bound could not be enabled (error: "no channel bound") and could not be bound (error: "is disabled"). All 13 adapter panels now auto-enable the adapter when the user explicitly tries to bind from the panel.
- **Provider retry fix**: `context.DeadlineExceeded` from HTTP client timeouts was incorrectly treated as non-retryable. Now only caller-context `DeadlineExceeded` and `Canceled` skip retry; HTTP-level timeouts correctly trigger retry with backoff.

## Bug fixes

- **Fix**: IM adapter dead loop — disabled adapters with no channel bound could not be enabled ("no channel bound") and could not be bound ("is disabled"). All 13 adapter panels (QQ, TG, DingTalk, Feishu, Slack, Discord, WeChat, WeCom, Matrix, Mattermost, IRC, Twitch, Nostr, Signal, WhatsApp) now auto-enable when user explicitly binds from panel.
- **Fix**: `isRetryableForContext()` in `provider/retry.go` treated all `ctx.Err() != nil` as non-retryable, preventing HTTP client timeouts from being retried. Now only `context.Canceled` and caller-context `context.DeadlineExceeded` are non-retryable.

## Internal

- 39 new Go source files created from structural splits
- `provider/retry.go`: `isRetryableForContext` now uses `errors.Is` for precise cancellation vs timeout distinction
- Zero functional changes — all refactoring verified by passing full test suite
