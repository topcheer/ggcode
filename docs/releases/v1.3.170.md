# ggcode v1.3.170

## Highlights

- **Karpathy golden rules**: Added concise coding golden rules to the system prompt to improve agent code quality and decision-making.
- **Resume-picker performance**: Eliminated 30-second delay when canceling or pressing Esc on the session resume picker.
- **Session migration fast-path**: Three streaming early-exit optimizations eliminate full-file scans during session migration checks.

## New Features

### Karpathy Golden Rules in System Prompt (`2cd09282`)

Added a curated set of coding golden rules (inspired by Andrej Karpathy's principles) as a section in the default system prompt. These rules guide the agent toward:

- Minimal, surgical changes
- Verification before claiming success
- Reading before editing
- Preserving existing behavior and APIs

A generalized version is also available as a standalone guide in `docs/guide/golden-rules.md`.

### Background Compression Indicator (`ba9f06af`)

The TUI status bar now shows a live indicator when background context compression is active, giving users visibility into context window management.

## Performance

### Resume-Picker 30-Second Delay Eliminated (`48cf652f`, `1fb5d302`, `2fe51d53`)

Three related fixes eliminate a 30-second delay when canceling the resume picker:

1. **Cancel leaves invalid resumeID** (`2fe51d53`): Canceling the picker left `resumeID="picker"`, which caused an error downstream. Now sets `resumeID="__new__"` sentinel.
2. **Esc falls into auto-load** (`1fb5d302`): Pressing Esc was treated as auto-load instead of cancel, triggering full session initialization. Now properly resolves to the sentinel.
3. **Picker moved before `run()`** (`48cf652f`): The picker was inside `run()`, so all post-picker initialization (including a 30s timeout path) executed even on cancel. Moving the picker before `run()` eliminates the delay entirely.

### Session Migration Fast-Path (`22d1080b`, `4de1aa35`, `369dc570`)

Three optimizations eliminate unnecessary full-file scans during session loading:

1. **Checkpoint format check** (`22d1080b`, `4de1aa35`): `hasNewFormatCheckpoint()` now streams the file and exits early when a new-format checkpoint is found, instead of reading the entire file.
2. **Timestamp backfill skip** (`369dc570`): `firstMessageHasTimestamp()` streams and exits early when the first message already has a timestamp, skipping the full backfill scan.
3. **Tests added** (`7b162216`): Coverage for the fast-scan path.

These changes combined reduce session load time from O(n) full-file reads to O(1) for sessions that don't need migration.

## Fixes

### Strategist Evidence Truncation (`6afc6c9f`)

The autopilot strategist was receiving full command output as evidence, which could exceed token limits. Now truncates to head + tail, preserving the beginning (context) and end (result/error) of output.

## Upgrade Notes

No breaking changes. All improvements are backward compatible.
