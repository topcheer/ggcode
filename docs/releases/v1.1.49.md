# ggcode v1.1.49

## Highlights

- **ACP (Agent Client Protocol) production-ready** — complete rewrite of the ACP agent loop to align with ACP spec v1. This release makes `ggcode acp` fully functional for JetBrains IDEs, Zed, and other ACP-compatible editors with proper tool call display, user interaction, and session management.

## New Features

### ACP Agent Loop (v1 spec compliance)

The ACP agent loop has been rewritten from scratch to match the [Agent Client Protocol spec v1](https://agentclientprotocol.com/):

- **Flattened SessionUpdate** — `session/update` notifications now use flat JSON fields (`toolCallId`, `kind`, `status`, `rawInput`) as direct children, matching the ACP spec. Tool calls are now visible to IDEs.
- **Formatted tool call display** — tool calls show human-readable titles in the IDE, matching the TUI's presentation. For example:
  - `read_file {"path":"/tmp/test.go"}` → `Read /tmp/test.go`
  - `run_command {"command":"go test ./..."}` → `go test ./...`
  - `edit_file {"file_path":"/tmp/new.go",...}` → `Create /tmp/new.go`
- **ask_user via IDE** — the `ask_user` tool now routes through `session/request_permission` to display choice dialogs in the IDE. Single/multi-choice questions work natively. Text questions and cancelled dialogs fall back to plain-text conversation (the LLM asks the user directly in its response).
- **CWD validation** — `session/new` now rejects invalid working directories (empty, root `/`, relative paths, nonexistent). Returns a clear error message to the IDE.

### Shared Tool Presentation

Extracted `DescribeTool()` from the TUI into a new shared package `internal/tool/labels.go`. This provides consistent tool display formatting across TUI, daemon, IM, and ACP — no more duplicated formatting logic.

## Fixes and Improvements

- **Tool call results no longer sent to IDE** — `tool_call_update` only sends status (`completed`/`failed`), not result content. This keeps the IDE display clean.
- **Removed stale SQLite references** from `GGCODE.md`.

## Architecture

```
IDE (JetBrains / Zed / ...)
    │
    │ stdin/stdout (JSON-RPC 2.0)
    │
    ▼
ggcode acp
    ├── Transport (stdio JSON-RPC, ACP spec v1)
    ├── Protocol Handler (initialize, session/*, authenticate)
    ├── Agent Loop
    │   ├── handleStreamEvent → session/update notifications
    │   ├── tool.DescribeTool → formatted titles (shared with TUI)
    │   ├── ask_user → session/request_permission to IDE
    │   └── RequestPermission → tool approval via IDE
    ├── Tool System (file ops, search, commands, git, web, MCP)
    └── Session Persistence (per-workspace JSON)
```
