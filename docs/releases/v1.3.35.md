## v1.3.35

### Mobile: Resume Cursor Loop Fix

Fixed a critical bug where the mobile client entered an infinite loop after a desktop agent loop was cancelled and new messages were sent. The tunnel cursor would get stuck at a stale value, causing the relay to replay hundreds of events repeatedly while the broker sent `snapshot_reset` in response, creating a tight cycle of replay → reset → replay.

**Root cause**: After `snapshot_reset`, the gap detection in `_shouldApplyEvent` saw discontinuous event ordinals (old replay events vs new broker snapshot events) and triggered `_beginReplayRecovery`, which sent `resume_from` with the stale cursor — exactly what `snapshot_reset` was trying to replace.

**Fix**: Added `_awaitingSnapshotProjection` flag that bypasses gap detection entirely while an authoritative snapshot is being seeded. Events flow through directly without triggering replay recovery. The flag clears when `session_info` (the anchor event of the new projection) arrives, restoring normal gap detection.

Additional defensive fixes:
- `_shouldApplyEvent` session-change branch re-marks `_hasAuthoritativeProjection = true` after clearing UI projection
- `_handleReplayWatchdogTimeout` final cache-restore fallback gated on `!_hasAuthoritativeProjection`
- `_awaitingSnapshotProjection` reset in all teardown paths (disconnect, leaveSession, permanentFailure)

### Build: Pre-push Hook Cleanup

Removed the stale `git pull --rebase` logic from the pre-push hook. This was originally added to sync CI-pushed commits (mobile build number bumps) but build numbers are now bumped manually. The rebase was redundant and added noisy log output on every push.
