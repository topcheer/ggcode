# ggcode v1.2.4

## Highlights

- **Follow panel rendering fix**: Subagent/teammate follow panel no longer shows residual characters from the main chat view. The root cause was a fall-through path that rendered the main chat list when the follow snapshot was temporarily unavailable (e.g., agent just started). Now shows a loading placeholder instead, and the follow list is rebuilt in `Update()` (not `View()`) to eliminate redundant rebuilds on every frame.
- **Grace period timer for follow strip**: Completed/failed sub-agents are now correctly removed from the follow strip after the 1-minute grace period. Previously `refreshSlots` only ran when new sub-agent events arrived, so completed agents with no further events would remain visible indefinitely. A 15-second tick timer now drives cleanup when terminal slots exist.
- **WebUI restart works in TUI mode**: The `/api/restart` endpoint previously only worked in daemon mode. TUI mode now registers a restart handler that sends `remoteRestartMsg` into the Bubble Tea event loop, triggering the same clean restart path as `/restart` slash command and IM `/restart`.

## Fixes and improvements

- **Follow panel: no fall-through to main view**: When follow mode is active but the entry list is empty, a placeholder is rendered instead of falling through to the main chat list. Prevents rendering artifacts from wider main-view content bleeding through.
- **Follow panel: immediate rebuild on activation**: All `activate()` call sites (ctrl+n, arrow keys) now synchronously call `rebuildActiveView()` so the first rendered frame has follow content instead of a blank panel.
- **Follow panel: separate rebuild and render**: `View()` only renders the cached entry list. `buildFollowList` is no longer called from the render path — it runs exclusively in `Update()` via `rebuildActiveView()`. This avoids per-frame rebuilds and snapshot access from the render path.
- **Follow strip grace period: timer-driven cleanup**: `followGraceTickMsg` (15s interval) re-runs `refreshSlots` to expire grace-period slots, auto-deactivates follow if the tracked agent is removed, and stops itself when no terminal slots remain.
- **WebUI restart in TUI mode**: `SetRestartFn` registered in `root.go` for TUI mode. Callback sends `remoteRestartMsg` via `repl.InjectRestart()`, which triggers the existing `quitting + restartRequested + tea.Quit → execRestart` pipeline.
- **Audit-driven fixes**: Error handling improvements, utility consolidation, and comprehensive test coverage for the changes above.

## Upgrade notes

- No breaking changes. All changes are backward-compatible.
