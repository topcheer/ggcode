# v1.3.13

## Gateway Incremental Resume (Mobile/Tunnel)

Complete redesign of the mobile tunnel/gateway reconnect protocol. Reconnects are now **ordered, idempotent, and incremental** — no more full chat history replays on every reconnection.

- **Event log protocol**: Every message now carries `session_id`, `event_id` (monotonic `ev-N`), and `stream_id`. The relay (gateway) maintains an in-memory event history per room and supports gap-based incremental resume.
- **`resume_hello` handshake**: On reconnect, the mobile client sends its last seen `event_id`. The relay replies with only the missing events (`incremental`), the full history (`full_history`), or signals that a snapshot is required (`snapshot_required`).
- **Idempotent deduplication**: Mobile client tracks `session_id` + `event_id` in a bounded 1000-entry set. Duplicate events are silently dropped, preventing double-rendering on reconnect.
- **Cursor persistence**: `last_event_id` and `client_id` (UUID, per-installation) are saved to SharedPreferences so resume works across app restarts.
- **Multi-client support**: Relay registers multiple mobile clients per room with independent cursors. Per-client ordered queues prevent slow clients from blocking others.
- **`snapshot_reset` replaces `chat_clear`**: Session switches now send `snapshot_reset` which rotates the `session_id` and event counter, providing a clean break without destroying connection state.

## Relay Session Persistence

- Relay now persists active rooms and event histories to a JSON store (`relay_state.json`). On restart, rooms are reloaded and clients can resume from their last cursor position.
- Store includes atomic write with rename-on-success to prevent corruption.

## Mobile Connection Status UX

- Replaced the blocking disconnect `AlertDialog` with a persistent status icon in the chat screen AppBar:
  - Green `cloud_done` = connected
  - Orange spinning `sync` = connecting/reconnecting
  - Red `cloud_off` = disconnected (tap to leave)
- Removed auto-navigate back to QR scan screen on disconnect. User must explicitly tap the red icon to disconnect.
- Increased max reconnect attempts from 10 to 30 (~10 minutes of retry window).

## Mobile Tool Call Display

- Tool calls now appear as collapsible entries in the chat message list instead of inline text.
- `tool_call` / `tool_result` event types are handled by the session provider and rendered with dedicated widgets.

## Streaming & Markdown

- Improved `stream_markdown.go` to handle partial fence blocks (triple backtick at line boundary). Prevents garbled rendering during live streaming.

## Bug Fixes

- **Double echo fix**: Mobile/webchat messages no longer appear twice in the TUI when agent is busy.
- **Graceful tunnel close**: `/unshare` and `/share stop` now send `sharing_stopped` before closing the WebSocket, ensuring mobile client receives the disconnect signal.
- **Leave flow**: Mobile "leave" action properly disconnects and navigates back without crash.
- **Broker cleanup**: Removed old `sentLog`-based replay, ACK flow control, and triple-replay paths (`ReplayToClient`, `PushChatHistory`, `OnClientConnect`). All replay now goes through the relay event log.
