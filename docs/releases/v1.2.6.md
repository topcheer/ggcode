# ggcode v1.2.6

## Highlights

- **WebUI security overhaul**: Random token authentication on all API endpoints. Every startup generates a unique 64-char hex token — no more unauthenticated access to config, sessions, or chat. API keys and secrets are sanitized in all API responses.
- **Provider retry improvements**: Fixed double retry loops (OpenAI 100x, Anthropic 90x → now 20x with exponential backoff). Added transport-level timeouts (dial/TLS/header) that don't interfere with SSE streaming. User-visible retry notifications rendered as system messages.
- **Session corruption fix**: Ctrl+C during tool execution no longer corrupts sessions. Cancelled tool_use blocks now get matching tool_result entries, so both in-memory continuation and JSONL resume work correctly.
- **API key leak prevention**: Provider SDK errors are sanitized before reaching TUI/session/IM. Removed debug function that wrote full request JSON (including keys) to predictable temp files.

## New features

- **WebUI token auth** (`internal/webui/auth.go`): All `/api/*` endpoints require Bearer token (header or `?token=` query param). SPA frontend auto-extracts token from URL hash. Token displayed in TUI and daemon startup.
- **Config API sanitization**: `api_key`, `api_secret`, `oauth_client_secret` fields masked as `***`. MCP `env`/`headers` removed. A2A `api_keys` replaced with `has_api_keys` boolean.
- **Retry notifications**: New `StreamEventSystem` type renders provider retry status as system messages (consistent style, not mixed into assistant text).
- **Sub-agent wake notification**: Main agent loop now wakes when sub-agents or swarm teammates complete, with explicit "do NOT spawn sub-agents" instruction to prevent infinite recursion.

## Bug fixes

- **Critical**: Context cancel orphaned tool_use blocks → session corruption (C-agent-1)
- **Critical**: WebUI zero authentication on all endpoints (C-web-1/2/3)
- **Critical**: OpenAI double retry 10×10=100 attempts (C-prov-1)
- **Critical**: Anthropic double retry 9×10=90 attempts (C-prov-2)
- **Critical**: dumpRequestJSON temp file leaked API keys with 0644 permissions (C-prov-3)
- **Critical**: HTTP Client no timeout — providers and MCP could hang indefinitely (C-res-1)
- **High**: API keys leaked through raw SDK error messages (H-5)
- **High**: subAgentDoneMsg could cause infinite sub-agent spawning loop (H-3)
- **High**: CancelAll data race on team Status field (H-1 swarm)
- **High**: Grace tick timer accumulation — duplicate timers per subAgentUpdateMsg (H-2 TUI)
- **Fix**: Tool icon styles unified — pending/running both show as "in progress" (orange circle)
- **Fix**: `wait_agent`/`list_agents` no longer suppressed in TUI follow panel

## Security

- WebUI endpoints require per-startup random token (crypto/rand 32 bytes)
- Config API responses mask all sensitive fields (api_key, env, headers, oauth_secret)
- Provider error messages sanitized (sk-*/Bearer patterns stripped)
- dumpRequestJSON function removed entirely
- A2A api_keys array no longer exposed in full

## Internal

- Transport-level HTTP timeouts (30s dial, 10s TLS, 30s response header) for OpenAI, Copilot, Gemini, MCP
- `sanitizeMap` recursively handles slices (MCP servers are `[]MCPServerConfig`)
- 11 new auth middleware tests + 2 sanitization tests
- Anthropic SDK `MaxRetries` set to 0 (retry handled by our loop only)
