# ggcode v1.1.72

## Highlights

Closes the promote CTA loop. Full interactive cycle: prompt → auto-run → Enter (review) → Enter (promote). Also fixes stale state cleanup and daemon nil guard.

## Changes

### Interactive promote CTA (closes the loop)
- `pendingHarnessPromote` model state set after review approval.
- **Enter** → `handleHarnessPromoteApply()` calls `PromoteApprovedTasks()`, shows result.
- **Esc** → skips promote, shows how to promote later with `/harness promote apply <id>`.
- `harnessPromoteResultMsg` handler renders promote success/failure.
- Full TUI cycle: prompt → auto-run → Enter (review approve) → Enter (promote apply) → done.

### Stale state cleanup
- New text submission clears `pendingHarnessReview` and `pendingHarnessPromote` (same as `pendingAutoRun`).
- Prevents stale review/promote prompts from triggering on later Enter/Esc.

### Daemon nil guard
- `tryHarnessAutoRun()` checks `result.Project != nil` before dereference.
- Falls through to normal agent if project is nil (defensive, shouldn't happen with current router).

### Design doc accuracy
- Readiness updated to 98%.
- "What is missing" updated: only LLM classifier (optional) and integration test coverage remain.
