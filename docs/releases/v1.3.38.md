# v1.3.38

## Fixes

### Tunnel / Mobile

- **Ask User response delivery fix**: Mobile responses to `ask_user` prompts were silently dropped because the TUI used `safego.Go` to deliver the response asynchronously, which could lose the message. Now delivers synchronously and clears pending state before sending. This is the fix for mobile users answering Ask User questions but the agent never seeing the response.
- **Tool label metadata preservation**: `BaseToolItem` now stores the original tool name (e.g. `run_command`) and raw JSON args separately from the display name. Tunnel history sends these original values to mobile, fixing garbled localized tool descriptions (e.g. Chinese characters appearing as replacement characters).
- **Broker enqueue unification**: Removed `newMessage()` which had an event ID ordering race. All enqueue paths now go through `enqueueWithBytes()` which allocates event ID and enqueues under the same lock, guaranteeing queue order matches event ID order.
- **`PrettifyToolName` Unicode fix**: Use `unicode.ToUpper` instead of `strings.ToUpper` to correctly handle non-ASCII first characters in tool names.
