## v1.3.25

### Tunnel Reliability & Agent Status Overhaul

This release fixes a critical mobile message delivery issue where streaming could freeze permanently, and redesigns the agent status system to be more reliable.

#### Tunnel Message Delivery Fixes

- **Broker senderLoop retry**: When `session.Send()` fails, messages are re-enqueued instead of silently dropped. Up to 100 retries with 1s backoff before giving up.
- **RelayClient buffer doubled**: `sendCh` increased from 256 to 512 to handle burst traffic.
- **Relay server write blocking**: Peer writes now block with a write deadline instead of dropping messages when the send channel is full.

#### Replay Gap Recovery

- **Retry mechanism**: When event ordinal gaps are detected, the mobile client retries `resume_from` up to 3 times with 2s intervals.
- **Snapshot reset fallback**: If all retries fail, the client requests a `snapshot_reset` from the relay and reconnects with a clean state.
- **Persist resume cursor before reconnect**: Prevents stale cursor from causing repeated replay failures.

#### Agent Status Redesign

- **Simplified to two states**: Agent status is now strictly `busy` or `idle`. The old multi-state system (thinking/running/waiting/error) caused flickering and missed transitions.
- **New `activity` event channel**: Transient activity text ("Collecting project knowledge...", "Running bash...") is now carried on a separate `activity` event, independent of the busy/idle lifecycle.
- **No premature idle on stream turn completion**: `StreamEventDone` no longer flips to idle — only the top-level done/error handlers in the Update loop set idle. This prevents the mobile status bar from briefly flashing idle between agent loop iterations (tool call → tool result → next LLM call).
- **Error state cleanup**: Error handlers now clear `statusToolName`/`statusToolArg`/`statusToolCount` before pushing idle, preventing stale tool info from appearing.

#### Tool Label System

- **Structured `LabelMeta`**: Tool labels now carry semantic metadata (`Type`: file/command/search/code_review, `FilePath`, `IsSubtle`) for richer mobile rendering.
- **Friendly display names**: Tool calls pushed to mobile include `displayName` (e.g., "Edit file" instead of "edit_file") and `filePath` for file operations.

#### Knight Runtime Toggle

- **`/knight on|off` now immediate**: In daemon mode, `/knight on` starts Knight immediately (persists config + calls `Knight.Enable()`). `/knight off` stops it immediately. Previously required restart.

#### Backward Compatibility

- Mobile client normalizes legacy status values (`thinking`/`running`/`waiting`/`error`) to `busy`.
- Old tunnel senders that pack activity into `status.message` are still supported — the mobile client extracts the message as activity text.
