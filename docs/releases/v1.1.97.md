# ggcode v1.1.97

## Highlights

- **6 architecture fixes** improving persistence durability, lifecycle management, and observability across Harness, Session, MCP, Swarm, and Tool subsystems.

## Fixes

### Persistence & durability

1. **Event log is now best-effort**. `recordTaskEvent()` and `recordReleasePlanEvent()` failures are logged as warnings instead of failing the operation. The main file is authoritative; the event log is an append-only audit trail. This eliminates the "two-phase commit" problem where event log write failure could incorrectly fail a successful state change.
2. **SaveTask uses atomic writes**. `os.WriteFile` replaced with `atomicWriteFile` (write-to-temp + rename) to prevent partial JSON on crash.
3. **Checkpoint fsync**. `appendRecordLine()` in session store now calls `f.Sync()` before `f.Close()`, ensuring checkpoint durability across process/machine crashes.

### Lifecycle & context

4. **MCP background startup uses cancellable context**. Instead of `context.Background()`, the TUI creates a `context.WithCancel` and cancels it during cleanup alongside `mcpMgr.Close()`. This ensures MCP retry goroutines are bound to the session lifecycle.
5. **Pre-write hook returns errors**. `SetPreWriteHook` signature changed to `func(...) error`. If the hook returns an error, the file write is aborted. `CheckpointSaver` always returns nil (in-memory only), but future hooks can enforce write guards.

### Observability & Swarm

6. **Swarm broadcast logs dropped messages**. When a teammate's inbox is full and a message is dropped, a debug log is emitted with teammate ID and count. No more silent "sent but not received".
7. **Swarm idle runner: event-driven task claiming**. Added `NotifyIdleRunners()` method that sends `task_available` hints to idle teammates. Default poller interval increased from 1s to 5s (hints provide instant response). The poller remains as fallback.
