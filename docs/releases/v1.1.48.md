# ggcode v1.1.48

## Highlights

- **Terminal display integrity hardening** — comprehensive audit eliminated all `fmt.Fprintf(os.Stderr, ...)` calls from runtime packages (`internal/`). No log output from A2A, ACP, auth, knight, or TUI code can corrupt the Bubble Tea display or daemon follow view. All diagnostics now use `debug.Log()` which writes to the debug log file instead.

## Fixes

- **hashicorp/mdns library log suppression** — `[INFO] mdns: Closing client {...}` messages silenced by passing `io.Discard` logger to `mdns.QueryParam.Logger`.
- **A2A OAuth prompts** — browser/device flow messages in `internal/auth/a2a_oauth.go` moved from stderr to debug log (daemon.go already has TUI-safe equivalents with ANSI colors).
- **ACP protocol logging** — 13 debug/diagnostic messages in `internal/acp/` (handler, agent_loop, mcp_bridge) moved to debug log.
- **Knight skill promoter** — 3 snapshot/staging warnings moved to debug log.
- **TUI restart debug** — 3 restart trace messages in `internal/tui/repl.go` moved to debug log.
- **Daemon background PID file warning** — moved to debug log.

## Verification

- `grep -rn 'fmt.Fprintf(os.Stderr,...)' internal/` returns **zero matches** (excluding daemon follow-display and pipe mode which are intentional).
- All tests pass: `go test ./internal/auth/ ./internal/knight/ ./internal/acp/ ./internal/a2a/` — 0 FAIL.
