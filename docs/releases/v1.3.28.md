## v1.3.28

### Reliable Message Delivery (Mobile Tunnel)

Mobile → Relay → Desktop now has two-layer ACK for reliable message delivery:

- **relay_ack**: Relay confirms receipt of encrypted message immediately (not persisted, not stored in history).
- **server_ack**: Desktop confirms message was submitted to agent loop, sent back through encrypted channel.
- **Message status in Flutter**: `sending` → `delivered` → `acknowledged` (relay confirmed) / `failed` (timeout).
- **5s timeout with graceful degradation**: If relay is old (no relay_ack), assumes delivered when connected — no false failures.
- **Backward compatible**: All component version combinations work. Full three-stage status only visible when Mobile + Relay are both upgraded.

### A2A Registry Cleanup

- **Dead PID files removed on startup**: `Register()` now proactively cleans all zombie files from dead processes, not just same PID+workspace. Fixes the `~/.ggcode/a2a/` directory bloat problem (19 zombie files observed).
- **Fixed `isPIDAlive` on macOS**: Replaced `Signal(nil)` (always returns "unsupported signal type" on macOS) with `syscall.Signal(0)` via `util.IsProcessAlive()`.
- **Fixed `isPIDAlive` on Windows**: Replaced forever-true stub with proper `OpenProcess(SYNCHRONIZE)` implementation using `golang.org/x/sys/windows`.

### Desktop UI

- **Message card rendering**: Accent stripe now uses pixel-level left-square right-rounded geometry for cleaner visual appearance.
