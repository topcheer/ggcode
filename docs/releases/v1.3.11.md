# v1.3.11

## Desktop GUI

- **Tool rendering alignment with TUI**: Tool headers now match TUI format exactly:
  - With LLM description field: `"description (PrettyName)"` — e.g. **Run tests (Bash)**
  - Without description: `"PrettyName"` — e.g. **Read** /tmp/test.go
- **Tool result formatting**: All render paths now apply `formatToolResult()`:
  - `multi_file_edit` / `multi_edit_file` — JSON → readable summary with filenames and edit counts
  - `read_command_output` — extracts only recent output
  - `ask_user` — formats as Q&A instead of hiding entirely
  - `web_fetch`, `web_search`, `start_command`, `stop_command`, `list_agents` — results suppressed
- **Fixed duplicate tool result rendering** in teammate/subagent panels:
  - Removed result pre-fill from tool_call events
  - Fixed tool_result matching by ToolID instead of event index
  - Added `ref.body.Refresh()` to prevent layout overlap
- **Added 4 regression tests** for tool result rendering in agent panels

## TUI

- **Ctrl+N toggle**: Follow panel now toggles open/close with Ctrl+N (previously only opened)
- **Hints enhancement**: Status bar shows "Ctrl+N follow" when panel is closed, "Ctrl+N unfollow" when open
- **Tunnel QR re-show**: Running `/share` or `/tunnel` when already connected re-opens the QR overlay
- **i18n fixes**: Replaced 3 hard-coded strings with proper i18n keys:
  - `"当前配对请求已被拒绝..."` → `pairing.rejected`
  - `"该 QQ 渠道因多次被拒绝..."` → `pairing.blacklisted`
  - `"Tunnel stopped."` → `tunnel.stopped`
- **multi_edit_file**: Added to `suppressToolResult` alongside `multi_file_edit`
