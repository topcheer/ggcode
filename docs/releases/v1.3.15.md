# v1.3.15

## Mobile Experience Overhaul

- **Workspace cache system**: New `WorkspaceCacheNotifier` persists workspaces, sessions, and message snapshots to SharedPreferences. Up to 10 workspaces with 20 sessions each and 300 messages per session.
- **Workspace switcher**: Tap the workspace name in the AppBar to open a switcher. Lists all known workspaces with their sessions. Selecting a historical session shows cached messages with a banner indicating it's a cached view (input disabled).
- **In-chat QR scanner**: New QR scanner button in AppBar allows scanning a new workspace without leaving the chat screen.
- **Bootstrap reconnect**: On app launch, automatically reconnects to the last active workspace/session.
- **Live projection**: Chat messages, subagent state, session info, and agent status are continuously persisted. On disconnect/reconnect, the cached state is displayed while reconnecting.
- **Input bar improvements**: Always enabled — send messages while agent is working. Animated pulsing border glow when agent is active. Contextual hint text for disconnected/cached/active states.

## Cron Message Display

- Cron-triggered prompts now appear as system messages in the mobile chat instead of user messages. New `MessageData` type with `text`, `display_text`, and `kind` fields. `bindRemoteUserMessage` absorbs locally-typed messages that match the remote echo.

## Tunnel Status Persistence

- `Broker.PushStatus` now persists current status in-memory. On relay reconnect, the status is re-seeded along with session info and history.
- `BrokerSnapshot` includes `Status` field. `SendSnapshot` sends session_info + history + status in one call.
- New `currentTunnelStatus()` in TUI derives status from pending approvals, questionnaires, tool activity, and loading state.

## Flutter 3.27+ Migration

- All `Color.withOpacity()` calls replaced with `Color.withValues(alpha:)`.
- Riverpod 3 migration: `withOpacity()` -> `withValues(alpha:)`, `.state = null` -> `.set(null)`.

## iOS Deploy Fix

- `remove_simulator_frameworks()` helper strips IOSSIMULATOR-only frameworks (objective_c) with re-sign using Apple Distribution certificate.
- Fixed all `__FILE__` paths to `__dir__` for reliable path resolution in Fastfile.
- Podfile `EXCLUDED_ARCHS` for release builds excludes simulator architectures.

## System Prompt

- Added 4 coding principle guidelines: reuse existing patterns, fix root causes, preserve existing behavior, validate incrementally.

## CI Release Fix

- Decoupled build and upload in release workflow. All build jobs now only build + upload artifacts. A new `upload-release-assets` job waits for all builds AND GoReleaser to complete, then uploads everything in one step. Eliminates race condition where desktop builds tried to upload before GoReleaser created the GitHub Release.
