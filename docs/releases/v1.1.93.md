# ggcode v1.1.93

## Highlights

- **IM adapter disable/enable persistence** — Disable an IM adapter from any panel (IM panel or channel panel) and the state persists to config. Restarting the daemon or opening a second instance correctly shows disabled adapters. Re-enable with the same key.

- **All panels support Ctrl+C to close** — Previously only some panels responded to Ctrl+C; now all 24 panels (IM, QQ, Telegram, Discord, Slack, DingTalk, Feishu, WeCom, WeChat, IRC, Matrix, WhatsApp, Mattermost, Nostr, Twitch, Signal, MCP, Model, Provider, Preview, Harness, Skills, Inspector, PC) close on both Esc and Ctrl+C.

- **IM tool result code blocks** — `run_command` and `start_command` tool results sent to IM channels now wrap command text in code blocks for better readability.

## Fixes and improvements

- **IM**: Disabled adapters now appear in channel panels with a "disabled" status instead of disappearing entirely, allowing re-enable from any panel.
- **IM**: IM panel reads disabled state from config (source of truth) instead of only runtime memory, fixing stale "active" display across instances.
- **IM**: `ApplyAdapterConfig` called at startup in all entry points (TUI, daemon, root) so disabled adapters are correctly restored on restart.
- **IM**: `closeActivePanel()` had an empty case for IM panel — Ctrl+C now correctly closes the IM panel via both `closeActivePanel` and `handleIMPanelKey`.
- **IM**: Signal panel had duplicate `closeIMPanel()` calls — fixed.
- **TUI**: All 15 channel panels now show disabled adapters with status and support `d` key toggle (Signal uses `t`).
- **TUI**: WhatsApp platform name added to `platformDisplayName` and `platformCNName`.
- **TUI**: `imPanelResultMsg` now forwards feedback to the currently active channel panel for visual confirmation.
- **IM**: `run_command`/`start_command`/`bash` tool call and result formatting uses triple-backtick code blocks instead of single backticks.

## Upgrade notes

- No config changes required.
- No breaking API changes.
- Existing `enabled: false` in `im.adapters.<name>` will now be correctly honored on startup.
