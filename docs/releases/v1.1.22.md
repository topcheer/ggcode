# ggcode v1.1.22

## Highlights

- Added a built-in **Aliyun Bailian Coding Plan** vendor with both `openai` and `anthropic`
  protocol presets.
- Made the right sidebar homepage render as a clickable `https://ggcode.dev` terminal hyperlink.
- Hardened the local `verify-ci` / pre-commit flow so it behaves more like CI even on machines with
  extra global Git hooks or provider env vars.

## Fixes and improvements

### Vendor catalog and configuration

- Added the new `aliyun` vendor preset with:
  - `coding-openai` → `https://coding.dashscope.aliyuncs.com/v1`
  - `coding-anthropic` → `https://coding.dashscope.aliyuncs.com/apps/anthropic`
- Added default API key env-var mapping for Aliyun DashScope via `DASHSCOPE_API_KEY`.
- Updated the bundled example config so both Aliyun coding endpoints are available out of the box.

### Local verification workflow

- Added a repo-local pre-commit hook plus `make verify-ci` for CI-aligned local checks.
- Updated the shared verification script to clear inherited provider env vars, inherited `GIT_*`
  hook context, and global git-hook configuration before running tests, which avoids local
  environment interference in nested Git test cases.

### TUI and documentation polish

- Updated the right sidebar branding link to render `https://ggcode.dev` as a clickable hyperlink.
- Refreshed README and release-process docs for current installer assets, built-in vendor coverage,
  and Aliyun Bailian configuration guidance.

## Upgrade notes

- No migration is required.
- Existing configs continue to work. To use Aliyun Bailian, set `DASHSCOPE_API_KEY` and switch to
  the built-in `aliyun` vendor.
