# ggcode v1.1.33

## Highlights

- Fixes QQ outbound routing so the default delivery path now targets QQ C2C users instead of QQ groups.

## Fixes and improvements

### QQ C2C default routing

- Changes the QQ adapter default chat type from `group` to `c2c`, so unremembered channels now send through `/v2/users/{openid}/messages`.
- Normalizes equivalent chat-type aliases such as `user`, `users`, and `c2c` to the same QQ users endpoint.
- Normalizes remembered chat types before persistence so restored bindings do not drift between synonymous values.
- Adds regression coverage for the new default and for `users` endpoint path selection.

## Upgrade notes

- If you rely on QQ C2C only, no config change is required after upgrading.
- If you explicitly configured QQ group-style delivery in adapter `chat_type` or `default_chat_type`, keep using `group` to preserve that behavior.
