# ggcode v1.1.25

## Highlights

- Ships the first stable local-LSP workflow for ggcode, with real agent-facing semantic navigation
  tools instead of text-search-only fallback.
- Fixes the session-lifetime bug that could close reused `gopls` connections between tool calls, so
  multi-step LSP runs stay connected in normal agent usage.

## Fixes and improvements

### Local LSP runtime and tools

- Added reusable LSP sessions keyed by workspace and binary instead of starting a fresh server for
  every call.
- Expanded the built-in LSP tool surface to include:
  - `lsp_hover`
  - `lsp_definition`
  - `lsp_references`
  - `lsp_symbols`
  - `lsp_workspace_symbols`
  - `lsp_diagnostics`
  - `lsp_code_actions`
  - `lsp_rename`
- `lsp_rename` now applies returned workspace edits to files directly, while still respecting the
  existing sandbox rules.

### Reliability fixes

- Fixed `lsp_symbols` parsing so both `DocumentSymbol[]` and `SymbolInformation[]` responses produce
  correct symbol positions instead of bogus `1:1-1:1` ranges.
- Added a diagnostics fallback path that also surfaces `textDocument/publishDiagnostics` results when
  servers do not return useful data from `textDocument/diagnostic`.
- Fixed the LSP process lifecycle so per-tool timeout cancellation no longer kills the reused server
  process behind the shared session.

### UX and prompting

- `/status` now surfaces detected local LSP support, resolved binaries, and install hints for
  supported languages.
- Added LSP-first system prompt guidance so the agent prefers semantic tools before broad text search
  when those tools are available.
- Added integration coverage for symbol lookup, workspace symbol search, rename, and sequential
  repo-file LSP call chains.

## Upgrade notes

- No migration is required.
- To use the semantic navigation flow, install the matching local language server for your workspace
  language (for example `gopls` for Go).
- Existing non-LSP workflows continue to work; text search remains available as fallback when no
  local server is installed.
